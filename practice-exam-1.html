Learning Target P.1
P.1: I can calculate and explain the results/effects of an attempt to match data involving algebraic data types (possibly incorporating tuples and lists) against Haskell patterns.

Given

data Spell = Hex Integer
           | Rune Char

data Portal = Closed [Char]
            | Open [Spell]

alpha, beta :: Portal
alpha = Open [Hex 7, Rune 'Q', Hex 3]
beta  = Closed ['x','y','z']

gamma :: [Portal]
gamma = [Closed ['a'], Open [Rune 'M'], Closed []]

delta :: (Spell, Portal)
delta = (Rune 'K', Open [Hex 42])

For each case below, determine what happens in an attempt to match the pattern with the indicated data.

  • If the data fails to match the pattern, write No match and briefly explain why.
  • If the data matches, give the resulting value for the indicated variable.
Q1.1
Pattern:(x:_:r) Data:gamma

Give the resulting value for r.

Show Solution
Solution

gamma is a list of 3 elements. Pattern (x:_:r) matches lists with at least 2 elements, so it matches.

  • x = first element = Closed ['a']
  • _ = second element = Open [Rune 'M'] (ignored)
  • r = everything after the first two = [Closed []]
r = [Closed []]
Q1.2
Pattern:(Open (s:_)) Data:alpha

Give the resulting value for s.

Show Solution
Solution

alpha = Open [Hex 7, Rune 'Q', Hex 3]. Outer matches Open (...), the inside is [Hex 7, Rune 'Q', Hex 3]. Inner pattern (s:_) matches a non-empty list, binding the head to s.

s = Hex 7
Q1.3
Pattern:p@(Rune c, Open ss) Data:delta

Give the resulting value for ss.

Show Solution
Solution

delta = (Rune 'K', Open [Hex 42]). As-pattern binds the whole tuple to p. Inner tuple (Rune c, Open ss) destructures both elements.

  • p = whole tuple = (Rune 'K', Open [Hex 42])
  • c = 'K'
  • ss = [Hex 42]
ss = [Hex 42]
Q1.4
Pattern:(Closed (x:y:_)) Data:beta

Give the resulting value for y.

Show Solution
Solution

beta = Closed ['x','y','z']. Outer matches Closed. Inner list is ['x','y','z'], matches (x:y:_) (at least 2 elements).

  • x = 'x'
  • y = 'y'
  • _ = ['z']
y = 'y'
Q1.5
Pattern:(Open (s1:s2:_)) Data:(gamma !! 1)

Give the resulting value for s2. (Note: gamma !! 1 is the element at index 1 of gamma.)

Show Solution
Solution

gamma !! 1 = Open [Rune 'M']. Outer Open (...) matches, inner list is [Rune 'M'], only 1 element.

Pattern (s1:s2:_) requires at least 2 elements. The list has only 1. Pattern fails.

No match. The inner pattern (s1:s2:_) requires at least 2 elements, but the list inside Open has only 1.
Q1.6
Pattern:((Closed xs):rest) Data:gamma

Give the resulting values for xs and rest.

Show Solution
Solution

gamma = [Closed ['a'], Open [Rune 'M'], Closed []]. Outer is list cons; first element must match (Closed xs), rest is rest.

First element of gamma is Closed ['a'], matches Closed xs with xs = ['a'].

Rest of the list is [Open [Rune 'M'], Closed []].

xs = ['a'], rest = [Open [Rune 'M'], Closed []]
Section 2 of 5
Learning Target P.2
P.2: I can use pattern matching and algebraic datatypes to write data-directed functions/programs.

Scenario

A coffee shop offers a few items:

  • They sell coffee in three sizes: Small, Medium, Large.
  • They sell pastries with a type (Croissant, Muffin, Scone).
  • They take custom orders with notes.

They're running a promotion:

  • Anyone ordering a Large coffee gets a free muffin added to their order.
  • Any pastry order automatically gets "warmed up" written on the notes.

We represent this with the following data types:

data Size   = Small | Medium | Large deriving (Show)

data Pastry = Croissant | Muffin | Scone deriving (Show)

data Item   = Coffee Size
            | Bake Pastry          -- a pastry item
            | Custom String        -- custom order with notes
            deriving (Show)

orderA, orderB :: Item
orderA = Coffee Large
orderB = Bake Croissant

orderC :: [Item]
orderC = [Coffee Small, Bake Scone, Custom "extra foam"]

Note: These types do not belong to the Eq class, so you cannot use == or /=, nor the show function in lieu of pattern matching.

Q2.1 Write a Haskell function

Write a function promo :: Item -> [Item] such that:

  • promo (Coffee Large) returns [Coffee Large, Bake Muffin]
  • promo (Bake p) returns [Custom "warmed up", Bake p]
  • Any other item is returned as a single-item list (unchanged).

Examples:

  • promo (Coffee Large) evaluates to [Coffee Large, Bake Muffin]
  • promo (Coffee Small) evaluates to [Coffee Small]
  • promo (Bake Croissant) evaluates to [Custom "warmed up", Bake Croissant]
  • promo (Custom "soy milk") evaluates to [Custom "soy milk"]
Show Solution
Solution
promo :: Item -> [Item]
promo (Coffee Large) = [Coffee Large, Bake Muffin]
promo (Bake p)       = [Custom "warmed up", Bake p]
promo item           = [item]

Explanation:

  • First equation matches the specific combination Coffee Large.
  • Second equation matches any Bake, binding the pastry to p and prepending the warmed-up note.
  • Third equation catches everything else (Coffee Small, Coffee Medium, Custom _) and wraps it in a single-element list.

Note the order: the specific Coffee Large must come before any general Coffee match would. Here we don't have a general Coffee equation because Coffee Small and Medium fall through to the wildcard item.

Q2.2 Write a Haskell function

Write a function countPastries :: [Item] -> Int that counts the number of Bake items in a list of items.

Examples:

  • countPastries orderC evaluates to 1 (one Bake in the list)
  • countPastries [] evaluates to 0
  • countPastries [Bake Muffin, Bake Scone, Coffee Medium] evaluates to 2
Show Solution
Solution
countPastries :: [Item] -> Int
countPastries [] = 0
countPastries (Bake _ : rest) = 1 + countPastries rest
countPastries (_ : rest)      = countPastries rest

Explanation:

  • Empty list: 0 pastries.
  • If the first item is a Bake (any pastry), count 1 and recurse.
  • Otherwise, skip this item and recurse.

Alternative using length and filter with a helper:

countPastries items = length (filter isBake items)
  where isBake (Bake _) = True
        isBake _        = False

Learning Target P.3
P.3: I can convert between the following list-processing approaches: recursion with patterns, list comprehensions, and higher-order functions (map and filter).

Consider the following two Haskell functions:

river :: [(Int, Int)] -> [Int]
river ps = [ x * y | (x, y) <- ps, x + y > 10 ]

brook :: Int -> [Int] -> [Int]
brook k []     = []
brook k (n:ns)
  | n `mod` k == 0 = (n `div` k) : brook k ns
  | otherwise      = brook k ns
Q3.1 Fill in the blanks to write an equivalent function to brook:
brook1 :: Int -> [Int] -> [Int]
brook1 k ns = map ________ (filter ________ ns)
Show Solution
Solution

Identify T and P from the recursion:

  • Keep branch: (n `div` k) : brook k ns. So T(n) = n `div` k.
  • Guard: n `mod` k == 0. So P(n) = n `mod` k == 0.

Both T and P need k, which is in scope from the outer function. Use lambdas.

brook1 :: Int -> [Int] -> [Int]
brook1 k ns = map (\n -> n `div` k) (filter (\n -> n `mod` k == 0) ns)
Q3.2 Fill in the blanks to write an equivalent function to brook:
brook2 :: Int -> [Int] -> [Int]
brook2 k ns = [ ________ | ________________ ]
Show Solution
Solution

Same T and P as before. Plug into the list-comprehension template [T n | n <- ns, P n]:

brook2 :: Int -> [Int] -> [Int]
brook2 k ns = [ n `div` k | n <- ns, n `mod` k == 0 ]
Q3.3 Write a recursive function

Write a recursive function river1 :: [(Int, Int)] -> [Int] that uses patterns and is equivalent to river.

Show Solution
Solution

From the list comprehension:

  • T((x, y)) = x * y
  • P((x, y)) = x + y > 10

Each element is a tuple, so use a nested pattern ((x, y) : rest).

river1 :: [(Int, Int)] -> [Int]
river1 [] = []
river1 ((x, y) : rest)
  | x + y > 10 = (x * y) : river1 rest
  | otherwise  = river1 rest

Explanation:

  • Base case: empty list → empty result.
  • Pattern ((x, y) : rest) matches a non-empty list, destructures the head tuple into x and y, names the remaining list rest.
  • Guard tests x + y > 10. If true, include x * y. Otherwise, skip.

Learning Target T.1
T.1: I can trace and explain the execution behavior of a recursive function defined over a tree-like data structure.

Consider the following Haskell code:

data BTree a = Empty
             | BNode a (BTree a) (BTree a)
             deriving (Show)

cedar :: BTree Integer
cedar = BNode 4 (BNode 7 (BNode 2 Empty Empty) Empty)
                (BNode 5 Empty (BNode 9 Empty Empty))

ripple :: BTree Integer -> [Integer]
ripple Empty = []
ripple (BNode v left right)
  | v > 3     = [v] ++ ripple left ++ ripple right
  | otherwise = ripple left
         4
       /   \
      7    5
     /      \
    2       9
Q4.1 Trace and explain

Trace and explain what happens when ripple cedar is evaluated.

Your answer should provide enough information to clearly demonstrate how the evaluation progresses: for each recursive call, explicitly indicate the result of the pattern matching (i.e., the bindings to variables), how the result is evaluated, and the final result of that function call.

Show Solution
Full trace
Call 1: ripple (BNode 4 L R)
  where L = BNode 7 (BNode 2 Empty Empty) Empty
        R = BNode 5 Empty (BNode 9 Empty Empty)
  Match (BNode v left right):
    v = 4, left = L, right = R
  Guard: 4 > 3 = True → take first branch
  Expr: [4] ++ ripple L ++ ripple R
  Need ripple L first:

    Call 2: ripple (BNode 7 (BNode 2 Empty Empty) Empty)
      Match: v = 7, left = BNode 2 Empty Empty, right = Empty
      Guard: 7 > 3 = True → first branch
      Expr: [7] ++ ripple (BNode 2 Empty Empty) ++ ripple Empty

        Call 3: ripple (BNode 2 Empty Empty)
          Match: v = 2, left = Empty, right = Empty
          Guard: 2 > 3 = False → otherwise
          Expr: ripple left = ripple Empty

            Call 4: ripple Empty → [] (base case)

          Back to Call 3: returns []

        Call 5: ripple Empty → [] (base case)

      Back to Call 2: [7] ++ [] ++ [] = [7]
      Returns [7].

  Need ripple R:

    Call 6: ripple (BNode 5 Empty (BNode 9 Empty Empty))
      Match: v = 5, left = Empty, right = BNode 9 Empty Empty
      Guard: 5 > 3 = True → first branch
      Expr: [5] ++ ripple Empty ++ ripple (BNode 9 Empty Empty)

        Call 7: ripple Empty → [] (base case)

        Call 8: ripple (BNode 9 Empty Empty)
          Match: v = 9, left = Empty, right = Empty
          Guard: 9 > 3 = True → first branch
          Expr: [9] ++ ripple Empty ++ ripple Empty

            Call 9: ripple Empty → []
            Call 10: ripple Empty → []

          Back to Call 8: [9] ++ [] ++ [] = [9]
          Returns [9].

      Back to Call 6: [5] ++ [] ++ [9] = [5, 9]
      Returns [5, 9].

  Back to Call 1: [4] ++ [7] ++ [5, 9] = [4, 7, 5, 9]
  Returns [4, 7, 5, 9].
Final: ripple cedar = [4, 7, 5, 9]

Note: the value 2 is never included because 2 > 3 is False. When that guard fails, we only recurse on the left subtree (which was Empty), and we don't include 2 in the result.

Section 5 of 5
Learning Target T.2
T.2: I can write correct and concise code that operates over tree-like data structures.

Setup

Recall the Haskell representation of trees:

data BTree a = Empty
             | BNode a (BTree a) (BTree a)
             deriving (Show)

birch, willow :: BTree Char
birch = BNode 'M' (BNode 'a' Empty Empty)
                  (BNode 'P' (BNode 'e' Empty Empty) (BNode 'Z' Empty Empty))

willow = BNode 'X' (BNode 'Y' (BNode 'q' Empty Empty) (BNode 'r' Empty Empty))
                   (BNode 'Z' Empty Empty)

oak :: BTree Integer
oak = BNode 10 (BNode 5 (BNode 3 Empty Empty) (BNode 8 Empty Empty))
               (BNode 20 (BNode 15 Empty Empty) (BNode 25 Empty Empty))
Q5.1 Write a recursive Haskell function

Write a recursive Haskell function

bigLeaf :: BTree Integer -> Bool

such that bigLeaf tr determines whether there is at least one leaf node in tr whose label is greater than 10.

A leaf is a node where both children are Empty.

Examples:

  • bigLeaf oak evaluates to True (leaf 15 is > 10, also 25, and 3, 8 are not but that doesn't matter)
  • bigLeaf (BNode 5 (BNode 3 Empty Empty) (BNode 7 Empty Empty)) evaluates to False (no leaf > 10)
  • bigLeaf Empty evaluates to False
Show Solution
Solution
bigLeaf :: BTree Integer -> Bool
bigLeaf Empty = False
bigLeaf (BNode v Empty Empty) = v > 10
bigLeaf (BNode _ left right)  = bigLeaf left || bigLeaf right

Three equations:

  1. Empty tree has no leaves, so False.
  2. Leaf case: check if the value is greater than 10.
  3. Internal node case: ignore the value, recurse on both subtrees with OR.

The leaf equation must come before the general BNode equation, or the wildcard would catch every BNode including leaves.

Q5.2 Write a recursive Haskell function

Write a recursive Haskell function

chop :: Integer -> BTree Integer -> BTree Integer

such that chop num tr returns the tree obtained from tr by removing any node (plus its descendants) whose label is greater than or equal to num.

Recall that removing a node is equivalent to replacing it with an empty tree.

Examples:

  • chop 15 oak evaluates to: 
    BNode 10 (BNode 5 (BNode 3 Empty Empty) (BNode 8 Empty Empty)) Empty
  • chop 5 oak evaluates to: 
    BNode 10 Empty Empty
    Wait, that's wrong. Let me recheck. If chop removes nodes >= num, chop 5 removes 10, 5, 8, 20, 15, 25, leaving only 3. So: 
    Empty
  • chop 100 oak evaluates to oak unchanged (no node >= 100).
Show Solution
Solution
chop :: Integer -> BTree Integer -> BTree Integer
chop _   Empty = Empty
chop num (BNode v left right)
  | v >= num  = Empty
  | otherwise = BNode v (chop num left) (chop num right)

Two equations plus a guard:

  1. Empty tree → Empty.
  2. If the current value is >= num, kill the whole subtree by returning Empty.
  3. Otherwise, keep the node, rebuild with chop'd subtrees.

Verification for chop 15 oak:

  • 10 >= 15? No. Keep, recurse.
  • Left: 5 >= 15? No. Keep, recurse on its children.
    • 3 >= 15? No. Both children Empty, stays as BNode 3 Empty Empty.
    • 8 >= 15? No. Stays as BNode 8 Empty Empty.
    Left returns BNode 5 (BNode 3 Empty Empty) (BNode 8 Empty Empty).
  • Right: 20 >= 15? Yes. Return Empty. (15 and 25 never visited.)
  • Result: BNode 10 (BNode 5 (BNode 3 Empty Empty) (BNode 8 Empty Empty)) Empty. ✓
