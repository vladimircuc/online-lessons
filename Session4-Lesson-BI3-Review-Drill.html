2. Loops

Three Loop Types

REPEAT n TIMESFixed count. No counter variable. Runs exactly n times.
REPEAT UNTIL (cond)Checks condition BEFORE each run. Stops when true.
FOR EACH item IN listIterates over every element. item = value, NOT index.
Trace every loop with a table. Columns for each variable, one row per iteration. This is the fastest way to avoid mistakes under time pressure.

6. Random, Simulations, Undecidable

Random & Simulations

RANDOM(a, b)Random integer a to b, INCLUSIVE. Both endpoints possible.
Possible valuesCount = b - a + 1
SimulationsModel real-world with code. Cheaper, safer, faster, repeatable.
LimitationSimplifies reality. Different runs → different results. Not exact.

Undecidable

UndecidableNo algorithm solves it for ALL inputs. Proven impossible.
Halting Problem"Will this program halt?" Cannot be answered in general.
Decidable"Is n even?" "Is list sorted?" → algorithms exist for these.

7. Top 12 Exam Traps

These are the specific things that trip students up on exam day. Know every one of these cold.

Trap 1: Lists are 1-indexed

First element is list[1], NOT list[0]. If list has 5 elements, valid indices are 1 through 5. If she codes in Python, her instinct will be wrong.

Trap 2: "3" + "5" = "35" not 8

Numbers in quotes are strings. + concatenates strings. If both sides are in quotes, it's string concatenation, not math.

Trap 3: Separate IFs vs IF/ELSE chain

Two separate IF statements BOTH get checked independently. An IF/ELSE chain stops at the first true. The exam puts separate IFs where the first one changes a value, and the second one sees the NEW value.

Trap 4: Index shift after REMOVE

After REMOVE(list, i), everything after shifts left. If you remove inside a loop and keep incrementing i, you skip elements.

Trap 5: RETURN vs DISPLAY

DISPLAY shows on screen but sends nothing back. RETURN sends a value back but shows nothing. A procedure that DISPLAYs but doesn't RETURN gives you nothing to assign to a variable.

Trap 6: RETURN exits immediately

Code after RETURN never runs. If there's a DISPLAY after a RETURN, it's dead code.

Trap 7: Binary search needs sorted data

If the question says the list is unsorted, binary search cannot be used. Period.

Trap 8: Max/Min initialized to 0

If all values are negative, max initialized to 0 stays at 0 (wrong). Always initialize to list[1].

Trap 9: = is comparison, ← is assignment

In IF(x = 5), the = checks equality. x ← 5 assigns. Don't confuse them.

Trap 10: REPEAT UNTIL checks before running

If the condition is already true when the loop starts, the body never runs. Also, if you forget to change the variable inside the loop, you get an infinite loop.

Trap 11: FOR EACH gives values, not indices

The loop variable in FOR EACH is the actual element, not its position. You can't use it to modify the list because you don't have the index.

Trap 12: RANDOM is inclusive on both ends

RANDOM(1, 6) can be 1, 2, 3, 4, 5, or 6. Both endpoints are possible. Number of values = high - low + 1.
