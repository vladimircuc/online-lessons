Conditional Probability

How does knowing that one event happened change the probability of another?

The Definition

Let A and B be two events (subsets of the sample space Ω). The conditional probability of A given B is:

P(A | B) = P(A ∩ B) / P(B)

Read P(A | B) as "the probability of A, given that B happened." The vertical bar "|" means "given" or "knowing that."

Intuitively: once we know B happened, the sample space "shrinks" to just the outcomes in B. The conditional probability asks what fraction of those outcomes in B also satisfy A.

⚠️ Important

P(B) must be greater than 0 for this to make sense. You can't condition on an event that's impossible.

Concrete Example: Two Dice

Worked Example

Roll two fair dice. Let:

  • A = "the sum is 8"
  • B = "the first die shows a 3"

Without conditioning: P(A) = 5/36 (the pairs summing to 8 are (2,6), (3,5), (4,4), (5,3), (6,2)).

With conditioning on B: if we know the first die is 3, the only way to get sum 8 is (3, 5). Out of the 6 equally likely outcomes where the first die is 3, exactly 1 gives sum 8.

P(A | B) = P(A ∩ B) / P(B) = (1/36) / (6/36) = 1/6

Knowing the first die is 3 changed the probability of "sum is 8" from 5/36 ≈ 0.139 to 1/6 ≈ 0.167. The information shifted things slightly upward.

The Chain Rule (Rearranging the Definition)

If we multiply both sides of the definition by P(B), we get the chain rule:

P(A ∩ B) = P(B) · P(A | B)

This is extremely useful: it lets us compute the probability that both events happen by breaking it into two pieces: the probability of the first event, times the conditional probability of the second given the first. We'll use this pattern repeatedly in the Karger proof.

More generally, for a sequence of events E₁, E₂, …, Eₙ:

P(E₁ ∩ E₂ ∩ … ∩ Eₙ) = P(E₁) · P(E₂ | E₁) · P(E₃ | E₁ ∩ E₂) · … · P(Eₙ | E₁ ∩ … ∩ Eₙ₋₁)

This is the generalized chain rule, and it's the exact tool we'll use to analyze Karger's algorithm step by step.

Independence

When knowing one event tells you nothing about another.

Definition: Independent Events

Two events A and B are independent if knowing B happened does not change the probability of A:

P(A | B) = P(A)

Using the definition of conditional probability, this is equivalent to:

P(A ∩ B) = P(A) · P(B)

This second form is the one you'll use most often, because it doesn't require P(B) > 0 as a prerequisite and is symmetric in A and B.

💡 Key Intuition

Independent = the events don't affect each other. Flipping a coin twice: the result of flip 1 doesn't change the probabilities for flip 2.

Not independent (dependent) = knowing one changes the other. Drawing cards without replacement: if the first card is an Ace, there are fewer Aces left, changing probabilities for the second draw.

Concrete Example

Worked Example

Roll a fair die. Let:

  • A = "the result is even" = {2, 4, 6}, so P(A) = 3/6 = 1/2
  • B = "the result is ≤ 3" = {1, 2, 3}, so P(B) = 3/6 = 1/2

A ∩ B = {2} (the only outcome that is both even and ≤ 3), so P(A ∩ B) = 1/6.

Check: P(A) · P(B) = (1/2)(1/2) = 1/4. But P(A ∩ B) = 1/6 ≠ 1/4.

A and B are NOT independent.

Knowing the die is ≤ 3 changes the probability of even: P(A | B) = P({2})/P({1,2,3}) = (1/6)/(3/6) = 1/3, which is different from P(A) = 1/2. So knowing B makes A less likely.

Independence of Random Variables

Two random variables X and Y are independent if for every pair of sets A and B:

P(X ∈ A, Y ∈ B) = P(X ∈ A) · P(Y ∈ B)

In practice: if X and Y are computed from completely separate random experiments (like two different coin flips, or two separate random edge choices), they're independent. If they share any source of randomness, they might not be.

⚠️ Common Mistake

Independent ≠ mutually exclusive. These are totally different concepts!

  • Mutually exclusive: A ∩ B = ∅ (they can't both happen). This means P(A ∩ B) = 0.
  • Independent: P(A ∩ B) = P(A)·P(B) (knowing one doesn't affect the other).

If two events both have positive probability, they CANNOT be both mutually exclusive and independent. Mutually exclusive events are actually maximally dependent: if A happens, B definitely didn't.

Quick Check

You flip two fair coins. Let A = "first coin is heads" and B = "second coin is heads." Are A and B independent?

Analyzing Karger's: The Setup

Setting up the proof by defining the right events and identifying the key claim.

The Algorithm (Quick Refresher)

Algorithm Karger(G = (V, E)):
    while |V| > 2:
        Pick edge e uniformly at random
        G ← Ge    // contract e
    return G    // edges between the 2 remaining super-vertices

Let n = |V| (number of vertices). The algorithm performs exactly n − 2 contractions. Let's call the randomly chosen edges e₁, e₂, …, eₙ₋₂ in order.

The Question

Fix a specific min-cut Δ (capital Greek letter Delta, "DEL-tuh"). This is the set of edges that cross the min-cut. Let |Δ| be the number of edges in the min-cut (i.e., the min-cut value).

The algorithm succeeds (returns Δ) if and only if none of the contracted edges belong to Δ. If we ever contract an edge from Δ, we merge two vertices that should be on opposite sides, destroying the cut forever.

🎯 What We Need to Prove
P(W returns Δ) = P(e₁ ∉ Δ, e₂ ∉ Δ, …, eₙ₋₂ ∉ Δ)

Using the chain rule from Part 2, this equals:

P(e₁ ∉ Δ) · P(e₂ ∉ Δ | e₁ ∉ Δ) · P(e₃ ∉ Δ | e₁,e₂ ∉ Δ) · … · P(eₙ₋₂ ∉ Δ | e₁,…,eₙ₋₃ ∉ Δ)

We'll bound each factor separately. This is where conditional probability becomes essential.

The Key Claim

Before bounding the first factor, we need one crucial observation about min-cuts:

Claim

If Δ is a min-cut of graph G, then every vertex v has degree at least |Δ|.

Why? Consider any single vertex v. The set of edges touching v (its "star") forms a cut: {v} on one side, everyone else on the other. This cut has value = deg(v). Since Δ is the minimum cut, its value |Δ| must be ≤ every other cut's value. So |Δ| ≤ deg(v).

In other words: deg(v) ≥ |Δ| for all v ∈ V.

Why This Matters

From the claim, we can derive a bound on the total number of edges:

2|E| = ∑v ∈ V deg(v) ≥ |V| · |Δ| = n · |Δ|

(The left side uses the handshaking lemma: every edge contributes 2 to the sum of degrees.)

Rearranging:

|E| ≥ n · |Δ| / 2     which means     |Δ| / |E| ≤ 2/n

This ratio |Δ|/|E| is the probability that a randomly chosen edge lands in the min-cut. So the probability of not hitting the min-cut is at least 1 − 2/n. This is the starting point for Step 1 of the proof.

Quick Check

A graph has 10 vertices and a min-cut of value 4. Using the claim, what is the minimum possible number of edges?

Recall: 2|E| ≥ n · |Δ|

Part 5 / 8

The Full Proof

Step by step through the probability calculation, using conditional probability and a beautiful telescoping product.

We'll walk through each step of the proof. Click "Next Step" to reveal each one, and make sure you understand it before moving on.

Step 1: First Contraction

P(e₁ ∉ Δ)

The first edge e₁ is chosen uniformly at random from all |E| edges. The "bad" edges (the ones in Δ) number |Δ|. So:

P(e₁ ∈ Δ) = |Δ| / |E|

Therefore:

P(e₁ ∉ Δ) = 1 − |Δ| / |E|

From Part 4, we showed |Δ|/|E| ≤ 2/n. So:

P(e₁ ∉ Δ) ≥ 1 − 2/n = (n − 2) / n
Step 2: Second Contraction

P(e₂ ∉ Δ | e₁ ∉ Δ)

Given that e₁ ∉ Δ (we didn't destroy the min-cut in step 1), the graph has been contracted once. The new graph Ge₁ has:

  • n − 1 vertices (one pair was merged)
  • Δ is still a min-cut of this contracted graph (since we didn't touch any edge in Δ)

By the exact same argument as Step 1, but now with n − 1 vertices:

P(e₂ ∉ Δ | e₁ ∉ Δ) ≥ 1 − 2/(n − 1) = (n − 3) / (n − 1)

This is the crucial insight: after a safe contraction, the graph shrinks by 1 vertex, but the min-cut is preserved, so the same degree-based argument applies again.

Step 3: General Step k

P(eₖ ∉ Δ | e₁, …, eₖ₋₁ ∉ Δ)

After k − 1 safe contractions, the graph has n − (k − 1) = n − k + 1 vertices. Δ is still a min-cut. So:

P(eₖ ∉ Δ | e₁, …, eₖ₋₁ ∉ Δ) ≥ 1 − 2/(n − k + 1) = (n − k − 1) / (n − k + 1)
Step 4: Multiply Everything Together

The Telescoping Product

By the chain rule, the probability that ALL n − 2 contractions avoid Δ is the product of all the conditional probabilities:

P(W returns Δ) ≥ (n−2)/n · (n−3)/(n−1) · (n−4)/(n−2) · … · 1/3

Look what happens: the numerator of each fraction cancels with the denominator two terms later! This is a telescoping product:

= (n−2)/n · (n−3)/(n−1) · (n−4)/(n−2) · (n−5)/(n−3) · … · 2/4 · 1/3

After cancellation, only two terms survive in the denominator (n and n−1) and two in the numerator (2 and 1):

P(W returns Δ) ≥ 2 / (n(n − 1))
✅ Theorem (Karger's Success Probability)

For a graph with n vertices, a single run of Karger's contraction algorithm returns the correct min-cut with probability at least:

2 / (n(n − 1)) = 2 / (n2)⁻¹ ≈ 2/n²

For n = 100 vertices, that's about 2/9900 ≈ 0.02%. Sounds terrible, but we can fix it.

Boosting the Probability

Running a bad algorithm many times to make it great. The power of repetition.

The Idea

A single run of Karger's succeeds with probability ≥ 2/n². That's small. But what if we run it N times and keep the best (smallest) cut we find?

Algorithm BoostedKarger(G, N):
    best ← ∞
    for i = 1 to N:
        cut ← Karger(G)
        if value(cut) < best:
            best ← cut
    return best

Each run is independent (fresh random choices). The algorithm fails only if all N runs fail to find the min-cut.

The Math

Let p = 2/n² be (a lower bound on) the success probability of one run. Then:

P(single run fails) ≤ 1 − 2/n²
P(all N runs fail) ≤ (1 − 2/n²)N

Since the runs are independent, we just multiply the failure probabilities. So:

P(at least one run succeeds) ≥ 1 − (1 − 2/n²)N

The Key Limit

To simplify this, we use a fundamental calculus fact:

Useful Limit
limm→∞ (1 + α/m)m = eα

e (Euler's number, "OY-lerz", ≈ 2.718) is the base of the natural logarithm. This limit appears everywhere in probability.

In our case, α = −2 and m = n². So (1 − 2/n²)n² ≈ e−2 ≈ 0.135.

Choosing N

The choice of N determines how confident we are:

Choice 1: N = n²/2
P(success) ≥ 1 − e−1 ≈ 63.2%

A single round of n²/2 runs gives >63% chance. Decent but not great.

Choice 2: N = n²
P(success) ≥ 1 − e−2 ≈ 86.5%

Almost 87%. Getting much better.

⭐ Choice 3: N = n² · ln(n)

This is the important one for the exam. Using the limit with N = n² ln(n):

(1 − 2/n²)n² ln(n) ≈ (e−2)ln(n) = e−2 ln(n) = n−2 = 1/n²

So:

P(success) ≥ 1 − 1/n²

For n = 100: P(success) ≥ 1 − 1/10000 = 99.99%. The algorithm is now almost certain to find the min-cut!

Runtime: Each run of Karger's takes O(n²) time. With N = n² ln(n) runs, total time is O(n⁴ ln n). That's polynomial!

Practice Problems

Problem 1: Conditional Probability Basics

A bag contains 5 red balls and 3 blue balls. You draw two balls without replacement.

  1. What is P(2nd ball is red | 1st ball is red)?
  2. What is P(both balls are red)?
  3. Are the events "1st ball is red" and "2nd ball is red" independent? Why or why not?

(a) If the 1st ball is red, 4 red and 3 blue remain (7 total). So P(2nd red | 1st red) = 4/7.

(b) Using the chain rule: P(both red) = P(1st red) · P(2nd red | 1st red) = (5/8) · (4/7) = 20/56 = 5/14 ≈ 0.357.

(c) NOT independent. P(2nd red) = 5/8 (by symmetry, each ball is equally likely to be in any position). But P(2nd red | 1st red) = 4/7 ≠ 5/8. Drawing the first ball changes the composition of what remains, so the events are dependent.

Problem 2: Independence Check

Roll a fair die. Let A = "result is prime" = {2, 3, 5} and B = "result is odd" = {1, 3, 5}.

  1. Compute P(A), P(B), and P(A ∩ B).
  2. Are A and B independent? Check using the product rule.
  3. Compute P(A | B). Is it different from P(A)?

(a) P(A) = 3/6 = 1/2. P(B) = 3/6 = 1/2. A ∩ B = {3, 5}, so P(A ∩ B) = 2/6 = 1/3.

(b) P(A) · P(B) = (1/2)(1/2) = 1/4. But P(A ∩ B) = 1/3 ≠ 1/4. NOT independent.

(c) P(A | B) = P(A ∩ B)/P(B) = (1/3)/(1/2) = 2/3. This is different from P(A) = 1/2, confirming dependence. Knowing the result is odd makes "prime" more likely (2/3 vs 1/2), because most odd numbers on a die happen to be prime.

Problem 3: Karger's Probability Calculation

A graph has n = 6 vertices.

  1. How many contractions does Karger's algorithm perform?
  2. What is the lower bound on the probability that a single run finds the correct min-cut?
  3. Write out the telescoping product explicitly (all 4 factors).
  4. Verify that the product simplifies to your answer in (b).

(a) n − 2 = 6 − 2 = 4 contractions.

(b) 2/(n(n−1)) = 2/(6 · 5) = 2/30 = 1/15 ≈ 6.67%.

(c) The four factors are:

(6−2)/6 · (6−3)/(6−1) · (6−4)/(6−2) · (6−5)/(6−3)
= 4/6 · 3/5 · 2/4 · 1/3

(d) 4/6 · 3/5 · 2/4 · 1/3:

Numerator: 4 · 3 · 2 · 1 = 24

Denominator: 6 · 5 · 4 · 3 = 360

24/360 = 1/15 ✓ (matches 2/(6·5) = 2/30 = 1/15).

Problem 4: Boosting Calculation

For a graph with n = 20 vertices:

  1. What is the success probability of a single Karger run?
  2. If we run Karger's N = n² = 400 times, what is the (approximate) probability of finding the min-cut?
  3. How many runs N = n² ln(n) do we need to get success probability ≥ 1 − 1/n²? What is that numerical probability?

(a) 2/(n(n−1)) = 2/(20 · 19) = 2/380 = 1/190 ≈ 0.526%.

(b) P(success with 400 runs) ≥ 1 − (1 − 2/400)⁴⁰⁰ ≈ 1 − (1 − 1/200)⁴⁰⁰ ≈ 1 − e⁻² ≈ 86.5%.

(More precisely: 1 − 1/190 per run, so (1 − 1/190)⁴⁰⁰ ≈ e⁻⁴⁰⁰/¹⁹⁰ = e⁻²·¹⁰⁵ ≈ 0.122, giving about 87.8%.)

(c) N = n² ln(n) = 400 · ln(20) ≈ 400 · 2.996 ≈ 1,198 runs.

Success probability ≥ 1 − 1/n² = 1 − 1/400 = 99.75%.

Total runtime: N · O(n²) = 1198 · O(400) = O(n⁴ ln n) = O(479,200) operations. Very manageable.

Problem 5: Chain Rule Application

A deck has 52 cards. You draw 3 cards without replacement. What is the probability that all 3 are Aces?

Use the chain rule: P(A₁ ∩ A₂ ∩ A₃) = P(A₁) · P(A₂ | A₁) · P(A₃ | A₁ ∩ A₂)

Let Aᵢ = "the i-th card drawn is an Ace."

P(A₁) = 4/52 (4 Aces in a 52-card deck)

P(A₂ | A₁) = 3/51 (3 Aces remain in 51 cards)

P(A₃ | A₁ ∩ A₂) = 2/50 (2 Aces remain in 50 cards)

P(all 3 Aces) = (4/52) · (3/51) · (2/50) = 24/132600 = 1/5525 ≈ 0.0181%

This is exactly the same pattern as the Karger proof: a chain of conditional probabilities where the "pool" shrinks each step.

Problem 6: Full Karger Pipeline

You have a graph with 50 vertices and want to find its min-cut using Karger's algorithm. You want to be at least 99% confident in the answer.

  1. What is the success probability of a single run?
  2. How many runs N do you need so that (1 − 2/n²)N ≤ 0.01?
  3. Compare this N with n² ln(n). Which is larger?

(a) 2/(50 · 49) = 2/2450 ≈ 0.0816%.

(b) We need (1 − 2/2500)N ≤ 0.01. Taking natural log of both sides:

N · ln(1 − 2/2500) ≤ ln(0.01)

N · (−0.0008003) ≤ −4.605

N ≥ 4.605 / 0.0008003 ≈ 5,754 runs.

(c) n² ln(n) = 2500 · ln(50) ≈ 2500 · 3.912 ≈ 9,780 runs, which gives P(success) ≥ 1 − 1/2500 = 99.96%.

So 5,754 runs suffices for 99%, while the n² ln(n) formula gives ~9,780 runs for 99.96%. The n² ln(n) formula overshoots a bit for 99% confidence, but it's the standard formula because the 1 − 1/n² guarantee is clean.

The Big Picture: How It All Connects

🗺️ The Story Arc

Sessions 1-6 were about dealing with NP-hard problems via approximation: we accepted giving up optimality but kept determinism (the answer is always within a guaranteed ratio of optimal).

Sessions 7-8 are about a different tradeoff: randomization. Karger's algorithm gives the exact optimal answer, but only sometimes. We give up guaranteed correctness in exchange for simplicity and speed, then use boosting (repetition) to make the failure probability negligible.

These are two of the three main strategies for coping with computational hardness. (The third, which this course may or may not cover, is parameterized algorithms.)
