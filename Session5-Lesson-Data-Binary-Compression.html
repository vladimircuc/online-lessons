Data: Binary, Compression & Errors

How computers store, represent, and compress data. This is 17-22% of the exam. Heavy on vocabulary and conversions.

1. Binary Number System

Computers store everything as binary, sequences of 0s and 1s. Each digit is called a bit. Binary is base-2, meaning each position is a power of 2.

Place Values (Right to Left)

Position76543210
Value1286432168421

Each position is double the previous. Memorize at least: 1, 2, 4, 8, 16, 32, 64, 128, 256, 512, 1024.

Converting Binary → Decimal

Add up the place values where there's a 1:

1011 in binary:

Position:  3  2  1  0
Value:     8  4  2  1
Bits:      1  0  1  1

Add the 1s: 8 + 0 + 2 + 1 = 11 in decimal

Converting Decimal → Binary

Find the largest power of 2 that fits, subtract, repeat:

Convert 45 to binary:

45 - 32 = 13  → 32 is in  (position 5: 1)
13 - 8  = 5   → 8 is in   (position 3: 1)
5 - 4   = 1   → 4 is in   (position 2: 1)
1 - 1   = 0   → 1 is in   (position 0: 1)

Fill in 0s for positions not used (4, 1):
101101 = 32 + 8 + 4 + 1 = 45 ✓
Quick way to check: Add up all the 1-positions. If they equal the original number, you're right.

Quick Check: Binary Conversions

Convert binary 11010 to decimal.
1 1 0 1 0 16+8+0+2+0 = 26
Convert decimal 100 to binary.
100 - 64 = 36 (64 in) 36 - 32 = 4 (32 in) 4 - 4 = 0 (4 in) Positions: 64, 32, 4 → 1100100 Check: 64+32+4 = 100 ✓
Convert binary 11111111 to decimal.
128+64+32+16+8+4+2+1 = 255 8 bits all set to 1 = 255. This is the maximum value for 8 bits (1 byte).

2. Bits, Bytes & Representing Values

How Many Values Can n Bits Represent?

The formula: 2ⁿ values, ranging from 0 to 2ⁿ - 1.

Working Backwards: How Many Bits Do I Need?

If you need to represent N different values, find the smallest power of 2 that's ≥ N.

Need to represent 50 different values:
  2⁵ = 32  (not enough)
  2⁶ = 64  (covers 50) → Need 6 bits

Need to represent 200 different values:
  2⁷ = 128 (not enough)
  2⁸ = 256 (covers 200) → Need 8 bits

Common Sizes

1 byte   = 8 bits
1 KB     = 1,024 bytes   (about 1 thousand)
1 MB     = 1,024 KB      (about 1 million bytes)
1 GB     = 1,024 MB      (about 1 billion bytes)
1 TB     = 1,024 GB      (about 1 trillion bytes)

Quick Check: Bits & Values

A system uses 5-bit color codes. How many unique colors can it represent?
2⁵ = 32 unique colors. Range: 0 to 31 (or binary 00000 to 11111)
A school has 1,200 students. Each student needs a unique binary ID. How many bits minimum?
2¹⁰ = 1,024 (not enough) 2¹¹ = 2,048 (covers 1,200) Need 11 bits minimum.

3. Overflow & Roundoff Errors

Overflow Error

Happens when a number is too large to fit in the available bits.

With 8 bits, max value = 255.
If you try to store 256:
  256 in binary = 100000000 (9 bits)
  But you only have 8 bits!
  → Overflow error: the value wraps around or crashes

Think of a car odometer at 999,999. One more mile and it rolls over to 000,000. Same idea.

Roundoff Error

Happens because some decimals can't be represented exactly in binary.

In decimal, 1/3 = 0.33333... (infinite)
In binary, 1/10 = 0.000110011... (infinite)

So when a computer stores 0.1:
  0.1 + 0.1 + 0.1 might NOT equal exactly 0.3
  It might be 0.30000000000000004

→ Roundoff error: tiny inaccuracies in decimal math
Overflow ≠ Roundoff. Overflow: the number is too BIG for the bits. Roundoff: the number can't be represented EXACTLY in binary. The exam will give you scenarios and ask which type of error occurs.

Quick Check: Errors

A program uses 16-bit unsigned integers (max value 65,535). What happens when it tries to compute 65,535 + 1?
Overflow error. 65,536 requires 17 bits (10000000000000000) but only 16 bits are available. The result wraps around or produces an incorrect value.
A program computes the total price of three items at $0.10 each. The result is $0.30000000000000004. What type of error is this?
Roundoff error. 0.1 cannot be represented exactly in binary. Each 0.1 has a tiny rounding error, and those errors accumulate when added. This is NOT overflow (the number isn't too big). It's a precision issue.

4. Analog vs Digital

The Difference

Analog:  Continuous. Infinite possible values.
         Example: sound waves, temperature, light

Digital: Discrete. Finite set of values (0s and 1s).
         Example: MP3 file, digital photo, text file

Analog-to-Digital Conversion (Sampling)

To convert analog data (like sound) to digital, you sample it at regular intervals:

Sound wave (analog):  ~~~~~∿∿∿~~~~~

Sample it at regular intervals:
  Point 1: value = 42
  Point 2: value = 67
  Point 3: value = 51
  ...

Store as discrete numbers: [42, 67, 51, ...]

Sampling Rate & Bit Depth

Sampling rate: How OFTEN you sample.
  Higher rate → more detail → larger file
  CD audio: 44,100 samples per second

Bit depth: How many bits per sample.
  More bits → more precise values → larger file
  CD audio: 16 bits per sample (65,536 levels)

Trade-off: Higher quality = larger file size.
           Lower quality = smaller file size.
Exam question pattern: "Which change would INCREASE the quality of a digital audio recording?" Answer: increase sampling rate OR increase bit depth. "Which change would DECREASE file size?" Answer: decrease either one.

Quick Check: Analog vs Digital

A digital audio file is recorded at 22,050 samples/second with 8-bit depth. A second recording uses 44,100 samples/second with 16-bit depth. Which has better quality? Which is larger?
The second recording has better quality (double the sampling rate AND double the bit depth). It is also MUCH larger: First: 22,050 * 8 = 176,400 bits/sec Second: 44,100 * 16 = 705,600 bits/sec 4x larger per second of audio. Higher quality always means bigger files.

5. Lossy vs Lossless Compression

Why Compress?

Files are big. Compression makes them smaller so they take up less storage and transfer faster over networks.

Two Types

LosslessLossy
Data preserved?100%, perfectly reconstructableSome data permanently lost
File size reductionModerate (maybe 50-70% of original)Dramatic (can be 10% of original)
Reversible?Yes, original fully recoverableNo, lost data is gone forever
Best forText, code, medical images, legal docsPhotos, music, video, streaming
ExamplesPNG, ZIP, FLAC, GIFJPEG, MP3, MP4, AAC

How Lossless Works (simplified)

Finds patterns and represents them more efficiently:

Original: AAAAAABBBCCCC
Compressed: 6A3B4C

Same information, fewer characters.
Can perfectly reconstruct the original.

How Lossy Works (simplified)

Removes data humans won't notice much:

JPEG: Removes subtle color differences
      your eyes can't distinguish.

MP3:  Removes frequencies outside
      normal human hearing range.

Result: Smaller file, looks/sounds "close enough"
but the removed data is GONE FOREVER.
You CANNOT recover lost data from lossy compression. If you compress a photo as JPEG and delete the original, the removed detail is gone permanently. If you keep compressing a lossy file, quality degrades further each time.

Quick Check: Compression

A hospital stores patient X-rays. Should they use lossy or lossless compression? Why?
LOSSLESS. Medical images cannot afford to lose detail. A tiny detail removed by lossy compression could be a critical medical finding. The larger file size is worth it for perfect accuracy in this context.
A streaming music service compresses songs for mobile users. Which type and why?
LOSSY (like MP3 or AAC). Mobile users need small files for streaming. The removed audio frequencies are mostly inaudible to humans, so the trade-off (smaller size for slightly less quality) is acceptable for casual listening.

6. Metadata

Data About Data

Metadata describes the properties of a file without being the content itself.

Photo metadata:
  Date taken: March 15, 2026
  Location: 30.2672° N, 97.7431° W
  Camera: iPhone 16
  File size: 3.2 MB
  Resolution: 4032 x 3024 pixels

Song metadata:
  Title: "Bohemian Rhapsody"
  Artist: Queen
  Album: A Night at the Opera
  Duration: 5:55
  Bitrate: 320 kbps

Email metadata:
  From, To, Date, Subject, IP address
  (the content of the email is NOT metadata)
Exam distinction: Metadata is information ABOUT the data, not the data itself. The photo's metadata includes when and where it was taken, but NOT what's in the picture. The email's metadata includes who sent it, but NOT what the email says.

Why Metadata Matters

1. Searching: find files by date, type, author
2. Organizing: sort photos by location
3. Privacy concern: metadata can reveal location,
   device info, and patterns even if content is hidden
4. Legal: email metadata used as evidence in court
Privacy risk of metadata. A photo might look innocent, but its metadata could reveal your exact GPS location, the time you were there, and what device you used. The exam asks about this.

Quick Check: Metadata

Which of these is metadata for a video file? A. The actual video frames B. The audio track C. The file size and duration D. The storyline
C. File size and duration are metadata. A and B are the actual content. D is a human description, not file metadata. Metadata = properties of the file (size, duration, format, creation date, etc.), not the content itself.
