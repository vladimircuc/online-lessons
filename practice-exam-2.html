Learning Target P.1
P.1: I can calculate and explain the results/effects of an attempt to match data involving algebraic data types (possibly incorporating tuples and lists) against Haskell patterns.

Given

data Ghost = Phantom Integer Char
           | Specter [Integer]

data Castle = Empty
            | Haunted Ghost [Castle]

ship, chest :: Ghost
ship  = Phantom 12 'H'
chest = Specter [5, 10, 15]

tower :: Castle
tower = Haunted (Phantom 3 'K') [Empty, Haunted (Specter [0]) []]

crew :: [Ghost]
crew = [Phantom 1 'A', Specter [], Phantom 99 'Z', Specter [7, 8]]

For each case below, determine what happens in an attempt to match the pattern with the indicated data.

  • If the data fails to match the pattern, write No match and briefly explain why.
  • If the data matches, give the resulting value for the indicated variable.
Q1.1
Pattern:(Phantom n c) Data:ship

Give the resulting values for n and c.

Show Solution
Solution

ship = Phantom 12 'H'. Pattern (Phantom n c) matches, binding:

n = 12, c = 'H'
Q1.2
Pattern:(Specter (x:y:_)) Data:chest

Give the resulting value for y.

Show Solution
Solution

chest = Specter [5, 10, 15]. Outer matches Specter (...). Inner list [5, 10, 15] matches (x:y:_).

  • x = 5
  • y = 10
  • _ = [15]
y = 10
Q1.3
Pattern:(Haunted g cs) Data:tower

Give the resulting values for g and cs.

Show Solution
Solution

tower = Haunted (Phantom 3 'K') [Empty, Haunted (Specter [0]) []]. Matches (Haunted g cs):

  • g = the Ghost = Phantom 3 'K'
  • cs = the [Castle] = [Empty, Haunted (Specter [0]) []]
g = Phantom 3 'K', cs = [Empty, Haunted (Specter [0]) []]
Q1.4
Pattern:(Haunted (Phantom n _) (_:h:_)) Data:tower

Give the resulting values for n and h.

Show Solution
Solution

Outer: Haunted (...) (...) matches. Inner ghost (Phantom n _) matches Phantom 3 'K', so n = 3.

Inner castle list (_:h:_) requires at least 2 elements. The list [Empty, Haunted (Specter [0]) []] has 2.

  • _ = Empty
  • h = Haunted (Specter [0]) []
  • _ = [] (rest)
n = 3, h = Haunted (Specter [0]) []
Q1.5
Pattern:((Phantom a _):(Specter xs):_) Data:crew

Give the resulting values for a and xs.

Show Solution
Solution

crew = [Phantom 1 'A', Specter [], Phantom 99 'Z', Specter [7, 8]]. Outer is a list cons pattern requiring at least 2 elements.

  • First element must match (Phantom a _). First element is Phantom 1 'A', matches, a = 1.
  • Second element must match (Specter xs). Second is Specter [], matches, xs = [].
  • _ = rest of list = [Phantom 99 'Z', Specter [7, 8]].
a = 1, xs = []
Q1.6
Pattern:(Specter (n:_)) Data:(crew !! 1)

Give the resulting value for n. (Note: crew !! 1 is the element at index 1 of crew.)

Show Solution
Solution

crew !! 1 = Specter []. Outer Specter (...) matches. Inner list is [], empty.

Pattern (n:_) requires a non-empty list. The list is empty. Pattern fails.

No match. The inner pattern (n:_) requires a non-empty list, but the list inside Specter is empty.
Section 2 of 5
Learning Target P.2
P.2: I can use pattern matching and algebraic datatypes to write data-directed functions/programs.

Scenario

A library lends out different types of media:

  • They lend books with a title, stored as a String.
  • They lend DVDs with a title and a runtime in minutes.
  • They lend magazines which come in three types: Weekly, Monthly, Quarterly.

The library has a late-fee policy:

  • A book returned late incurs a fee of $2.
  • A DVD returned late incurs a fee of $1 if its runtime is less than 90 minutes, or $3 otherwise.
  • A monthly magazine returned late incurs a fee of $0.50; other magazines are free.

We represent this in Haskell with the following types:

data Freq   = Weekly | Monthly | Quarterly deriving (Show)

data Item   = Book String
            | DVD String Integer
            | Mag Freq
            deriving (Show)

item1, item2, item3 :: Item
item1 = Book "Haskell Handbook"
item2 = DVD "Lord of the Rings" 178
item3 = Mag Monthly

batch :: [Item]
batch = [Book "Dune", DVD "Toy Story" 81, Mag Weekly, Mag Monthly]

Note: These types do not derive Eq, so you cannot use == or /=, nor the show function, in place of pattern matching.

Q2.1 Write a Haskell function

Write a function lateFee :: Item -> Double that computes the late fee for a given item according to the policy above.

Examples:

  • lateFee (Book "Dune") evaluates to 2.0
  • lateFee (DVD "Lord of the Rings" 178) evaluates to 3.0
  • lateFee (DVD "Toy Story" 81) evaluates to 1.0
  • lateFee (Mag Monthly) evaluates to 0.5
  • lateFee (Mag Weekly) evaluates to 0.0
Show Solution
Solution
lateFee :: Item -> Double
lateFee (Book _) = 2.0
lateFee (DVD _ runtime)
  | runtime < 90 = 1.0
  | otherwise    = 3.0
lateFee (Mag Monthly) = 0.5
lateFee (Mag _)       = 0.0

Explanation using the 4-step template:

  1. Read the type: 3 constructors (Book, DVD, Mag).
  2. One equation per constructor: plus a sub-case for Monthly mags.
  3. Destructure: DVD needs the runtime (ignore title), Mag needs the freq.
  4. Guards: DVD uses a guard on runtime.

The specific Mag Monthly case must come before the general Mag _ wildcard, otherwise it would never match.

Q2.2 Write a Haskell function

Write a function totalLateFees :: [Item] -> Double that computes the total late fees for a list of items. You may (and should) reuse lateFee.

Examples:

  • totalLateFees batch evaluates to 3.5 (Book $2 + DVD 81min $1 + Mag Weekly $0 + Mag Monthly $0.5 = $3.50)
  • totalLateFees [] evaluates to 0.0
Show Solution
Solution

Option A, using recursion:

totalLateFees :: [Item] -> Double
totalLateFees [] = 0.0
totalLateFees (x:xs) = lateFee x + totalLateFees xs

Option B, using map + sum:

totalLateFees :: [Item] -> Double
totalLateFees items = sum (map lateFee items)

Both are correct. The second is more idiomatic Haskell; the first is cleaner when you're getting started.

Learning Target P.3
P.3: I can convert between the following list-processing approaches: recursion with patterns, list comprehensions, and higher-order functions (map and filter).

Consider the following two Haskell functions:

tide :: [(String, Integer)] -> [String]
tide ps = [ s ++ "!" | (s, n) <- ps, n >= 100 ]

marsh :: String -> [String]
marsh _ [] = []
marsh ch (s:ss)
  | head s == ch = reverse s : marsh ch ss
  | otherwise    = marsh ch ss

-- Wait, marsh takes a Char, not a String. Let me fix.

Correction, here are the two functions:

tide :: [(String, Integer)] -> [String]
tide ps = [ s ++ "!" | (s, n) <- ps, n >= 100 ]

marsh :: Char -> [String] -> [String]
marsh ch [] = []
marsh ch (s:ss)
  | not (null s) && head s == ch = reverse s : marsh ch ss
  | otherwise                    = marsh ch ss

(Note: null s checks if the string is empty, and head s would crash on an empty string, so we guard against it with not (null s).)

Q3.1 Fill in the blanks to write an equivalent function to marsh:
marsh1 :: Char -> [String] -> [String]
marsh1 ch ss = map ________ (filter ________ ss)
Show Solution
Solution

From the recursion:

  • Keep branch: reverse s : marsh ch ss. So T(s) = reverse s.
  • Guard: not (null s) && head s == ch. So P(s) = not (null s) && head s == ch.

T is just reverse. P needs ch which is in scope, use a lambda:

marsh1 :: Char -> [String] -> [String]
marsh1 ch ss = map reverse (filter (\s -> not (null s) && head s == ch) ss)
Q3.2 Fill in the blanks to write an equivalent function to marsh:
marsh2 :: Char -> [String] -> [String]
marsh2 ch ss = [ ________ | ________________ ]
Show Solution
Solution

Same T and P. List comp template: [T s | s <- ss, P s]:

marsh2 :: Char -> [String] -> [String]
marsh2 ch ss = [ reverse s | s <- ss, not (null s) && head s == ch ]
Q3.3 Write a recursive function

Write a recursive function tide1 :: [(String, Integer)] -> [String] that uses patterns and is equivalent to tide.

Show Solution
Solution

From the list comp:

  • T((s, n)) = s ++ "!"
  • P((s, n)) = n >= 100

Each element is a tuple, destructure in the pattern.

tide1 :: [(String, Integer)] -> [String]
tide1 [] = []
tide1 ((s, n) : rest)
  | n >= 100  = (s ++ "!") : tide1 rest
  | otherwise = tide1 rest

The pattern ((s, n) : rest) simultaneously matches a non-empty list, destructures the head tuple, and names the remaining list.

Learning Target T.1
T.1: I can trace and explain the execution behavior of a recursive function defined over a tree-like data structure.

Consider the following Haskell code:

data BTree a = Empty
             | BNode a (BTree a) (BTree a)
             deriving (Show)

ash :: BTree Integer
ash = BNode 8 (BNode 3 (BNode 4 Empty Empty) (BNode 1 Empty Empty))
              (BNode 6 Empty (BNode 2 Empty Empty))

puzzle :: BTree Integer -> [Integer]
puzzle Empty = []
puzzle (BNode v left right)
  | v > 5     = puzzle left ++ [v] ++ puzzle right
  | otherwise = [v]
          8
        /   \
       3    6
      / \    \
     4   1    2
Q4.1 Trace and explain

Trace and explain what happens when puzzle ash is evaluated.

Your answer should provide enough information to clearly demonstrate how the evaluation progresses: for each recursive call, explicitly indicate the result of the pattern matching (i.e., the bindings to variables), how the result is evaluated, and the final result of that function call.

Show Solution
Full trace
Call 1: puzzle (BNode 8 L R)
  where L = BNode 3 (BNode 4 Empty Empty) (BNode 1 Empty Empty)
        R = BNode 6 Empty (BNode 2 Empty Empty)
  Match (BNode v left right):
    v = 8, left = L, right = R
  Guard: 8 > 5 = True → first branch
  Expr: puzzle L ++ [8] ++ puzzle R
  Need puzzle L first:

    Call 2: puzzle (BNode 3 (BNode 4 Empty Empty) (BNode 1 Empty Empty))
      Match: v = 3, left = BNode 4 Empty Empty, right = BNode 1 Empty Empty
      Guard: 3 > 5 = False → otherwise
      Expr: [3]
      Returns [3].
      (No recursion. Children 4 and 1 never visited.)

  Need puzzle R:

    Call 3: puzzle (BNode 6 Empty (BNode 2 Empty Empty))
      Match: v = 6, left = Empty, right = BNode 2 Empty Empty
      Guard: 6 > 5 = True → first branch
      Expr: puzzle Empty ++ [6] ++ puzzle (BNode 2 Empty Empty)
      Need puzzle Empty:

        Call 4: puzzle Empty → [] (base case)

      Need puzzle (BNode 2 Empty Empty):

        Call 5: puzzle (BNode 2 Empty Empty)
          Match: v = 2, left = Empty, right = Empty
          Guard: 2 > 5 = False → otherwise
          Expr: [2]
          Returns [2].

      Back to Call 3: [] ++ [6] ++ [2] = [6, 2]
      Returns [6, 2].

  Back to Call 1: [3] ++ [8] ++ [6, 2] = [3, 8, 6, 2]
  Returns [3, 8, 6, 2].
Final: puzzle ash = [3, 8, 6, 2]

Key observation: the otherwise branch returns [v] without any recursion. So when we hit node 3 (which is ≤ 5), we immediately return [3] and never visit its children 4 and 1. Same thing at node 2, but there were no children to skip.

Learning Target T.2
T.2: I can write correct and concise code that operates over tree-like data structures.

Setup

Recall the Haskell representation of trees:

data BTree a = Empty
             | BNode a (BTree a) (BTree a)
             deriving (Show)

grove, orchard :: BTree Integer
grove = BNode 4 (BNode 2 Empty Empty) (BNode 6 Empty Empty)

orchard = BNode 50 (BNode 30 (BNode 10 Empty Empty) (BNode 40 Empty Empty))
                   (BNode 70 (BNode 60 Empty Empty) (BNode 90 Empty Empty))

sapling :: BTree Char
sapling = BNode 'R' (BNode 'o' (BNode 'x' Empty Empty) Empty)
                    (BNode 's' Empty (BNode 'E' Empty Empty))
Q5.1 Write a recursive Haskell function

Write a recursive Haskell function

onlyVowels :: BTree Char -> Bool

such that onlyVowels tr returns True if every node's label in tr is a vowel (one of 'a', 'e', 'i', 'o', 'u', 'A', 'E', 'I', 'O', 'U'), and False otherwise. An empty tree should return True (vacuously).

Examples:

  • onlyVowels Empty evaluates to True
  • onlyVowels (BNode 'a' (BNode 'E' Empty Empty) (BNode 'i' Empty Empty)) evaluates to True
  • onlyVowels sapling evaluates to False (R is not a vowel)
  • onlyVowels (BNode 'a' (BNode 'z' Empty Empty) Empty) evaluates to False

You may use a helper function.

Show Solution
Solution
onlyVowels :: BTree Char -> Bool
onlyVowels Empty = True
onlyVowels (BNode v left right) = isVowel v && onlyVowels left && onlyVowels right

isVowel :: Char -> Bool
isVowel c = c `elem` "aeiouAEIOU"

Explanation using the 3 questions:

  1. Empty answer: True (vacuous truth).
  2. Data needed: v, recurse on both subtrees.
  3. Combine: AND. Current must be vowel AND both subtrees all vowels.

&& short-circuits, so if the current node fails, the recursive calls don't even run. Efficient.

Q5.2 Write a recursive Haskell function

Write a recursive Haskell function

bandPass :: Integer -> Integer -> BTree Integer -> BTree Integer

such that bandPass lo hi tr returns the tree obtained from tr by removing any node (plus its descendants) whose label is outside the range [lo, hi] inclusive. In other words, a node is removed if its label is less than lo OR greater than hi.

Recall that removing a node is equivalent to replacing it with an empty tree.

Examples:

  • bandPass 20 80 orchard evaluates to: 
    BNode 50 (BNode 30 Empty (BNode 40 Empty Empty))
         (BNode 70 (BNode 60 Empty Empty) Empty)
    (10 is below 20, 90 is above 80)
  • bandPass 0 100 orchard returns orchard unchanged (every node is in range).
  • bandPass 100 200 orchard evaluates to Empty (every node is below 100).
Show Solution
Solution
bandPass :: Integer -> Integer -> BTree Integer -> BTree Integer
bandPass _  _  Empty = Empty
bandPass lo hi (BNode v left right)
  | v < lo || v > hi = Empty
  | otherwise        = BNode v (bandPass lo hi left) (bandPass lo hi right)

Explanation using the 3 questions:

  1. Empty answer: Empty (nothing to filter).
  2. Data needed: v, left, right, plus the bounds lo and hi.
  3. Combine: if out of range, kill the whole subtree (return Empty). Otherwise rebuild with filtered subtrees.

Same pattern as prune from the real exam, just with a two-sided comparison.

Verification for bandPass 20 80 orchard:

  • 50: in range. Rebuild with filtered children.
  • 30 (left): in range. Rebuild:
    • 10: < 20. Empty.
    • 40: in range, leaf, stays as BNode 40 Empty Empty.
    Returns BNode 30 Empty (BNode 40 Empty Empty).
  • 70 (right): in range. Rebuild:
    • 60: in range, leaf, stays.
    • 90: > 80. Empty.
    Returns BNode 70 (BNode 60 Empty Empty) Empty.
  • Root result: BNode 50 (BNode 30 Empty (BNode 40 Empty Empty)) (BNode 70 (BNode 60 Empty Empty) Empty). ✓
