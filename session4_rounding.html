Part 1 / 8 — Warm-Up: Recap from Session 3

The LP Relaxation Idea (Quick Refresher)

In Session 3 we learned that many NP-hard problems can be written as Integer Linear Programs (ILP), where we want to optimize a linear objective subject to linear constraints, with the extra requirement that variables must be integers (usually 0 or 1).

ILP is NP-hard. But if we drop the integrality constraint and let variables take any real value between 0 and 1, we get a regular LP, which is solvable in polynomial time. This is called the LP relaxation.

ILP (NP-hard)
minimize Σv xv s.t. constraints... xv ∈ {0, 1}

Variables forced to 0 or 1. Exact optimal, but no poly-time algorithm known.

⇒
LP Relaxation (Polynomial)
minimize Σv xv s.t. same constraints... 0 ≤ xv ≤ 1

Variables can be fractional. Solvable in poly time, but answer may not be integer.

The Three-Step Pipeline

Today's entire session follows one recipe. Every LP-rounding algorithm does exactly these three things:

Step 1: Formulate as ILP

Write the NP-hard problem as an integer linear program. Variables are 0/1 decisions.

Step 2: Relax & Solve

Drop the integrality constraint. Solve the resulting LP in polynomial time. Get a fractional solution.

Step 3: Round

Convert the fractional solution to an integer one using a rounding rule. Analyze the approximation ratio.

🔑 Why this works: The LP relaxation optimal is always at least as good as (or equal to) the ILP optimal, because the LP has more freedom. So LPOPT gives us a bound on the true optimal, which we use to prove the approximation ratio.

For minimization: LPOPT ≤ ILPOPT ≤ Rounded solution.
For maximization: Rounded solution ≤ ILPOPT ≤ LPOPT.

Quick Self-Check

Warm-Up Question
Why is LPOPT ≤ ILPOPT for minimization?

Think about this before expanding. The answer is one sentence.

The LP relaxation allows all the same solutions as the ILP (every 0/1 vector is also a valid real-valued vector in [0,1]), plus additional fractional solutions. More options means the minimum can only be the same or lower. So LPOPT ≤ ILPOPT.

Think of it like this: if you're trying to minimize cost and I give you more options to choose from, your minimum cost can only go down (or stay the same), never go up.

Part 2 / 8 — Vertex Cover: The ILP Formulation

What is Vertex Cover?

You saw this in Session 2 when we discussed approximation algorithms. Here's the formal definition:

Vertex Cover: Given an undirected graph G = (V, E), find the smallest subset S ⊆ V such that every edge in E has at least one endpoint in S.

In other words, S "covers" every edge. For any edge (u, v), at least one of u or v must be in S. This is a minimization problem, and it is NP-hard (we showed this in Session 1 via reduction from Independent Set).

Translating to an ILP

For each vertex v ∈ V, define a binary variable:

xv = 1 if vertex v is in the cover xv = 0 if vertex v is NOT in the cover

The ILP for our example graph {A, B, C, D, E}:

minimize xA + xB + xC + xD + xE subject to: xA + xB ≥ 1 (edge A-B) xA + xC ≥ 1 (edge A-C) xB + xC ≥ 1 (edge B-C) xB + xD ≥ 1 (edge B-D) xC + xD ≥ 1 (edge C-D) xC + xE ≥ 1 (edge C-E) xD + xE ≥ 1 (edge D-E) xv ∈ {0, 1} for all v

Each constraint says: for this edge, at least one endpoint must be in the cover. The objective minimizes the total number of vertices chosen. This is an exact formulation of Vertex Cover as an optimization problem.

⚠️ Solving this ILP exactly is NP-hard. For our 5-vertex example, brute force (checking all 25 = 32 subsets) is trivial. But for a graph with 1000 vertices, checking 21000 subsets is impossible. That's why we need LP relaxation.

Part 3 / 8 — LP Relaxation & Deterministic Rounding for Vertex Cover

Step 2: Relax the ILP

We take the Vertex Cover ILP from Part 2 and replace xv ∈ {0, 1} with 0 ≤ xv ≤ 1. Now xv can be any real number between 0 and 1, like 0.5 or 0.7.

minimize xA + xB + xC + xD + xE subject to: xA + xB ≥ 1 (edge A-B) xA + xC ≥ 1 (edge A-C) ... (same 7 constraints) ... 0 ≤ xv ≤ 1 for all v ← RELAXED!

Solving this LP (using any poly-time LP solver) might give a fractional answer like:

VariableLP SolutionInterpretation
xA0.5"Half in the cover"
xB0.5"Half in the cover"
xC0.5"Half in the cover"
xD0.5"Half in the cover"
xE0.5"Half in the cover"

LP objective = 2.5. But "half in the cover" is meaningless for a real vertex cover. We need 0 or 1 values. That's where rounding comes in.

Step 3: Deterministic Rounding

The rounding rule for Vertex Cover is beautifully simple:

Rounding Rule: Include vertex v in the cover if and only if xv* ≥ 0.5 (where x* is the LP optimal solution).

In other words: round each variable to the nearest integer. If xv* ≥ 0.5, set x̄v = 1 (include v). Otherwise set x̄v = 0 (exclude v).

For our example: every variable is exactly 0.5, so every vertex rounds up to 1. We get the cover {A, B, C, D, E} with size 5. The true optimal is 3 (e.g., {B, C, D}). So our rounded answer is 5/3 ≈ 1.67 times the optimal. Not bad, but is this always at most 2 times optimal? Let's prove it.

Part 4 / 8 — Proving the 2-Approximation

The Claim

Theorem: The LP-rounding algorithm (round xv* ≥ 0.5 to 1, otherwise 0) gives a 2-approximation for Vertex Cover. That is, the rounded cover S satisfies |S| ≤ 2 · OPT, where OPT is the size of the optimal vertex cover.

The proof has two parts: (1) the rounded solution is a valid cover, and (2) the cover is at most twice the size of optimal.

Part 1: Feasibility (it IS a valid cover)

We need to show that for every edge (u, v), at least one of u or v is included in the rounded cover.

Take any edge (u, v). The LP constraint says xu* + xv* ≥ 1. Since both values are at most 1, if neither rounds up (both < 0.5), then xu* + xv* < 0.5 + 0.5 = 1. But that contradicts the constraint xu* + xv* ≥ 1.

Therefore, at least one of xu* or xv* must be ≥ 0.5, which means at least one endpoint is in our rounded cover. Edge (u, v) is covered. ✓

Key insight: The threshold 0.5 is not arbitrary. It is exactly the right value because of the constraint xu + xv ≥ 1. If both are below 0.5, their sum is below 1, violating the constraint. So at least one must be ≥ 0.5.

Part 2: Approximation Ratio (at most 2× optimal)

Let x* be the LP optimal solution, and let S = {v : xv* ≥ 0.5} be our rounded cover. We need to show |S| ≤ 2 · OPT.

The proof chains three inequalities together:

|S| ≤ 2 · Σv xv* = 2 · LPOPT ≤ 2 · ILPOPT = 2 · OPT

Let's justify each step:

|S| ≤ 2 · Σ xv*

For each v ∈ S, we know xv* ≥ 0.5, so 1 ≤ 2 · xv*. Summing over all v in S: |S| = Σv∈S 1 ≤ Σv∈S 2xv* ≤ 2 · Σall v xv*.

LPOPT ≤ ILPOPT

LP relaxation is a less constrained version of the ILP. More feasible solutions means the minimum can only be lower or equal. So LPOPT ≤ ILPOPT.

Putting them together: |S| ≤ 2 · LPOPT ≤ 2 · OPT. Done! The LP-rounding algorithm is a 2-approximation for Vertex Cover.

Approximation ratio on this instance: |S| / OPT = 3 / 2 = 1.5. The algorithm uses 3 vertices; optimal is 2. That's 1.5× optimal, well within the 2× guarantee.

Summary of the Proof Structure

The proof template for any LP-rounding approximation works the same way:

1. Show the rounded solution is FEASIBLE (satisfies all constraints) 2. Relate the rounded cost to LPOPT: cost(rounded) ≤ α · LPOPT 3. Use LPOPT ≤ OPT to conclude: cost(rounded) ≤ α · OPT For Vertex Cover: α = 2, because rounding at 0.5 can at most double the fractional cost (each xv* ≥ 0.5 contributes at most 2 · xv* to the rounded cost).

Part 5 / 8 — Set Cover: ILP & LP Relaxation

The Set Cover Problem

Set Cover: Given a universe U = {e1, e2, ..., en} of n elements, and m sets S1, S2, ..., Sm (each a subset of U), find the smallest collection of sets whose union covers all of U.

Example: Suppose you need to cover 6 towns with fire stations. Each potential station location covers some nearby towns. You want the fewest stations that cover all towns.

Set Cover as an ILP

For each set Sj, define a binary variable xj ∈ {0, 1}: whether we pick set j or not.

minimize x1 + x2 + x3 + x4 subject to: x1 + x4 ≥ 1 (element 1 in S1, S4) x1 + x2 ≥ 1 (element 2 in S1, S2) x1 + x3 ≥ 1 (element 3 in S1, S3) x2 + x4 ≥ 1 (element 4 in S2, S4) x2 + x3 ≥ 1 (element 5 in S2, S3) x3 + x4 ≥ 1 (element 6 in S3, S4) xj ∈ {0, 1} for all j

Each constraint says: for element ei, at least one set containing it must be chosen. Notice the structure is very similar to Vertex Cover, but now each element might appear in many sets (not just 2 endpoints).

The LP Relaxation

Replace xj ∈ {0, 1} with 0 ≤ xj ≤ 1. Solve in polynomial time. The LP might return fractional values like x1 = 0.4, x2 = 0.6, x3 = 0.6, x4 = 0.4.

Check constraints: element 1 is in S1 and S4, so 0.4 + 0.4 = 0.8... wait, that's less than 1! Let's use a feasible LP solution instead: x1 = 0.5, x2 = 0.5, x3 = 0.5, x4 = 0.5. All constraints become 1.0 ≥ 1. ✓

LPOPT = 2.0. The true integer optimal is also 2 (pick S1 and S2 and S3... actually, no two sets cover everything: S1 ∪ S2 = {1,2,3,4,5} misses 6. So we need at least 3 sets, e.g., {S1, S2, S3}). Let's verify: S1 ∪ S2 ∪ S3 = {1,2,3} ∪ {2,4,5} ∪ {3,5,6} = {1,2,3,4,5,6}. ✓ But can we do it with 2? S1 ∪ S4 = {1,2,3} ∪ {1,4,6} = {1,2,3,4,6}, missing 5. S2 ∪ S3 = {2,4,5} ∪ {3,5,6} = {2,3,4,5,6}, missing 1. S1 ∪ S3 = {1,2,3} ∪ {3,5,6} = {1,2,3,5,6}, missing 4. S1 ∪ S2 = {1,2,3,4,5}, missing 6. No pair works!

So ILPOPT = 3. But LPOPT = 2 (fractional). The integrality gap on this instance = 3/2 = 1.5.

⚠️ Why the 0.5 threshold doesn't work for Set Cover: In Vertex Cover, each constraint has exactly 2 variables (xu + xv ≥ 1), so rounding at 0.5 always keeps the constraint satisfied. But in Set Cover, a constraint might have just 1 variable (if element e is in only one set). If xj* = 0.4, rounding at 0.5 would set it to 0, leaving element e uncovered. We need a different rounding rule.

Part 6 / 8 — Deterministic Rounding for Set Cover

The Frequency-Based Rounding Rule

For Set Cover, the rounding threshold depends on the frequency of elements. Define:

f = the maximum frequency of any element, i.e., the maximum number of sets that any single element appears in.

In our example, every element appears in exactly 2 sets, so f = 2.

The rounding rule: include set Sj if and only if xj* ≥ 1/f.

Rounding rule: x̄j = 1 if xj* ≥ 1/f, else x̄j = 0 where f = max number of sets any element belongs to.

Why Does This Work?

Feasibility

Take any element ei. Suppose it appears in sets Sj1, Sj2, ..., Sjk where k ≤ f. The LP constraint says:

xj1* + xj2* + ... + xjk* ≥ 1

Since there are at most f terms, and their sum is ≥ 1, at least one of them must be ≥ 1/f (by the pigeonhole principle: if all were < 1/f, the sum would be < f · (1/f) = 1, a contradiction). That term rounds up to 1, so element ei is covered. ✓

Approximation Ratio

For each set Sj that we include, we know xj* ≥ 1/f, so 1 ≤ f · xj*. Summing over all included sets:

|rounded solution| ≤ f · Σj xj* = f · LPOPT ≤ f · OPT

So this is an f-approximation algorithm for Set Cover.

When f = 2

Each element in at most 2 sets. Threshold = 1/2. This is exactly Vertex Cover! Every edge (element) has exactly 2 endpoints (sets). We get a 2-approximation, matching what we proved in Part 4.

When f is large

If elements can appear in many sets, the approximation gets worse. For example, if f = 10, we get a 10-approximation. The threshold drops to 1/10, so more sets get included.

Worked Example: Rounding Our Instance

Rounding with f = 2

Our instance has f = 2 (each element appears in exactly 2 sets). Threshold = 1/f = 0.5.

S1 = {1,2,3}0.5Yes ✓1 (include)

All sets included, cover size = 4. OPT = 3. Ratio = 4/3 ≈ 1.33. Well within the f = 2 guarantee.

Because the LP solution is symmetric (all = 0.5) and threshold is exactly 0.5, every set gets included. A non-symmetric LP solution would be more selective.

Vertex Cover is a Special Case

This is an important connection. Vertex Cover is exactly Set Cover where:

  • Each "element" is an edge
  • Each "set" is the collection of edges touching a vertex
  • Every element (edge) appears in exactly f = 2 sets (its two endpoints)

So the f-approximation for Set Cover, with f = 2, gives us the 2-approximation for Vertex Cover. The two results are the same theorem!

LP rounding is more powerful than greedy for many problems because it gives us LPOPT as a provable lower bound. The greedy approach for EDP had to use a clever counting argument to get its bound, but LP rounding gets the bound "for free" from LP duality.

Connection to LP Duality

Remember from Session 3: strong duality says the LP optimal equals the dual optimal. For Vertex Cover:

Primal (Vertex Cover LP)

Minimize Σ xv, subject to xu + xv ≥ 1 for each edge. This is our LP relaxation.

Dual (Matching LP)

Maximize Σ ye, subject to Σe incident to v ye ≤ 1 for each vertex. This is a fractional matching!

Strong duality tells us: max fractional matching = min fractional vertex cover. And since LPOPT ≤ OPT (integer), the fractional matching gives a lower bound on the optimal vertex cover. This is the same Max-Flow Min-Cut connection we saw in Sessions 2-4 of the original course, just in a different setting.

(a) The key property: for each vertex v in S, we know xv* ≥ 0.5, so 1 ≤ 2 · xv*. Summing gives |S| ≤ 2 · Σ xv* = 2 · LPOPT. The factor 2 comes directly from 1/threshold = 1/0.5 = 2.

With threshold 0.3: we'd get 1 ≤ (1/0.3) · xv* ≈ 3.33 · xv*, so |S| ≤ 3.33 · LPOPT. Worse ratio! The inequality still holds, but with a larger constant.

(b) LPOPT ≤ OPT holds because the LP relaxation has strictly more feasible solutions than the ILP (every integer solution is also a valid fractional solution, but not vice versa). More options ⇒ the minimum can only decrease or stay the same. This is universal for any ILP-to-LP relaxation, not specific to Vertex Cover.

(c) Incorrect. The approximation ratio depends on the specific problem's constraint structure. For Vertex Cover, each constraint has exactly 2 variables summing to ≥ 1, which ensures the 0.5 threshold works and gives factor 2. For Set Cover with higher frequency f, the same approach gives an f-approximation, which could be much worse than 2. For some problems, LP rounding might not give any bounded ratio at all if the integrality gap is unbounded. The "2" is specific to Vertex Cover's structure (f = 2), not a universal LP rounding guarantee.
