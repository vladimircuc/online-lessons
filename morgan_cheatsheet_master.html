ASCII Quick Reference

'0'=48   'A'=65   'Z'=90   'a'=97   'z'=122   '\0'=0

Lexicographic order: digits < uppercase < lowercase. So "2apples" < "Banana" < "banana".

Strings & Arrays

C-Strings (char arrays)

char s1[] = "Hello"; // size 6: H,e,l,l,o,\0 char s2[] = {'H','i','\0'}; // valid c-string (has \0) char s3[4] = {'H','U','G','E'}; // NOT a valid c-string (no room for \0)
C-String Functions (unsafe)
strlen(s) , length (not counting \0)
strcpy(dst, src) , copy (no bounds check!)
strcat(dst, src) , concatenate
strcmp(a, b) , compare (0 = equal)
Header: <cstring>
String Object (safe)
s.length() or s.size()
s1 = s2; , copy
s1 + s2; , concatenate
s1 == s2 , compare (true/false)
Header: <string>

Why c-string is "unsafe": No bounds checking. strcpy/strcat can write past the end of the buffer (buffer overflow). String objects manage their own memory and resize automatically.

Reading Strings

cin >> name; // stops at space getline(cin, name); // reads full line (string object) cin.getline(buf, size); // reads full line (c-string/char array)
⚠️ The cin >> then getline trap
cin >> leaves a \n in the buffer. Next getline reads it as empty string.
Fix: Add cin.ignore(); between them.

Arrays

int arr[5] = {10, 20, 30, 40, 50}; arr[0] // first element (10) arr[4] // last element (50) // Passed to functions by reference automatically, must send size too

Character Functions (<cctype>)

toupper(c) tolower(c) isalpha(c) isdigit(c) isspace(c) isupper(c) islower(c)

cmath Functions

abs() pow(base,exp) sqrt() sin() cos() ceil() floor()

rand() is in <cstdlib>, NOT cmath.

Functions

Three Ways to Pass Parameters

MethodDeclarationCallModify original?
Valuevoid f(int x)f(num)No (copy)
Referencevoid f(int &x)f(num)Yes (alias)
Addressvoid f(int *x)f(&num)Yes (pointer)

Reference variable: a formal parameter that is a nickname for the actual parameter. Changes to it affect the original. Example: void change(int &ref);

Function Overloading

Rules
  • Same name, different parameter types or count
  • Return type alone does NOT count
  • Default parameters don't create new overloads
  • Different parameter NAMES don't count
int foo(char b); // original int foo(float f); // valid overload (different param type) char foo(char c); // INVALID (same param type, only return differs) int foo(char c='z'); // INVALID (same signature as original)

Identifier Rules

  • Can contain: letters, digits, underscores
  • Cannot start with a digit (1st is illegal)
  • Can start with underscore (_python is legal)
  • Cannot be a reserved word (short, int, struct, etc.)

Stream Parameters

⚠️ Streams cannot be passed by value
void f(ofstream file); ILLEGAL (copy constructor deleted)
void f(ofstream &file); LEGAL (pass by reference)
void f(istream &in, ostream &out); LEGAL (generic streams)

Switch Statement

Valid case labels
Integer literals, constant chars, const int
case 5: ✓
case 'A': ✓
Invalid case labels
Variables (non-const), floats, strings
case x: ✗ (not const)
case 3.14: ✗ (float)

Fall-through: Without break, execution continues into the next case. switch(2){ case 2: cout<<"B"; case 3: cout<<"C"; } prints "BC".

Pointer Arithmetic & Arrays

int arr[5] = {10,20,30,40,50}; int* p = arr; // same as &arr[0] // ALL of these access arr[3] = 40: arr[3] *(arr+3) *(p+3) p[3] p++; // moves to next element (adds sizeof(int) bytes) p--; // moves back one element

*arr[i] is a double dereference error if arr is int[]. arr[i] is already an int, you can't dereference an int.

Dynamic Memory

Allocate
int* p = new int; (one)
int* arr = new int[n]; (array)
Deallocate
delete p; (one)
delete[] arr; (array, MUST use [])

Why initialize to null?

Uninitialized pointers hold garbage addresses. Dereferencing = crash or silent corruption. nullptr gives a clear error and can be checked: if(ptr != nullptr).

Memory Leak

int* p = new int(5); // allocate p = new int(10); // LEAK: first allocation's address is lost forever

Leaked memory can never be freed. Over time, it exhausts available memory.

char* Special Rules

char* p = st; cout << p; // prints ENTIRE STRING (not address!) cout << *p; // prints first character only cout << (void*)p; // prints the actual address

3D Dynamic Allocation Pattern

// Allocate int*** m = new int**[H]; for(h) m[h] = new int*[W]; for(h) for(w) m[h][w] = new int[D]; // Deallocate (reverse order!) for(h) for(w) delete[] m[h][w]; for(h) delete[] m[h]; delete[] m;

Structures

Define, Create, Access

struct Student { string name; int age; double gpa; }; Student s1; s1.name = "Morgan"; // dot operator for regular variables

Nested Structs

struct Date { int month, day, year; }; struct Flight { string flNo; Date date; // struct inside struct }; Flight f; f.date.year = 2026; // double dot for nested access

Arrays of Structs

Student roster[30]; roster[0].name = "Alice"; roster[i].gpa // index first, then dot

Dynamic Structs

Student* arr = new Student[n]; // dynamic array arr[i].name = "Bob"; // still use dot with [] delete[] arr; Student* ptr = new Student; // single dynamic struct ptr->name = "Carol"; // arrow operator for pointer-to-struct (*ptr).name = "Carol"; // same thing, ugly version delete ptr;

Structs & Functions

// Print (doesn't modify, pass by value is fine) void print(Student s); // Read (modifies, MUST use reference) void read(Student &s); // Array (always by reference automatically) void readAll(Student arr[], int size);

Self-Referential Structs (Linked Structures)

struct relay { int i; relay* r; // pointer to another relay }; relay a, b, c; a.r = &b; b.r = &c; c.r = &a; // circular: a→b→c→a a.r->r->i // follows chain: a→b→c, gets c.i

Exam trick: Count the arrows to trace which struct you land on. a.r->r->r->r chains through the circle.

Structs with Pass by Ref vs Value

⚠️ Exam Trap (from Practice Test)
void f(Type& x, Type y): x is the original (changes stick), y is a copy (changes lost).
If you call f(obj, obj), x IS obj but y is a COPY of obj at call time.

File I/O

Setup

#include<fstream> ifstream inFile; // for reading inFile.open("data.txt"); if(!inFile) { /* error */ } // always check inFile.close(); // close when done ofstream outFile; // for writing outFile.open("out.txt"); outFile << "Hello" << endl; outFile.close();

Reading Patterns

Known count
for(int i=0; i<n; i++) inFile >> arr[i];
Unknown count
while(inFile >> val) { sum += val; count++; }
Read lines
while(getline(inFile, line)) { // process line }

Reading Formatted Input (slashes, colons)

// Input: 04/20/2026 13:20 char slash, colon; cin >> month >> slash >> day >> slash >> year >> hour >> colon >> minute; // char variables "eat" the separators

Counting Words in a Line

int words = 0; if(line.length() > 0) words = 1; for(int i = 0; i < line.length(); i++) if(line[i] == ' ') words++; // words = spaces + 1

The cin.ignore() Rule

inFile >> number; // reads number, leaves \n inFile.ignore(); // eat the \n getline(inFile, line); // NOW reads the actual next line

Caesar Cipher Pattern (from Practice Test)

// Shift letter by 3, wrapping z→c if(c >= 'a' && c <= 'z') out << (char)('a' + (c - 'a' + 3) % 26); else if(c >= 'A' && c <= 'Z') out << (char)('A' + (c - 'A' + 3) % 26); else out << c; // non-letters unchanged

Use in.get(c) to read character by character (including spaces/newlines).

Tricky Patterns

Patterns from the practice test "Hostile Code" and "You're the Compiler" sections.

Pointer Chasing Through an Array

int a[5] = {5,3,1,4,0}; int* f = a+2; // f → a[2] = 1 f = a + *f; // f = a + (value at f) = a + 1 → a[1]

Pattern: the value at the pointer becomes the next index. Trace step by step.

Ternary Operator with Chars

cout << (ka.ts ? 'B' : 'I'); // if ka.ts is non-zero (any char except '\0') → 'B' // if ka.ts is '\0' (null character = 0) → 'I'

'\0' is the ONLY char that evaluates to false. Every other char is "truthy".

Double Increment in Loop

for(int i=0; i < strlen(st); i++) st[i++] = toupper(st[i]); // i++ in the BODY (post-increment) + i++ in the HEADER // = skip every other character. Only even indices get uppercased.

Circular Struct Pointers

a.r=&b; b.r=&c; c.r=&a; // a→b→c→a→b→c→... a.r->r->r->r->i // a.r=b, ->r=c, ->r=a, ->r=b, ->i = b.i // Count: 4 arrows from a = goes around circle once + 1 = lands on b

Common Exam Mistakes Summary

Don't Do
  • delete arr when used new[]
  • *arr[i] double dereference
  • f(val) when f expects int*
  • Forget cin.ignore()
  • Pass stream by value
  • Function name starting with digit
  • Use reserved word as identifier
  • Overload by return type only
Do Remember
  • delete[] matches new[]
  • Check if(!inFile) after open
  • Close files when done
  • Init pointers to nullptr
  • Pass streams by reference
  • '\0' = false, all other chars = true
  • Integer division truncates
  • Divide by 3.0 not 3 for averages
