Overview
Read This First

Why Does This Problem Exist?

We want All-Pairs Shortest Paths (APSP) — the shortest path between every pair of vertices in a directed weighted graph that may have negative edges.

Floyd-Warshall

Handles negative edges, gives APSP. But runs in O(n³). For large sparse graphs this is terrible.

Dijkstra from every vertex

Runs in O((n+m) log n) per source, so O(mn log n + n² log n) total. Much faster on sparse graphs — but fails on negative edges.

Johnson's Idea

What if we could reweight all edges to be non-negative, run Dijkstra from every vertex, then undo the reweighting to recover the true distances? That's exactly what Johnson's algorithm does.

The trick: find a function h(v) for every vertex, then define new weights w'(x,y) = w(x,y) + h(x) - h(y). This section proves step by step that this works correctly.

🗺️ The Road Map — How the 6 Parts Connect
Each part builds on the previous ones. Understanding the flow makes each proof obvious.
4.1
Define and compute h(x) — the shortest distance from any vertex to x. Use a dummy node + Bellman-Ford. Also detects negative cycles.
4.2
Triangle inequality for h — prove l(p) + h(a) ≥ h(b). This is the key property that makes reweighting work.
4.3
Reweighting formula — prove that l'(p) = l(p) + h(a) - h(b). The change in path length only depends on endpoints, not the path taken.
4.4
All new weights ≥ 0 — follows immediately from 4.2 applied to a single edge. Now Dijkstra is safe to use.
4.5
Shortest paths preserved — follows from 4.3. If p is shortest in G, it's also shortest in G'. Running Dijkstra on G' gives correct relative orderings.
4.6
Full Johnson's algorithm — put it all together: h(x) + reweight + n×Dijkstra + undo reweighting. O(mn + n² log n).
Problem 4.1
4 pts — Computing h(x)

Computing h(x) for All Vertices

The Question

For each vertex x, define h(x) = minz∈V d(z, x) — the shortest distance to x from any vertex (including x itself). Compute h(x) for all x in O(mn) time. If G has a negative cycle, return False.

🤔

Think about what h(x) means. It's not the shortest path from one fixed source — it's the shortest path to x from the best possible starting point. How do you compute "shortest path from anywhere to x" efficiently?

The Dummy Node Trick

The insight: add a fake "super-source" node s* with a zero-weight edge to every vertex. Now the shortest path from s* to any vertex x equals h(x).

Why does d(s*, x) = h(x)?

From s*, you can reach any vertex z in one hop for free (cost 0), then take the shortest path from z to x. So d(s*, x) = min over all z of d(z, x) = h(x). ✓

The zero-weight edges don't create shortcuts between existing vertices, so they don't change any existing distances.

Why Bellman-Ford?

Graph has negative edges. Dijkstra would fail. Bellman-Ford handles negative edges correctly and also detects negative cycles — which we need to report as False. Runtime from one source: O(mn). ✓

The Algorithm

COMPUTE_H(G):
  // Add dummy super-source s* with zero edges to all vertices
  G' ← copy of G
  add node s* to G'
  for each vertex v in V:
    add edge (s* → v) with weight 0

  // Run Bellman-Ford from s* on the augmented graph
  dist ← BELLMAN_FORD(G', s*)

  // If negative cycle detected, return False
  if dist = False: return False

  // h(x) = shortest distance from s* to x
  for each vertex x in V:
    h[x] ← dist[x]

  return h
⚡

Runtime: Adding s* adds n+1 vertices and n edges to G. Bellman-Ford on a graph with n+1 vertices and m+n edges runs in O((n+1)(m+n)) = O(mn + n²) = O(mn) since m ≥ n-1 for a connected graph. ✓

📝 EXAM ANSWER

Correctness: Since s* has a zero-weight edge to every vertex, the shortest path from s* to x is min over all z of (0 + d(z,x)) = min over z of d(z,x) = h(x). The zero-weight edges do not create new paths between existing vertices.

Runtime: Bellman-Ford on n+1 vertices, m+n edges: O((n+1)(m+n)) = O(mn). ✓

Problem 4.2
3 pts — Triangle Inequality for h

Proving l(p) + h(a) ≥ h(b)

The Question

Let p be any path from a to b. Prove that l(p) + h(a) ≥ h(b), assuming G has no negative cycles.

💡

What does this say intuitively? The best you can ever do to get to b is: go to a as cheaply as possible (cost h(a)), then take path p (cost l(p)). So the total cost h(a) + l(p) must be at least h(b) — because h(b) is already the cheapest way to reach b from anywhere.

Formal Proof

1
Recall the definition: h(b) = min over all vertices z of d(z, b). This is the length of the shortest path to b, starting from the best possible vertex.
2
Construct a specific path to b: There exists some vertex z* that achieves h(a) — i.e., d(z*, a) = h(a). Consider the combined path: z* →(shortest)→ a →p→ b. This path has length d(z*, a) + l(p) = h(a) + l(p).
3
Apply the definition of h(b): h(b) is the minimum path length to b over all starting vertices. The path we constructed (from z* to b) has length h(a) + l(p). Therefore:
h(b) ≤ h(a) + l(p)
4
Rearrange: h(b) ≤ h(a) + l(p) directly gives us l(p) + h(a) ≥ h(b). ∎
⚠️

Wait — is step 2 valid? We concatenated a shortest path with an arbitrary path p. The resulting path may not be simple (it could revisit vertices), but path length is still well-defined. Since G has no negative cycles, this concatenated walk still has finite length h(a) + l(p), and h(b) ≤ any walk length to b.

📝 EXAM ANSWER

Concatenating the shortest path from z* to a with the path p gives a walk from z* to b of length h(a) + l(p). Since h(b) is the minimum over all starting vertices, h(b) ≤ h(a) + l(p), which rearranges to l(p) + h(a) ≥ h(b). ∎

Problem 4.3
4 pts — The Reweighting Formula

Proving l'(p) = l(p) + h(x₁) − h(xᵢ)

The Question

In G', edge weights are defined as w'(x,y) = w(x,y) + h(x) - h(y). For a path p = [x₁, x₂, ..., xᵢ], prove that l'(p) = l(p) + h(x₁) - h(xᵢ).

🔭

This is the magical telescoping property. The reweighting formula looks complicated, but when you sum it over an entire path, almost everything cancels. The change in path length only depends on the first and last vertex — not the intermediate vertices or the specific path taken.

Proof by Telescoping Sum

1
Expand l'(p) using the definition of w':
l'(p) = Σj=1..i-1 w'(xj, xj+1) = Σj=1..i-1 [w(xj, xj+1) + h(xj) - h(xj+1)]
2
Split into two sums:
= Σ w(xj, xj+1) + Σ [h(xj) - h(xj+1)]
The first sum is l(p). The second sum telescopes!
3
The telescoping sum: Write out the h terms explicitly:
[h(x₁)-h(x₂)] + [h(x₂)-h(x₃)] + ... + [h(xi-1)-h(xi)]
Every middle term cancels: +h(x₂) and -h(x₂), etc. Only the first and last survive.
= h(x₁) - h(xi)
4
Combine:
l'(p) = l(p) + h(x₁) - h(xi)    ∎

📝 EXAM ANSWER

l'(p) = Σj=1..i-1 w'(xj, xj+1) = Σj=1..i-1 [w(xj, xj+1) + h(xj) - h(xj+1)] = l(p) + Σj=1..i-1 [h(xj) - h(xj+1)]. The sum Σ[h(xj) - h(xj+1)] telescopes to h(x₁) - h(xi). Therefore l'(p) = l(p) + h(x₁) - h(xi). ∎

Problem 4.4
3 pts — All Weights Non-Negative

All Edge Weights in G' are Non-Negative

The Question

Using the result from 4.2, prove that w'(x,y) ≥ 0 for every edge (x,y) in G'.

💡

This is a one-liner once you see it. Apply 4.2 to the "path" consisting of a single edge (x→y). A single edge is a valid path of length w(x,y).

1
Apply 4.2 with p = single edge (x→y), a = x, b = y:
Path length l(p) = w(x,y). By 4.2: l(p) + h(x) ≥ h(y), so w(x,y) + h(x) ≥ h(y).
2
Rearrange: w(x,y) + h(x) - h(y) ≥ 0.
3
Substitute the definition: w'(x,y) = w(x,y) + h(x) - h(y) ≥ 0. ∎

📝 EXAM ANSWER

Problem 4.5
4 pts — Shortest Paths Preserved

Shortest Paths in G Remain Shortest in G'

The Question

Show that if p is a shortest path from a to b in G, then p is also a shortest path from a to b in G'.

🔑

This is why the whole algorithm works. We run Dijkstra on G' (because G' has no negative edges). This tells us shortest paths in G'. If those same paths are also shortest in G, then we've correctly found shortest paths in the original graph.

1
By 4.3: for any path q from a to b: l'(q) = l(q) + h(a) - h(b). This applies to every path from a to b, because the h(a) - h(b) term depends only on the endpoints.
2
In particular for p: l'(p) = l(p) + h(a) - h(b).
3
Compare p to any other path q from a to b:
l'(p) - l'(q) = [l(p) + h(a) - h(b)] - [l(q) + h(a) - h(b)] = l(p) - l(q).
The h terms cancel completely.
4
Since p is shortest in G: l(p) ≤ l(q) for all paths q. Therefore l'(p) ≤ l'(q) for all paths q. So p is also shortest in G'. ∎
⭐

The key insight: reweighting shifts all path lengths from a to b by the same constant h(a) - h(b). Shifting all values by the same constant doesn't change which one is smallest. It's like adding $100 to everyone's salary — the person who earned the least still earns the least.

📝 EXAM ANSWER

By 4.3, for any path q from a to b: l'(q) = l(q) + h(a) - h(b). The term h(a) - h(b) is the same constant for all paths from a to b. Therefore for any two paths p and q from a to b: l'(p) ≤ l'(q) iff l(p) ≤ l(q). Since p is shortest in G (l(p) ≤ l(q) for all q), it is also shortest in G' (l'(p) ≤ l'(q) for all q). ∎

Problem 4.6
5 pts — Full Johnson's Algorithm

Putting It All Together — Full Johnson's Algorithm

The Question

Using the results from all previous parts, describe an algorithm that computes all-pairs shortest paths on G in O(mn + n² log n) time if G has no negative cycles, and returns False otherwise.

The Full Algorithm

JOHNSON_APSP(G):
  // Step 1: Compute h(x) for all vertices using dummy node + Bellman-Ford
  h ← COMPUTE_H(G)           // O(mn) — from part 4.1
  if h = False: return False  // negative cycle detected

  // Step 2: Build reweighted graph G' using w'(x,y) = w(x,y) + h(x) - h(y)
  G' ← copy of G with edge weights:
    for each edge (x,y): w'(x,y) ← w(x,y) + h[x] - h[y]  // O(m)

  // Step 3: Run Dijkstra from every vertex in G'
  // (safe because all weights in G' are ≥ 0, by 4.4)
  for each vertex u in V:
    d'[u] ← DIJKSTRA(G', u)   // O((n+m) log n) per source

  // Step 4: Recover true distances by undoing the reweighting
  // By 4.3: l'(p) = l(p) + h(a) - h(b), so l(p) = l'(p) - h(a) + h(b)
  for each pair (u, v):
    dist[u][v] ← d'[u][v] - h[u] + h[v]   // O(1) per pair, O(n²) total

  return dist

Correctness

Why do we recover d(u,v) = d'(u,v) - h(u) + h(v)?

By 4.3: for any path p from u to v, l'(p) = l(p) + h(u) - h(v). So l(p) = l'(p) - h(u) + h(v).

By 4.5: the shortest path in G' is the same as in G. So d'(u,v) is achieved by the same path that achieves d(u,v). Therefore d(u,v) = d'(u,v) - h(u) + h(v). ✓

📝 EXAM ANSWER

Algorithm: (1) Run COMPUTE_H from 4.1 to get h(x) for all x in O(mn); return False if negative cycle. (2) Build G' with w'(x,y) = w(x,y)+h(x)-h(y) for all edges in O(m). (3) Run Dijkstra from every vertex u in G', giving d'(u,v) for all v, in O(n(n+m)logn) = O(mn logn + n² logn). (4) Recover d(u,v) = d'(u,v) - h(u) + h(v) for all pairs in O(n²).

Correctness: By 4.4, all weights in G' are ≥ 0, so Dijkstra is correct. By 4.5, shortest paths in G' match those in G. By 4.3, l(p) = l'(p) - h(u) + h(v), so d(u,v) = d'(u,v) - h(u) + h(v).

Runtime: Steps 1+2+4 are O(mn) + O(m) + O(n²) = O(mn). Step 3 is O(n(n+m)logn). Total: O(mn + n² logn). ✓

Common Exam Mistakes

❌ Forgetting to subtract h(u) + h(v) when recovering distances
❌ Applying Dijkstra directly on G (negative edges!)
