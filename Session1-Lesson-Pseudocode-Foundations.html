Pseudocode Foundations

Variables, math, booleans, conditionals, and how to read AP pseudocode. This is 30-35% of your exam.

In this lesson: 1. The AP Pseudocode Reference 2. Variables & Assignment 3. Math Expressions & MOD 4. Strings 5. Boolean Expressions 6. Conditionals (IF / ELSE) 7. End of Lesson Review (18 Questions)

1. The AP Pseudocode Reference

On exam day you get a reference sheet with all the pseudocode commands. You don't need to memorize the exact syntax, but you DO need to know what each command does instantly. Here's what matters for today:

Assignment & Display

PseudocodeWhat it does
a ← expressionStores the value of expression into variable a. The arrow points LEFT, value flows from right to left.
DISPLAY(expression)Prints the value to the screen. This is what "what does it display?" questions are asking about.

Math

PseudocodeWhat it does
a + bAddition
a - bSubtraction
a * bMultiplication
a / bDivision (gives a decimal result, NOT integer division)
a MOD bRemainder after division. 10 MOD 3 = 1 because 3*3=9, remainder 1.

Comparison & Logic

PseudocodeWhat it does
a = bEqual to (this is comparison, NOT assignment!)
a ≠ bNot equal to
a > b, a < bGreater than, less than
a ≥ b, a ≤ bGreater than or equal, less than or equal
NOT conditionFlips true to false, false to true
a AND bTrue only if BOTH are true
a OR bTrue if at LEAST one is true
= means two different things. In a ← 5, the arrow is assignment. In IF (a = 5), the = is comparison. AP pseudocode uses ← for assignment and = for comparison. Don't confuse them.

2. Variables & Assignment

A variable is a named container that holds a value. In AP pseudocode, you assign values with the ← arrow.

x ← 5          ← x now holds 5
y ← 10         ← y now holds 10
x ← x + 3      ← x is now 8 (old value 5 + 3)

The key thing: when you reassign a variable, the old value is gone. It gets overwritten.

name ← "Alice"
name ← "Bob"     ← "Alice" is gone forever. name is now "Bob"

The Classic Swap Problem

If you want to swap two variables, you NEED a temporary variable. This shows up on the exam constantly.

a ← 5
b ← 10
← WRONG way (a becomes 10, but then b also becomes 10):
a ← b       ← a is now 10, but 5 is lost!
b ← a       ← b is now 10... both are 10. FAIL.

← RIGHT way:
temp ← a     ← save a's value (5) in temp
a ← b        ← a becomes 10
b ← temp     ← b becomes 5 (from temp)
If you see a question asking "which code correctly swaps two variables," look for the one that uses a third temporary variable.

Quick Check: Variables

What does this display? a ← 3 b ← 7 a ← a + b b ← a - b DISPLAY(a) DISPLAY(b)
a = 3+7 = 10 b = 10-7 = 3 DISPLAY(a) → 10 DISPLAY(b) → 3 (This actually swaps a and b without a temp variable, using math. Sneaky.)
What does this display? x ← 4 x ← x * x x ← x + 1 DISPLAY(x)
x = 4 x = 4 * 4 = 16 x = 16 + 1 = 17 DISPLAY → 17
After this code runs, what are the values of a and b? a ← 10 b ← a a ← 20
a = 20 b = 10 b got a COPY of a's value (10) at the time of assignment. When a changed to 20 later, b was not affected. Variables hold independent copies, not links.

3. Math Expressions & MOD

Standard math works how you'd expect. The one tricky operator is MOD (modulo), which gives the remainder after division.

10 MOD 3 = 1     ← 3 goes into 10 three times (9), remainder 1
15 MOD 5 = 0     ← 5 goes into 15 exactly 3 times, no remainder
7 MOD 2 = 1      ← odd numbers MOD 2 always give 1
8 MOD 2 = 0      ← even numbers MOD 2 always give 0
3 MOD 5 = 3      ← 5 doesn't fit into 3 at all, so remainder is 3
MOD is how you check if a number is even or odd. If n MOD 2 = 0, the number is even. If n MOD 2 = 1, it's odd. This pattern shows up on the exam ALL the time.

Order of Operations

Same as regular math: multiplication and division and MOD happen before addition and subtraction. Use parentheses to force a different order.

3 + 4 * 2 = 11       ← 4*2 first, then +3
(3 + 4) * 2 = 14     ← parentheses first: 7*2

Division in AP Pseudocode

Unlike Java, AP pseudocode division gives decimals. So 7 / 2 = 3.5, not 3. This is important if you've been studying Java or Python integer division.

Quick Check: Math & MOD

What does this display? x ← 17 MOD 5 DISPLAY(x)
17 MOD 5 = 2 (5 goes into 17 three times = 15, remainder = 2) DISPLAY → 2
What does this display? a ← 10 b ← 3 DISPLAY(a / b) DISPLAY(a MOD b)
10 / 3 = 3.333... 10 MOD 3 = 1 DISPLAY → 3.333... then 1
Is the number 42 even or odd? How would you check using MOD?
42 MOD 2 = 0 Since the remainder is 0, 42 is EVEN. The check: IF (n MOD 2 = 0) → even, ELSE → odd

4. Strings

Strings are text wrapped in quotes. You can glue them together with concatenation (using +).

first ← "Hello"
second ← "World"
result ← first + " " + second
DISPLAY(result)    ← "Hello World"
Numbers inside quotes are strings, not numbers. "3" + "5" gives "35" (concatenation), NOT 8 (addition). If it's in quotes, it's text.
a ← "3" + "5"     ← a = "35" (string concatenation)
b ← 3 + 5         ← b = 8 (math addition)

Quick Check: Strings

What does this display? word ← "AP" word ← word + " " + "CSP" DISPLAY(word)
"AP" + " " + "CSP" = "AP CSP" DISPLAY → AP CSP
What does this display? x ← "10" + "20" DISPLAY(x)
"10" + "20" = "1020" (string concatenation, NOT math) DISPLAY → 1020 Both values are in quotes, so they're strings. + concatenates.

5. Boolean Expressions

A boolean is either true or false. That's it. Boolean expressions are the foundation of every IF statement and every loop condition.

Comparison Operators

5 = 5     → true
5 = 3     → false
5 ≠ 3     → true      ← "not equal"
5 > 3     → true
5 < 3     → false
5 ≥ 5     → true      ← "greater than OR equal"
5 ≤ 4     → false

AND, OR, NOT

These combine or flip boolean values.

AND: Both sides must be true
  true AND true   → true
  true AND false  → false
  false AND false → false

OR: At least one side must be true
  true OR false   → true
  false OR false  → false
  true OR true    → true

NOT: Flips the value
  NOT true   → false
  NOT false  → true

Combining Them

You can chain these together. NOT applies first, then AND, then OR (unless parentheses override).

← Example: x = 7
(x > 5) AND (x < 10)       → true AND true  → true
(x > 5) AND (x < 6)        → true AND false → false
(x = 7) OR (x = 8)         → true OR false  → true
NOT (x > 10)                → NOT false      → true
NOT (x > 5 AND x < 10)     → NOT true       → false
Exam trap: NOT with AND/OR. NOT (A AND B) is the same as (NOT A) OR (NOT B). And NOT (A OR B) is the same as (NOT A) AND (NOT B). This is De Morgan's Law and it shows up in tricky MC questions.

Quick Check: Booleans

x = 12. Evaluate each: 1. (x > 10) AND (x < 20) 2. (x = 10) OR (x = 12) 3. NOT (x > 15) 4. (x MOD 2 = 0) AND (x > 10)
1. true AND true → true 2. false OR true → true 3. NOT false → true 4. (0 = 0) AND (12 > 10) → true AND true → true All four are true!
Evaluate: NOT (true AND false) OR true
Step by step: 1. true AND false → false 2. NOT false → true 3. true OR true → true Answer: true
Are these two expressions equivalent? A: NOT (x > 5 AND y < 3) B: (x ≤ 5) OR (y ≥ 3)
YES, they are equivalent. (De Morgan's Law) NOT (A AND B) = (NOT A) OR (NOT B) NOT (x > 5) = x ≤ 5 NOT (y < 3) = y ≥ 3 So NOT (x > 5 AND y < 3) = (x ≤ 5) OR (y ≥ 3) ✓

6. Conditionals (IF / ELSE)

Conditionals let a program make decisions. The code inside the IF block only runs if the condition is true.

Basic IF

IF (condition)
{
    ← this code runs ONLY if condition is true
}

IF / ELSE

IF (condition)
{
    ← runs if condition is true
}
ELSE
{
    ← runs if condition is false
}

One or the other always runs. Never both. Never neither.

Nested IFs (IF inside IF)

IF (x > 0)
{
    IF (x > 100)
    {
        DISPLAY("big positive")
    }
    ELSE
    {
        DISPLAY("small positive")
    }
}
ELSE
{
    DISPLAY("not positive")
}

When IFs are nested, trace from the outside in. Check the outer condition first. If it's true, THEN check the inner one.

Chained Conditions (IF / ELSE IF / ELSE pattern)

AP pseudocode doesn't have a formal "else if" keyword, but you can nest them to create the same effect:

IF (score ≥ 90)
{
    grade ← "A"
}
ELSE
{
    IF (score ≥ 80)
    {
        grade ← "B"
    }
    ELSE
    {
        IF (score ≥ 70)
        {
            grade ← "C"
        }
        ELSE
        {
            grade ← "F"
        }
    }
}
These chains stop at the first true condition. If score is 95, it matches ≥ 90 and gets "A". It never checks the rest, even though 95 is also ≥ 80 and ≥ 70.

Quick Check: Conditionals

What does this display? (x = 7) IF (x > 10) { DISPLAY("A") } ELSE { IF (x > 5) { DISPLAY("B") } ELSE { DISPLAY("C") } }
x = 7 Is 7 > 10? NO → go to ELSE Is 7 > 5? YES → DISPLAY "B" Answer: B
What does this display? age ← 16 IF (age ≥ 18) { msg ← "adult" } ELSE { msg ← "minor" } IF (age ≥ 16) { msg ← msg + " who can drive" } DISPLAY(msg)
age = 16 First IF: 16 ≥ 18? NO → msg = "minor" Second IF: 16 ≥ 16? YES → msg = "minor" + " who can drive" = "minor who can drive" DISPLAY → "minor who can drive" KEY INSIGHT: The second IF is NOT an ELSE IF. It's a completely separate IF that runs independently. Both IF blocks get checked.
What does this display? n ← 15 IF (n MOD 3 = 0 AND n MOD 5 = 0) { DISPLAY("fizzbuzz") } ELSE { IF (n MOD 3 = 0) { DISPLAY("fizz") } ELSE { IF (n MOD 5 = 0) { DISPLAY("buzz") } ELSE { DISPLAY(n) } } }
n = 15 15 MOD 3 = 0? YES. 15 MOD 5 = 0? YES. true AND true = true → DISPLAY "fizzbuzz" (The chain stops at the first match. Never reaches the inner checks.)
