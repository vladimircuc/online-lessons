Learning Target P.2
P.2: I can use pattern matching and algebraic datatypes to write data-directed functions/programs.

Scenario

A parking garage charges different rates based on vehicle type and duration:

  • Cars: $3 per hour.
  • Motorcycles: $1 per hour.
  • Trucks of three classes: Light, Medium, Heavy.
    • Light: $4 per hour.
    • Medium: $6 per hour.
    • Heavy: $10 per hour.
data TruckClass = Light | Medium | Heavy deriving (Show)

data Vehicle = Car
             | Motorcycle
             | Truck TruckClass
             deriving (Show)

v1, v2, v3, v4 :: Vehicle
v1 = Car
v2 = Motorcycle
v3 = Truck Heavy
v4 = Truck Light

garage :: [(Vehicle, Integer)]   -- (vehicle, hours)
garage = [(Car, 2), (Truck Medium, 3), (Motorcycle, 5), (Truck Light, 1)]

Note: These types do not derive Eq, so you cannot use == or /=, nor the show function, in place of pattern matching.

Q2.1 Write a Haskell function

Write a function rate :: Vehicle -> Integer that returns the hourly rate (in dollars) for a given vehicle.

Examples:

  • rate Car evaluates to 3
  • rate Motorcycle evaluates to 1
  • rate (Truck Heavy) evaluates to 10
  • rate (Truck Light) evaluates to 4
Show Solution
Solution
rate :: Vehicle -> Integer
rate Car              = 3
rate Motorcycle       = 1
rate (Truck Light)    = 4
rate (Truck Medium)   = 6
rate (Truck Heavy)    = 10

Five equations, one per distinct case. For the truck cases, we pattern-match on the inner constructor.

Alternative using a helper function:

rate :: Vehicle -> Integer
rate Car          = 3
rate Motorcycle   = 1
rate (Truck tc)   = truckRate tc

truckRate :: TruckClass -> Integer
truckRate Light  = 4
truckRate Medium = 6
truckRate Heavy  = 10
Q2.2 Write a Haskell function

Write a function totalRevenue :: [(Vehicle, Integer)] -> Integer that sums up the total revenue from a list of (vehicle, hours) pairs. The revenue for each pair is rate(vehicle) * hours.

Examples:

  • totalRevenue garage evaluates to (3*2) + (6*3) + (1*5) + (4*1) = 6 + 18 + 5 + 4 = 33
  • totalRevenue [] evaluates to 0
Show Solution
Solution

Option A, recursion with tuple pattern:

totalRevenue :: [(Vehicle, Integer)] -> Integer
totalRevenue [] = 0
totalRevenue ((v, h) : rest) = rate v * h + totalRevenue rest

Option B, higher-order functions:

totalRevenue :: [(Vehicle, Integer)] -> Integer
totalRevenue ps = sum [rate v * h | (v, h) <- ps]

Both are correct.

Learning Target P.3
P.3: I can convert between the following list-processing approaches: recursion with patterns, list comprehensions, and higher-order functions (map and filter).

Consider the following two Haskell functions:

canyon :: [(Char, Integer)] -> [Integer]
canyon ps = [ n * 2 | (c, n) <- ps, isUpper c ]

dune :: Integer -> [Integer] -> [Integer]
dune d [] = []
dune d (n:ns)
  | n + d <= 50 = (n + d) : dune d ns
  | otherwise   = dune d ns
Q3.1 Fill in the blanks to write an equivalent function to dune:
dune1 :: Integer -> [Integer] -> [Integer]
dune1 d ns = map ________ (filter ________ ns)
Show Solution
Solution

Identify T and P:

  • Keep branch: (n + d) : dune d ns. So T(n) = n + d.
  • Guard: n + d <= 50. So P(n) = n + d <= 50.

Both need d, use lambdas:

dune1 :: Integer -> [Integer] -> [Integer]
dune1 d ns = map (\n -> n + d) (filter (\n -> n + d <= 50) ns)
Q3.2 Fill in the blanks to write an equivalent function to dune:
dune2 :: Integer -> [Integer] -> [Integer]
dune2 d ns = [ ________ | ________________ ]
Show Solution
Solution

Same T and P. Plug into the list-comp template:

dune2 :: Integer -> [Integer] -> [Integer]
dune2 d ns = [ n + d | n <- ns, n + d <= 50 ]
Q3.3 Write a recursive function

Write a recursive function canyon1 :: [(Char, Integer)] -> [Integer] that uses patterns and is equivalent to canyon.

Show Solution
Solution

From the list comp:

  • T((c, n)) = n * 2
  • P((c, n)) = isUpper c
canyon1 :: [(Char, Integer)] -> [Integer]
canyon1 [] = []
canyon1 ((c, n) : rest)
  | isUpper c = (n * 2) : canyon1 rest
  | otherwise = canyon1 rest

Standard tuple-destructuring recursion pattern. isUpper comes from Data.Char.

Learning Target T.1
T.1: I can trace and explain the execution behavior of a recursive function defined over a tree-like data structure.

Consider the following Haskell code:

data BTree a = Empty
             | BNode a (BTree a) (BTree a)
             deriving (Show)

aspen :: BTree Integer
aspen = BNode 6 (BNode 2 (BNode 4 Empty Empty) Empty)
                (BNode 9 Empty (BNode 3 Empty Empty))

ripple :: BTree Integer -> [Integer]
ripple Empty = []
ripple (BNode v left right)
  | odd v     = [v] ++ ripple left
  | otherwise = ripple right ++ [v]
          6
        /   \
       2    9
      /      \
     4       3
Q4.1 Trace and explain

Trace and explain what happens when ripple aspen is evaluated.

Your answer should provide enough information to clearly demonstrate how the evaluation progresses: for each recursive call, explicitly indicate the result of the pattern matching (i.e., the bindings to variables), how the result is evaluated, and the final result of that function call.

Show Solution
Full trace
Call 1: ripple (BNode 6 L R)
  where L = BNode 2 (BNode 4 Empty Empty) Empty
        R = BNode 9 Empty (BNode 3 Empty Empty)
  Match (BNode v left right):
    v = 6, left = L, right = R
  Guard: odd 6 = False → otherwise
  Expr: ripple R ++ [6]
  Need ripple R first:

    Call 2: ripple (BNode 9 Empty (BNode 3 Empty Empty))
      Match: v = 9, left = Empty, right = BNode 3 Empty Empty
      Guard: odd 9 = True → first branch
      Expr: [9] ++ ripple Empty

        Call 3: ripple Empty → [] (base case)

      Back to Call 2: [9] ++ [] = [9]
      Returns [9].

  Back to Call 1: [9] ++ [6] = [9, 6]
  Returns [9, 6].
Final: ripple aspen = [9, 6]

Key observation: the left subtree (nodes 2 and 4) is never visited because at the root 6 (even), the guard sent us to ripple right. At node 9 (odd), the guard sent us to ripple left, which was Empty. Node 3 is never visited.

Learning Target T.2
T.2: I can write correct and concise code that operates over tree-like data structures.

Setup

Recall the Haskell representation of trees:

data BTree a = Empty
             | BNode a (BTree a) (BTree a)
             deriving (Show)

oak :: BTree Char
oak = BNode 'x' (BNode '5' Empty Empty)
                (BNode 'A' (BNode '7' Empty Empty) (BNode 'k' Empty Empty))

meadow :: BTree Integer
meadow = BNode 20 (BNode 10 (BNode 5 Empty Empty) (BNode 15 Empty Empty))
                  (BNode 30 (BNode 25 Empty Empty) (BNode 40 Empty Empty))
Q5.1 Write a recursive Haskell function

Write a recursive Haskell function

digitLeaf :: BTree Char -> Bool

such that digitLeaf tr returns True if there is at least one leaf node in tr whose label is a digit character ('0' through '9'), and False otherwise.

A leaf is a node with both children Empty. You may use isDigit from Data.Char.

Examples:

  • digitLeaf oak evaluates to True (leaves '5', '7', 'k'; '5' and '7' are digits)
  • digitLeaf (BNode 'a' (BNode 'b' Empty Empty) Empty) evaluates to False (leaf 'b' isn't a digit)
  • digitLeaf Empty evaluates to False
  • digitLeaf (BNode '9' Empty Empty) evaluates to True
Show Solution
Solution
digitLeaf :: BTree Char -> Bool
digitLeaf Empty = False
digitLeaf (BNode v Empty Empty) = isDigit v
digitLeaf (BNode _ left right)  = digitLeaf left || digitLeaf right

Three equations, leaf-aware:

  1. Empty → False.
  2. Leaf case: check with isDigit.
  3. Internal case: OR the subtrees.

Same structural shape as smallLeaf from the real exam.

Q5.2 Write a recursive Haskell function

Write a recursive Haskell function

thin :: Integer -> BTree Integer -> BTree Integer

such that thin num tr returns the tree obtained from tr by removing any node (plus its descendants) whose label is a multiple of num.

Recall that removing a node is equivalent to replacing it with an empty tree. Assume num is always positive. Use mod for the divisibility check.

Examples:

  • thin 10 meadow evaluates to Empty (root 20 is a multiple of 10)
  • thin 7 meadow returns meadow unchanged (no node is a multiple of 7).
  • thin 3 meadow removes 15 and 30 (multiples of 3): 
    BNode 20 (BNode 10 (BNode 5 Empty Empty) Empty) Empty
    (15 is removed, 30 is removed with its children 25 and 40)
Show Solution
Solution
thin :: Integer -> BTree Integer -> BTree Integer
thin _   Empty = Empty
thin num (BNode v left right)
  | v `mod` num == 0 = Empty
  | otherwise        = BNode v (thin num left) (thin num right)

Explanation using the 3 questions:

  1. Empty answer: Empty.
  2. Data needed: v, left, right, plus num.
  3. Combine: if v is a multiple of num, return Empty (kill subtree). Otherwise rebuild.

Verification for thin 3 meadow:

  • 20: 20 mod 3 = 2, not 0. Rebuild.
  • 10 (left): 10 mod 3 = 1, not 0. Rebuild.
    • 5: 5 mod 3 = 2, not 0. Leaf, stays.
    • 15: 15 mod 3 = 0. Empty.
    Returns BNode 10 (BNode 5 Empty Empty) Empty.
  • 30 (right): 30 mod 3 = 0. Empty. (Children 25 and 40 never visited.)
  • Result: BNode 20 (BNode 10 (BNode 5 Empty Empty) Empty) Empty. ✓
