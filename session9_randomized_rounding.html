Randomized Rounding for Set Cover

What happens when you use the LP's fractional solution as a probability instead of rounding deterministically? A powerful technique that gives an O(log n) approximation.

← Session 4 covered deterministic rounding: solve LP relaxation, round x* ≥ 1/f up to 1. Sessions 7-8 covered probability, expectation, linearity, Karger's analysis, and boosting. Today we combine both threads.

Where This Fits

In Session 4, you learned the deterministic rounding pipeline: write an ILP, relax to LP, solve, then round fractional values to integers using a fixed rule. Today we do the same first three steps, but the rounding step is random: we use the LP solution as probabilities.

Set Cover & ILP Refresher

The problem, its ILP formulation, and the LP relaxation, all quick since you've seen this in Session 4.

The Set Cover Problem

Problem Definition

Input:

  • A universe U = {1, 2, …, n} of n elements
  • Sets S₁, S₂, …, Sₘ where each Sᵢ ⊆ U
  • Weights w₁, w₂, …, wₘ (cost of including each set)

Output: A collection I ⊆ {1, 2, …, m} such that:

  • Every element is covered: ∪ᵢ∈I Sᵢ = U
  • Total cost Σᵢ∈I wᵢ is minimized

ILP Formulation

Introduce binary variable xᵢ ∈ {0, 1} for each set: xᵢ = 1 means "include Sᵢ."

Minimize   Σᵢ wᵢ xᵢ
Subject to:
     Σ{i : j ∈ Sᵢ} xᵢ ≥ 1     for every element j ∈ {1, …, n}
     xᵢ ≥ 0     for all i
     xᵢ ∈ {0, 1}     (the integer constraint)

The coverage constraint says: for each element j, the sum of xᵢ over all sets containing j must be at least 1 (at least one set covering j is selected).

LP Relaxation

Drop the integer constraint. Replace xᵢ ∈ {0, 1} with 0 ≤ xᵢ ≤ 1. Solve in polynomial time. Get an optimal fractional solution x* where each xᵢ* ∈ [0, 1].

Key Property

The LP optimal value OPTLP ≤ OPTILP (relaxing constraints can only help, never hurt). So OPTLP is a lower bound on the true optimal cost.

Quick Check

If the LP relaxation gives x₁* = 0.6, x₂* = 0.8, x₃* = 0.3, what does the coverage constraint for element j guarantee?

Part 3 / 8

The Randomized Rounding Idea

Instead of a fixed rule, use the LP solution as a coin-flip probability.

Deterministic vs Randomized Rounding

Session 4: Deterministic

If xᵢ* ≥ 1/f, set x̂ᵢ = 1, else x̂ᵢ = 0.

Fixed threshold. Always the same output for the same LP solution. Gave f-approximation.

Today: Randomized

Set x̂ᵢ = 1 with probability xᵢ*, and x̂ᵢ = 0 with probability 1 − xᵢ*.

Random coin flip using LP value. Different output each time. Will give O(log n) approximation.

💡 The Intuition

If the LP says xᵢ* = 0.7, that means the LP "wants" Sᵢ to be 70% included. In deterministic rounding, we'd include it (since 0.7 > threshold). In randomized rounding, we flip a coin that lands heads with probability 0.7, and include Sᵢ if heads.

If xᵢ* = 0.1, we only include Sᵢ 10% of the time. The LP's "confidence" directly becomes the inclusion probability.

Concrete Example

Worked Example

Suppose the LP gives: x₁* = 0.8, x₂* = 0.5, x₃* = 0.3

Randomized rounding does:

  • Flip a coin with P(heads) = 0.8 for S₁. Heads → include S₁.
  • Flip a coin with P(heads) = 0.5 for S₂. Tails → don't include S₂.
  • Flip a coin with P(heads) = 0.3 for S₃. Tails → don't include S₃.

Result: only S₁ is selected. This might not cover everything!

Run it again with fresh coin flips, you might get {S₁, S₂}, or {S₂, S₃}, or even all three. The output is random.

⚠️ The Problem

Randomized rounding might not cover every element. Even if the LP constraint guarantees Σxᵢ* ≥ 1 for each element, flipping coins based on these probabilities doesn't guarantee coverage. Some elements might get unlucky and none of their covering sets get selected.

So two questions arise: (1) What's the probability an element IS covered? (2) How do we fix the coverage gap?

Part 4 / 8

Coverage Analysis

What's the probability a single element gets covered, and the key inequality that makes it work.

P(element j is covered)

Fix an element j. It's covered if at least one set containing j is selected. It's easier to compute the complement: not covered means every set containing j was NOT selected.

P(j is NOT covered) = ∏{i : j ∈ Sᵢ} P(x̂ᵢ = 0) = ∏{i : j ∈ Sᵢ} (1 − xᵢ*)

We can multiply because the coin flips for different sets are independent (each set's rounding is a separate coin flip).

The Key Inequality

Now we need a bound. The crucial tool is:

⭐ The Inequality: 1 − x ≤ e−x

For all real x. This comes from the Taylor expansion of e−x = 1 − x + x²/2 − x³/6 + …, where the extra positive terms make e−x at least as large as 1 − x.

Applying the Inequality

Substitute into our product:

P(j NOT covered) = ∏{i : j ∈ Sᵢ} (1 − xᵢ*) ≤ ∏{i : j ∈ Sᵢ} e−xᵢ* = e−Σ xᵢ*

The product of exponentials becomes the exponential of the sum. Now, the LP coverage constraint guarantees Σ{i : j ∈ Sᵢ} xᵢ* ≥ 1. So:

P(j NOT covered) ≤ e−1 ≈ 0.368

Equivalently:

P(j IS covered) ≥ 1 − e−1 ≈ 0.632
⚠️ The Issue

63.2% chance of covering each element isn't bad, but it's not a guarantee. With n elements, some will likely be uncovered. We need to fix this. The fix? Repeat!

Part 5 / 8

Repeated Rounding & Union Bound

Run the rounding multiple times, take the union of selected sets, and use a union bound to guarantee coverage.

The Idea

One round of randomized rounding gives each element a ~63.2% chance of being covered. What if we repeat the rounding N times (independently) and take the union of all sets selected across all rounds?

An element j is uncovered only if it's uncovered in ALL N rounds. Since rounds are independent:

P(j uncovered after N rounds) ≤ (e−1)N = e−N

Choosing N = c · ln(n)

We want ALL n elements to be covered. By the union bound (also called Boole's inequality: P(A ∪ B) ≤ P(A) + P(B)):

P(∃ any uncovered element) ≤ ∑j=1n P(j uncovered) ≤ n · e−N

Set N = c · ln(n) (ln is the natural logarithm, "natural log"):

n · e−c·ln(n) = n · n−c = n1−c

For c ≥ 2, this gives n−1 = 1/n, which goes to 0 as n grows. So with high probability, everything is covered.

✅ Result

After N = c · ln(n) rounds of randomized rounding (c ≥ 2), the union of all selected sets covers every element with probability at least 1 − 1/n.

Part 6 / 8

The O(log n) Approximation Ratio

How good is the solution we find? Bounding the expected cost.

Cost of One Round

After a single round of randomized rounding, what's the expected cost of the selected sets?

Let X = Σ wᵢ x̂ᵢ be the total cost. By linearity of expectation:

E[X] = ∑i wᵢ · E[x̂ᵢ] = ∑i wᵢ · P(x̂ᵢ = 1) = ∑i wᵢ · xᵢ* = OPTLP
💡 Beautiful Fact

The expected cost of one round of randomized rounding is exactly the LP optimum. Since we use xᵢ* as the probability, E[x̂ᵢ] = xᵢ*, so the expected cost matches OPTLP perfectly.

Cost of N = c · ln(n) Rounds

We run N rounds and take the union. The expected total cost is:

E[total cost] = N · E[one round] = c · ln(n) · OPTLP

Since OPTLP ≤ OPTILP (the LP lower bound):

E[total cost] ≤ c · ln(n) · OPT
✅ Theorem: Randomized Rounding for Set Cover

The randomized rounding algorithm (with c · ln(n) repetitions) produces a feasible set cover with high probability, and its expected cost is at most O(log n) · OPT.

This is an O(log n)-approximation algorithm. Compare with deterministic rounding's f-approximation from Session 4.
