Question 1 Integer Division
What is the value of result after the following code executes?
int a = 17; int b = 5; double result = a / b + a % b;
A 5.4
B 5.0
C 5.2
D 3.0
Answer: B (5.0)

17 / 5 = 3 (integer division truncates)
17 % 5 = 2 (remainder)
3 + 2 = 5, stored as double = 5.0

The division happens BEFORE assignment to double. Both operands are int, so the result is int.
Question 2 String Methods
What is printed by the following code?
String s = "COMPUTER"; System.out.println(s.substring(3, 6));
A "MPU"
B "MPUT"
C "PUT"
D "PUTE"
Answer: C ("PUT")

Indices: C=0, O=1, M=2, P=3, U=4, T=5, E=6, R=7
substring(3, 6) returns chars at index 3, 4, 5 (end index is exclusive).
That gives us "PUT".
Question 3 Array Traversal
What is the value of count after the code executes?
int[] arr = {4, 7, 2, 9, 1, 5, 8}; int count = 0; for (int i = 1; i < arr.length - 1; i++) { if (arr[i] > arr[i - 1] && arr[i] > arr[i + 1]) { count++; } }
A 1
B 3
C 4
D 2
Answer: D (2)

We check indices 1 through 5 (skipping first and last):
i=1: arr[1]=7 > arr[0]=4 AND > arr[2]=2? Yes, count=1
i=2: arr[2]=2 > arr[1]=7? No
i=3: arr[3]=9 > arr[2]=2 AND > arr[4]=1? Yes, count=2
i=4: arr[4]=1 > arr[3]=9? No
i=5: arr[5]=5 > arr[4]=1 AND > arr[6]=8? No (5 < 8)

This algorithm counts "local peaks", elements greater than both neighbors.
Question 4 ArrayList Remove Trap
What does nums contain after this code runs?
ArrayList<Integer> nums = new ArrayList<>(); nums.add(3); nums.add(3); nums.add(5); nums.add(3); nums.add(7); for (int i = 0; i < nums.size(); i++) { if (nums.get(i) == 3) { nums.remove(i); } }
A [3, 5, 7]
B [5, 7]
C [5, 3, 7]
D [3, 5, 3, 7]
Answer: A ([3, 5, 7])

This is the classic index-drift trap:

Start: [3, 3, 5, 3, 7]
i=0: nums.get(0)=3, remove(0) → list becomes [3, 5, 3, 7]. The second 3 shifts down to index 0!
i=1: nums.get(1)=5, skip (i jumped past the 3 that shifted down)
i=2: nums.get(2)=3, remove(2) → list becomes [3, 5, 7]
i=3: 3 is not less than nums.size()=3, loop ends.

Final: [3, 5, 7]. One 3 survives because it shifted into an index we already passed.

Key lesson: When removing from an ArrayList while traversing forward, elements shift left and you skip the next one. Fix: traverse backward, or only increment i when you don't remove.
Question 5 Boolean Logic
Given int x = 4; and int y = 7;, what does the following evaluate to?
!(x > 5) && (y > 3 || x == 4)
A false
B true
C Compile error
D Cannot be determined
Answer: B (true)

Step by step:
x > 5 → 4 > 5 → false
!(false) → true
y > 3 → 7 > 3 → true
x == 4 → true
true || true → true
true && true → true

Key concept: ! (NOT) applies before &&, and || is evaluated inside parentheses first.
Question 6 for-each Loop
What is printed?
int[] vals = {10, 20, 30, 40, 50}; int sum = 0; for (int v : vals) { if (v % 20 == 0) { sum += v; } } System.out.println(sum);
A 30
B 50
C 60
D 90
Answer: C (60)

We add values divisible by 20:
10 % 20 = 10 → no
20 % 20 = 0 → add 20, sum=20
30 % 20 = 10 → no
40 % 20 = 0 → add 40, sum=60
50 % 20 = 10 → no
Final: 60
Question 7 String .equals() vs ==
What is printed?
String a = new String("hello"); String b = new String("hello"); if (a == b) System.out.print("same "); if (a.equals(b)) System.out.print("equal");
A same equal
B same
C Nothing prints
D equal
Answer: D ("equal")

== compares references (memory addresses). Since we used new String(), a and b are different objects, so a == b is false.

.equals() compares the actual content of the strings. Both contain "hello", so this is true.

This is one of the top 5 AP CSA exam traps. Always use .equals() for String comparison.
Question 8 2D Array Traversal
What value is stored in total?
int[][] grid = { {1, 2, 3}, {4, 5, 6}, {7, 8, 9} }; int total = 0; for (int r = 0; r < grid.length; r++) { total += grid[r][r]; }
A 45
B 15
C 12
D 6
Answer: B (15)

grid[r][r] accesses the diagonal:
r=0: grid[0][0] = 1
r=1: grid[1][1] = 5
r=2: grid[2][2] = 9
total = 1 + 5 + 9 = 15

Key pattern: when row and column index are the same, you're traversing the main diagonal.

Given the class below, what is printed?
public class Player { private String name; private int score; public Player(String n, int s) { name = n; score = s; } public void addPoints(int pts) { score += pts; } public int getScore() { return score; } } // In main: Player p = new Player("Soham", 10); p.addPoints(5); p.addPoints(3); System.out.println(p.getScore());
A 18
B 8
C 10
D 15
Answer: A (18)

Constructor sets score = 10

Question 10 Math.random()
Which expression generates a random integer from 5 to 15 inclusive?
A(int)(Math.random() * 15) + 5
B(int)(Math.random() * 10) + 5
C(int)(Math.random() * 11) + 5
D(int)(Math.random() * 15 + 5)
Answer: C

Formula: (int)(Math.random() * range) + min
range = max - min + 1 = 15 - 5 + 1 = 11
min = 5

Math.random() returns [0.0, 1.0)
× 11 → [0.0, 11.0)
(int) cast → 0 to 10
+ 5 → 5 to 15 ✓

Option B only gives 5-14 (off by one). Option A gives 5-19. Option D gives 5-19.

Mini FRQ 1 · Type 1 Methods & Control

Write a method countMatch that takes an array of integers and a target value, and returns the number of elements in the array that are equal to the target.

/** Returns the count of elements in arr equal to target. * Precondition: arr is not null. */ public static int countMatch(int[] arr, int target)

Part (b): Using countMatch, write a method hasMajority that returns true if a given value appears in more than half the elements of the array.

/** Returns true if target appears more than arr.length / 2 times. */ public static boolean hasMajority(int[] arr, int target)
Sample Solution
// Part (a) public static int countMatch(int[] arr, int target) { int count = 0; for (int val : arr) { if (val == target) { count++; } } return count; } // Part (b) public static boolean hasMajority(int[] arr, int target) { return countMatch(arr, target) > arr.length / 2; }
Scoring points to watch:
Part (a): Correct loop structure (+1), correct comparison (+1), correct accumulator and return (+1)
Part (b): Calls countMatch correctly (+1), correct comparison with arr.length/2 (+1), returns boolean (+1)

Common mistakes: using == with objects (not relevant here since ints), off-by-one on "more than half" (> not >=), forgetting to return.
Mini FRQ 2 · Type 3 ArrayList

Write a method removeShortWords that takes an ArrayList<String> and an integer minLen, and removes all strings with length less than minLen.

/** Removes all strings from words that have length less than minLen. * Precondition: words is not null. */ public static void removeShortWords(ArrayList<String> words, int minLen)

Example:

// Before: ["hi", "hello", "ok", "wonderful", "a"] // minLen = 3 // After: ["hello", "wonderful"]
Sample Solution
// Approach 1: Traverse backwards (avoids index drift) public static void removeShortWords(ArrayList<String> words, int minLen) { for (int i = words.size() - 1; i >= 0; i--) { if (words.get(i).length() < minLen) { words.remove(i); } } } // Approach 2: Forward with index adjustment public static void removeShortWords(ArrayList<String> words, int minLen) { int i = 0; while (i < words.size()) { if (words.get(i).length() < minLen) { words.remove(i); // do NOT increment i } else { i++; } } }
This is THE most important FRQ pattern to master.

If Soham traverses forward with a regular for loop and increments i every time, he will SKIP elements after a removal. This is the #1 ArrayList trap on the AP exam.

Two correct strategies: (1) traverse backwards, or (2) only increment i when you DON'T remove. Watch which approach he tries, if he doesn't know either pattern, this needs serious drilling.
Bonus FRQ · Type 4 2D Array

Write a method rowWithLargestSum that takes a 2D array of ints and returns the index of the row with the largest sum.

/** Returns the index of the row with the largest sum. * Precondition: grid has at least one row. */ public static int rowWithLargestSum(int[][] grid)

Example:

// grid = { {1, 2, 3}, row 0 sum = 6 // {10, 20, 30}, row 1 sum = 60 // {4, 5, 6} } row 2 sum = 15 // Returns: 1
Sample Solution
public static int rowWithLargestSum(int[][] grid) { int maxSum = Integer.MIN_VALUE; int maxRow = 0; for (int r = 0; r < grid.length; r++) { int rowSum = 0; for (int c = 0; c < grid[r].length; c++) { rowSum += grid[r][c]; } if (rowSum > maxSum) { maxSum = rowSum; maxRow = r; } } return maxRow; }
Watch for:
Does he use grid.length for rows and grid[r].length for columns? (common mistake: mixing these up)
Does he initialize maxSum to Integer.MIN_VALUE or 0? (0 fails if all values are negative)
Does he track both the max sum AND the max row index?
Nested loop structure: outer = rows, inner = columns.
