Part 1 / 8
Part 1 / 8 — What is Linear Programming?

The Big Idea

A Linear Program (LP) is an optimization problem where you want to maximize (or minimize) a linear objective function subject to a set of linear inequality constraints.

Both of those words matter. Linear means no x², no x·y, no √x — everything is just variables multiplied by constants and added together. Inequality means constraints like ≤ or ≥, not equalities.

From your notes: "An LP instance is an optimization problem where we aim to optimize a linear function constrained by linear inequalities."

Before the formal notation, here's the intuition. Say you're a farmer with 100 acres of land. You can grow corn (profit $3/acre) or wheat (profit $5/acre). But corn needs 2 units of water per acre and wheat needs 4 — and you only have 280 units of water. How much of each should you plant?

That's a linear program. The objective (profit) is linear. The constraints (land, water) are linear inequalities. And the solution is some combination of corn and wheat that maximizes profit.

Why LP is Important for This Course

LP sits right at the center of everything coming up:

LP fundamentals, geometry, standard form, max flow as LP
LP relaxation + rounding → approximation algorithms for Set Cover
FPTAS for Knapsack uses LP-style scaling ideas
✅ LP is solvable in polynomial time (Ellipsoid method 1979, Interior Point methods). This is crucial — it means LP is a tractable tool we can use as a subroutine inside approximation algorithms.
Part 2 / 8 — Standard Form LP

The Standard Form

Your notes define the standard form LP with three inputs: a matrix A, and vectors b and c. Here's what each piece means:

// Input:
A ∈ Mm×n(ℝ)   // m×n matrix of constraint coefficients
b ∈ ℝm          // m right-hand side values (one per constraint)
c ∈ ℝn          // n objective coefficients (one per variable)

// Problem:
maximize   cᵀx
subject to Ax ≤ b   // coordinate-wise (each row is one constraint)
            x ≥ 0    // all variables non-negative

The phrase "coordinate-wise" from your notes just means each row of Ax ≤ b is a separate constraint — row 1 gives constraint 1, row 2 gives constraint 2, and so on.

Breaking It Down — What Each Part Means

An n-dimensional vector. These are what we're solving for. In the farm example: x₁ = acres of corn, x₂ = acres of wheat.
c is a vector of profits/costs. cᵀx = c₁x₁ + c₂x₂ + ... + cₙxₙ. This is what we're maximizing.
Each row i of A gives one constraint: aᵢ₁x₁ + ... + aᵢₙxₙ ≤ bᵢ. These are your resource limits.
Variables can't go negative. You can't plant −10 acres. This is standard in most real LP problems.
The variables x can be any real numbers (not just integers!). This is what makes LP tractable — as soon as you force x to be integers, you get Integer Linear Programming (ILP), which is NP-hard. The LP relaxation (allowing real values) is what we exploit in approximation algorithms.

Worked Example — Translating to Standard Form

Farm problem: maximize profit with corn (x₁, $3/acre) and wheat (x₂, $5/acre). Constraints: 100 acres total, 280 units of water (corn needs 2/acre, wheat needs 4/acre).

maximize  3x₁ + 5x₂   // c = [3, 5]
subject to
  x₁ + x₂  ≤ 100   // land constraint
  2x₁ + 4x₂ ≤ 280   // water constraint
  x₁, x₂    ≥ 0     // can't plant negative acres

In matrix form: A = [[1,1],[2,4]], b = [100, 280], c = [3, 5], x = [x₁, x₂].

💡 Notice: this has 2 variables and 2 constraints (plus non-negativity). Real LPs can have thousands of variables and constraints — but the structure is always the same.

Converting to Standard Form

Your notes show that ALL LP variants can be converted to standard form. Some common transformations:

Original formStandard form conversion
Minimize cᵀxMaximize −cᵀx (flip the sign of c)
Constraint: aᵀx ≥ bMultiply both sides by −1: −aᵀx ≤ −b
Equality: aᵀx = bTwo constraints: aᵀx ≤ b AND −aᵀx ≤ −b
Unbounded variable xᵢ (can be negative)Replace xᵢ = xᵢ⁺ − xᵢ⁻ where xᵢ⁺, xᵢ⁻ ≥ 0
The fact that ALL LP variants convert to standard form is why we only need to design one LP solver — and why standard form is the universal representation.
Part 3 / 8 — The Geometry of LP

Constraints Define a Region — The Feasible Set

Every linear constraint cuts the space in half. The set of all points satisfying ALL constraints is called the feasible region. For 2 variables this is a polygon (or empty, or unbounded).

The objective function cᵀx is a family of parallel lines (hyperplanes in higher dimensions). Moving these lines in the direction of c increases the objective. The optimal solution is where this moving line last touches the feasible region.

💡 The key geometric fact: the optimal solution is always at a vertex (corner point) of the feasible region — unless the LP is unbounded or infeasible. This is why LP is solvable: there are finitely many vertices to check.

Three Possible Outcomes for an LP

Constraints define a bounded polygon. Optimal solution exists at a vertex. This is the normal case.
Constraints contradict each other. No point satisfies all of them simultaneously. No solution exists.
Feasible region extends to infinity in the direction of the objective. Objective can grow forever. No finite optimal.
Part 4 / 8 — LP Duality

Every LP Has a "Mirror" — The Dual

For every LP (called the primal), there's a corresponding dual LP that encodes the same information from the opposite perspective. This is one of the deepest ideas in LP theory.

The dual of the standard form primal:

// PRIMAL:
maximize  cᵀx
s.t.       Ax ≤ b,  x ≥ 0

// DUAL:
minimize  bᵀy
s.t.       Aᵀy ≥ c,  y ≥ 0

The dual has m variables (one per primal constraint) and n constraints (one per primal variable). It's like the primal flipped upside down.

Where Do the Dual Variables Come From?

Here's the intuition with the farm example. Each constraint has a "shadow price" — how much would the objective improve if you relaxed that constraint by one unit?

If you had 1 more acre of land, how much more profit could you make? y₁ captures this.
If you had 1 more unit of water, how much more profit? y₂ captures this.

The dual LP asks: what are the "prices" of resources such that the total cost of all resources is minimized, while still being consistent with the objective values?

The dual for our farm LP:
minimize 100y₁ + 280y₂
subject to: y₁ + 2y₂ ≥ 3 (corn constraint)
            y₁ + 4y₂ ≥ 5 (wheat constraint)
            y₁, y₂ ≥ 0

Weak and Strong Duality

Two fundamental theorems connect the primal and dual:

Weak Duality: For any feasible primal solution x and any feasible dual solution y: cᵀx ≤ bᵀy. Any dual solution gives an UPPER BOUND on the primal optimum.
Strong Duality: At optimality, cᵀx* = bᵀy*. The primal and dual optimal values are EQUAL. The dual optimal IS the primal optimal.

This should feel familiar — you've seen this before! Max-Flow = Min-Cut is EXACTLY LP strong duality applied to the flow LP. The primal is max flow, the dual is min cut, and strong duality says they're equal at optimality.

💡 Max-Flow Min-Cut theorem = LP Strong Duality theorem applied to network flow. The two theorems are the same idea in different clothing. This is not a coincidence — it's because max flow IS an LP.

Complementary Slackness

One more key theorem from your notes: Complementary Slackness. At optimality, for each constraint either:

The constraint holds with equality (Aᵢx = bᵢ). The resource is fully used — at a bottleneck.
The shadow price of that resource is 0. Having more of it wouldn't help — it's not the bottleneck.
In the farm example: at optimum (60,40), both constraints are tight (60+40=100, 2·60+4·40=280). Both resources fully used → both dual variables y₁,y₂ > 0. This is complementary slackness in action.
Part 5 / 8 — Max Flow is an LP

You've Been Solving LPs All Along

Max flow — which you mastered in Sessions 2-4 of the original course — is secretly a linear program. Let's write it out explicitly and you'll see exactly how it maps to LP standard form.

Given a flow network G=(V,E) with capacities c(e), find a flow that maximizes total flow from s to t.

Writing Max Flow as an LP

The decision variables are the flow values on each edge: one variable f(e) per edge e ∈ E.

maximize   Σe=(s,v) f(e)   // total flow leaving s
subject to
  f(e) ≤ c(e)                // capacity constraint: ∀e ∈ E
  Σe into v f(e) = Σe out of v f(e)   // conservation: ∀v ≠ s,t
  f(e) ≥ 0                    // non-negativity: ∀e ∈ E
This is a valid LP! Objective is linear. All constraints are linear (in)equalities. Variables are the flows f(e) ≥ 0. This means max flow is in P via LP solvers — consistent with what you already know from Edmonds-Karp.

Mapping to Standard Form A, b, c

Let's make this concrete on a tiny example. Graph: s→A (cap 3), s→B (cap 2), A→t (cap 2), B→t (cap 3). Variables: f₁=f(s,A), f₂=f(s,B), f₃=f(A,t), f₄=f(B,t).

maximize  f₁ + f₂   // total flow out of s
s.t.
  f₁ ≤ 3               // cap(s,A)
  f₂ ≤ 2               // cap(s,B)
  f₃ ≤ 2               // cap(A,t)
  f₄ ≤ 3               // cap(B,t)
  f₁ − f₃ = 0         // conservation at A: in=f₁, out=f₃
  f₂ − f₄ = 0         // conservation at B: in=f₂, out=f₄
  f₁,f₂,f₃,f₄ ≥ 0

From conservation: f₁=f₃ and f₂=f₄. So we can simplify to: maximize f₁+f₂ subject to f₁≤2 (binding: min of cap(s,A)=3 and cap(A,t)=2), f₂≤2 (min of cap(s,B)=2 and cap(B,t)=3). Optimal: f₁=2, f₂=2, total flow = 4.

✅ The LP solution IS the max flow! And notice: the LP solution is automatically integer here (all capacities are integers). This is because the constraint matrix for flow networks is totally unimodular — a special structure that guarantees integer LP solutions when b is integer.

Totally Unimodular — Why Flow LP Has Integer Solutions

A matrix A is totally unimodular (TU) if every square submatrix has determinant 0, 1, or −1. For TU constraint matrices with integer b, the LP always has an integer optimal solution — no rounding needed.

The constraint matrix for flow networks is TU because each column (edge variable) appears in at most two rows: +1 in the conservation constraint of its head vertex, and −1 in the conservation constraint of its tail vertex.

This explains something that might have seemed like magic before: why Ford-Fulkerson always finds integer flows when capacities are integers. It's because max flow is an LP with a TU constraint matrix — the LP solution is automatically integer. No special handling needed.
⚠️ Not all LP formulations have TU constraint matrices. For example, Set Cover (Session 4) does NOT — which is precisely why LP rounding is needed there and why we can't just solve the LP and get an integer solution for free.
Part 6 / 8 — LP Relaxation: The Bridge to Approximation

The Key Trick for Approximation Algorithms

Many NP-hard problems can be written as Integer Linear Programs (ILP) — LPs where variables must be integers (0 or 1 for decision problems). ILP is NP-hard.

The LP relaxation removes the integrality constraint — allowing variables to be any real number in [0,1] instead of just 0 or 1. This makes the problem tractable (LP is in P), but the solution might be fractional.

Strategy: solve the LP relaxation (fast, polynomial), then round the fractional solution to integers. The rounding introduces approximation error — but a bounded one that gives us an α-approximation.

Concrete Example: Vertex Cover as ILP then LP Relaxation

Vertex Cover: for each vertex v, let xᵥ ∈ {0,1} (1 = in the cover). Minimize the number of vertices chosen such that every edge is covered.

// Integer Linear Program (ILP) — NP-hard:
minimize  Σᵥ xᵥ
s.t.       xᵤ + xᵥ ≥ 1   // for each edge (u,v): at least one endpoint covered
          xᵥ ∈ {0,1}   // integer constraint — makes this NP-hard

// LP Relaxation — polynomial:
minimize  Σᵥ xᵥ
s.t.       xᵤ + xᵥ ≥ 1   // same constraints
          0 ≤ xᵥ ≤ 1   // RELAXED — allows fractions
The LP relaxation can give fractional solutions like xᵥ = 0.5 for all vertices. To get a valid integer cover, you round up: include v if xᵥ ≥ 0.5. This rounding gives a 2-approximation for Vertex Cover — the full analysis is Session 4.

The Integrality Gap

The integrality gap of an ILP is the worst-case ratio between the LP relaxation optimal and the ILP optimal:

integrality gap = maxinstances ( ILP optimal / LP relaxation optimal )

The integrality gap is always ≥ 1 (LP relaxation is at least as good as ILP — it has more freedom). The gap tells you the best approximation ratio you can hope for from LP rounding.

Integrality gap = 2. LP rounding gives a 2-approximation. This is tight — there are instances where LP gives 0.5 everywhere.
Integrality gap = O(log n). LP rounding gives O(log n)-approximation. This too is tight.
Integrality gap = the price you pay for rounding. Understanding it is key to analyzing LP-rounding-based approximation algorithms.
Part 7 / 8 — How LPs Are Solved & The Big Picture

The Simplex Method (Conceptually)

The most famous LP algorithm is the Simplex method. The key insight from geometry: optimal is always at a vertex. So Simplex just walks along the edges of the feasible polytope, moving to better vertices until no improvement is possible.

SIMPLEX WALK — step through the algorithm on the farm LP

Simplex starts at a feasible vertex (origin). It moves to adjacent better vertices. Click Step.
Simplex is not polynomial in the worst case (exponential on adversarial inputs) but works extremely well in practice. The Ellipsoid method (1979) was the first proven polynomial-time LP algorithm. Interior point methods are polynomial and fast in practice.

Problem 1 — Write the LP (Draw This!)
Convert a word problem to standard form LP

A factory produces two products: chairs (profit $40 each) and tables (profit $90 each). Making one chair requires 2 hours of labor and 3 kg of wood. Making one table requires 5 hours of labor and 2 kg of wood. Available: 40 hours of labor, 24 kg of wood. How many of each to maximize profit?

On your board: (1) Write the LP in standard form with A, b, c explicitly. (2) Identify the feasible region vertices. (3) Solve by checking all vertices.

Solution

Variables: x₁ = chairs, x₂ = tables.

maximize  40x₁ + 90x₂
s.t.       2x₁ + 5x₂ ≤ 40   // labor
          3x₁ + 2x₂ ≤ 24   // wood
          x₁, x₂ ≥ 0

Matrix form: A = [[2,5],[3,2]], b = [40,24], c = [40,90].

Feasible vertices:

• (0,0): profit = 0

• (8,0): x₁=8 from 3x₁≤24. Profit = 320

• (0,8): x₂=8 from 5x₂≤40. Profit = 720

• Intersection: solve 2x₁+5x₂=40 AND 3x₁+2x₂=24. From first: x₁=(40−5x₂)/2. Sub: 3(40−5x₂)/2 + 2x₂ = 24 → 60−7.5x₂+2x₂=24 → −5.5x₂=−36 → x₂≈6.55, x₁≈(40−32.7)/2≈3.64. Profit = 40(3.64)+90(6.55) = 145.6+589.5 ≈ 735.1

Optimal: x₁≈3.64 chairs, x₂≈6.55 tables, profit≈$735.

Note: these are fractional. In an ILP you'd round: (3,6) gives 40(3)+90(6)=120+540=660, or (4,6) gives 160+540=700 but check feasibility: 2(4)+5(6)=38≤40 ✅, 3(4)+2(6)=24≤24 ✅. So ILP optimal = (4,6), profit=700.

Problem 2 — LP Duality
Write the dual and verify strong duality holds

Given primal LP:

maximize 2x₁ + 3x₂
s.t. x₁ + 2x₂ ≤ 4
     2x₁ + x₂ ≤ 6
     x₁, x₂ ≥ 0

(a) Write the dual LP.

(b) Solve the primal (by vertices). What is the optimal value?

(c) Find a dual feasible solution with objective value equal to the primal optimal. Verify strong duality.

Solution

(a) Dual LP: m=2 constraints → 2 dual variables y₁, y₂. n=2 primal variables → 2 dual constraints.

minimize  4y₁ + 6y₂
s.t.       y₁ + 2y₂ ≥ 2   // dual constraint for x₁
          2y₁ + y₂  ≥ 3   // dual constraint for x₂
          y₁, y₂ ≥ 0

(b) Primal vertices:

• (0,0): obj = 0

• (3,0): from 2x₁≤6 → x₁=3. obj = 6

• (0,2): from x₂≤2 → x₂=2. obj = 6

• Intersection: x₁+2x₂=4 AND 2x₁+x₂=6. Subtract: x₁−x₂=2 → x₁=x₂+2. Sub: x₂+2+2x₂=4 → 3x₂=2 → x₂=2/3, x₁=8/3. obj = 2(8/3)+3(2/3) = 16/3+6/3 = 22/3 ≈ 7.33

Primal optimal = 22/3 at (8/3, 2/3).

(c) Find dual solution with value 22/3. At optimum, both primal constraints are tight (binding). By complementary slackness, both dual variables are nonzero. Solve: y₁+2y₂=2 AND 2y₁+y₂=3 (dual constraints tight since primal variables x₁,x₂>0). Same system: y₁=4/3, y₂=1/3.

Dual objective: 4(4/3)+6(1/3) = 16/3+6/3 = 22/3. ✅ Equal to primal. Strong duality verified.

Problem 3 — LP Relaxation vs ILP
Find the integrality gap on a small instance

Consider the following Vertex Cover ILP on a triangle graph (vertices {1,2,3}, edges {(1,2),(2,3),(1,3)}):

minimize x₁ + x₂ + x₃
s.t. x₁ + x₂ ≥ 1 (edge 1-2)
     x₂ + x₃ ≥ 1 (edge 2-3)
     x₁ + x₃ ≥ 1 (edge 1-3)
     xᵢ ∈ {0,1}

(a) What is the optimal integer solution? (ILP optimal)

(b) Relax to LP (allow xᵢ ∈ [0,1]). What is the LP optimal?

(c) What is the integrality gap on this instance?

(d) If you round up all xᵢ ≥ 0.5, what cover do you get? What is the approximation ratio?

Solution

(a) ILP optimal: Triangle graph — every pair of vertices shares an edge. You must include at least 2 vertices (e.g., {1,2}) to cover all 3 edges. Check {1,2}: covers (1,2)✅, (2,3)✅, (1,3)✅. Optimal = 2.

(b) LP relaxation optimal: Set x₁=x₂=x₃=0.5. Check all constraints: 0.5+0.5=1≥1 ✅ for all edges. Objective = 1.5. Can we do better? If any xᵢ < 0.5, say x₁=0.4, then x₂+x₃≥1 from the other edges, and x₂≥1−x₁=0.6, giving total > 0.5+0.6+0.4=1.5. By symmetry, x₁=x₂=x₃=0.5 is optimal. LP optimal = 1.5.

(c) Integrality gap = ILP optimal / LP optimal = 2 / 1.5 = 4/3 ≈ 1.33. This instance achieves a gap of 4/3, which is actually the known integrality gap for Vertex Cover LP.

(d) Round up all xᵢ ≥ 0.5: All three variables = 0.5, so all round up to 1. Cover = {1,2,3}, size = 3. Approximation ratio = 3/2 = 2 = α. This matches the 2-approximation guarantee for the LP-rounding algorithm.

The triangle is the canonical tight example for Vertex Cover LP — integrality gap exactly 4/3, rounding gives exactly 2×. The 2-approximation is NOT because the gap is 2 (it's only 4/3), but because rounding can be wasteful.
Problem 4 — Max Flow as LP (Write it out)
Convert a flow network to LP standard form

Flow network with vertices {s, u, v, t} and edges: s→u (cap 4), s→v (cap 3), u→t (cap 2), v→t (cap 5), u→v (cap 2).

On your board:

(1) Write the LP with all variables and constraints. Name variables fₑ for each edge e.

(2) Solve the LP to find max flow and optimal flow assignment.

(3) Identify the min cut and verify max flow = min cut.

Solution

Variables: f₁=f(s,u), f₂=f(s,v), f₃=f(u,t), f₄=f(v,t), f₅=f(u,v).

maximize  f₁ + f₂
s.t.
  f₁ ≤ 4, f₂ ≤ 3, f₃ ≤ 2, f₄ ≤ 5, f₅ ≤ 2   // capacities
  f₁ = f₃ + f₅                            // conservation at u
  f₂ + f₅ = f₄                            // conservation at v
  all fₑ ≥ 0

Solving: Bottlenecks: u→t limited to 2. Even if we push 4 into u, only 2 can leave via u→t. The other 2 can go u→v→t. v→t has cap 5, so it can absorb: 2 from u→v plus up to 3 from s→v. s→v limited to 3. Total via v→t = min(5, 3+2) = 5, but s→v only gives 3 and u→v gives 2, so 5 units reach t via v→t.

Optimal: f₁=4, f₂=3, f₃=2, f₄=5, f₅=2. Total flow = f₁+f₂ = 7. Verify: u conservation: 4=2+2 ✅. v conservation: 3+2=5 ✅.

Max flow = 7.

Min cut: Cut {s,u,v} | {t}. Crossing edges: u→t (cap 2), v→t (cap 5). Cap = 2+5 = 7. ✅ Min cut = max flow = 7.

Problem 5 — Conceptual: LP vs ILP vs Approximation
Why can't we always just solve the LP and round?

This problem tests deep understanding of why LP relaxation + rounding doesn't always give good approximations.

(a) Max flow LP has TU constraint matrix → LP always gives integer optimal. Does this mean max flow ILP = max flow LP always? What would break if capacities were irrational?

(b) Vertex Cover has integrality gap 2. If someone claims "I found an instance with integrality gap 3," what would that imply about the complexity of Vertex Cover?

(c) Why is LP relaxation + rounding NOT a useful approach for k-EDP (edge-disjoint paths)? What goes wrong?

Solution

(a) Yes — max flow LP always equals max flow ILP when capacities are integers. TU + integer b guarantees the LP optimal is integer, so LP relaxation and ILP have the same optimal value and an optimal solution that works for both. If capacities are irrational: the TU property still holds (it's about the constraint matrix structure, not the values), so the LP still has a unique optimal. But the optimal flow values might themselves be irrational — which is fine for LP but causes Ford-Fulkerson to potentially not terminate (the irrational capacity problem from Session 2). This is a separate issue from LP vs ILP.

(b) An integrality gap of 3 would mean the LP relaxation is at most 1/3 of the ILP optimal. This would imply no LP-rounding algorithm can achieve better than a 3-approximation. But it wouldn't by itself say anything about the NP-hardness of Vertex Cover — integrality gap is a property of the LP formulation, not of the underlying problem. You could potentially find a different LP formulation with a smaller gap, or use a completely different algorithmic approach. The current best known integrality gap for Vertex Cover LP is 2, and it's conjectured (via the Unique Games Conjecture) that no poly-time algorithm can beat a 2-approximation.

(c) LP relaxation for k-EDP gives a fractional multi-commodity flow. In the LP relaxation, you can route 0.5 units of each commodity through shared edges. This fractional solution can achieve much higher "flow" than any integer EDP solution. When you try to round: you need to turn fractional paths into 0/1 paths (either routed or not), but rounding up all paths that carry any flow would violate the edge-disjoint constraint (multiple paths using the same edge). The integrality gap for multi-commodity flow is Θ(√n), which is large. So LP rounding for k-EDP gives the same √|E| approximation as greedy — and the LP approach is much more complex to analyze for no gain.
