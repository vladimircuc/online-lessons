Passing Arrays to Functions

Arrays are always passed by reference (technically by pointer). You don't need &. But you DO need to pass the size separately since the function doesn't know it.

void printArray(int arr[], int size) { for(int i = 0; i < size; i++) cout << arr[i] << " "; cout << endl; } int main() { int nums[5] = {10, 20, 30, 40, 50}; printArray(nums, 5); // pass array name + size return 0; }

Reading Strings with Spaces

⚠️ Common Trap

cin >> name stops at the first space. To read a full line, use getline:

string fullName; cout << "Enter name: "; getline(cin, fullName); // reads "Morgan Smith" completely

If you use cin >> before getline, you need cin.ignore() in between to clear the leftover newline character.

Functions: Pass by Value vs Reference

Pass by Value (copy)
void addTen(int x) { x = x + 10; // only changes the COPY } int num = 5; addTen(num); cout << num; // still 5!
Pass by Reference (&)
void addTen(int &x) { x = x + 10; // changes the ORIGINAL } int num = 5; addTen(num); cout << num; // 15!

Pointers: The Basics

What a pointer is, how to declare one, and the two key operators: & (address-of) and * (dereference).

What is a Pointer?

Every variable lives somewhere in your computer's memory. That "somewhere" has an address, like a house on a street. A pointer is a variable that stores the address of another variable.

💡 The Analogy

Think of it like this: a regular variable is a box that holds a value (like the number 42). A pointer is a piece of paper with the box's address written on it. The paper doesn't hold 42 itself, it tells you WHERE to find the box that holds 42.

Declaring a Pointer

int num = 42; // regular variable, holds the value 42 int* ptr = # // pointer to int, holds the ADDRESS of num

Breaking this down:

  • int* means "pointer to an int." The * in the declaration says "this is a pointer."
  • &num means "the address of num." The & operator gets the address.
  • So ptr now holds the memory address where num lives.

The Two Operators You Must Know

& (Address-of)

Put before a variable to get its memory address.

int x = 10; cout << &x; // prints address like 0x7fff5c

"Where does x live?"

* (Dereference)

Put before a pointer to access the value at that address.

int* ptr = &x; cout << *ptr; // prints 10 (the value at that address)

"What's stored at the address ptr holds?"

⚠️ Don't Confuse the Two Uses of *

int* ptr in a declaration = "ptr is a pointer to int" (creating it)

*ptr in an expression = "go to the address ptr holds and get the value" (using it)

Same symbol, two different meanings depending on context. This trips everyone up at first.

Quick Check

What is the output?

int a = 25; int* p = &a; cout << *p;

What is the output?

int x = 10; int* p = &x; *p = 99; cout << x;

Pointer Arithmetic

Moving pointers through memory, and the deep connection between pointers and arrays.

Pointers + Arrays: The Secret Connection

Here's the big reveal: an array name IS a pointer to the first element. When you write arr, it's the same as &arr[0].

int arr[5] = {10, 20, 30, 40, 50}; int* ptr = arr; // same as ptr = &arr[0] cout << *ptr; // 10 (first element) cout << *(ptr+1); // 20 (second element) cout << *(ptr+2); // 30 (third element)

When you add 1 to a pointer, it doesn't move 1 byte forward. It moves forward by the size of the data type. For an int*, adding 1 moves 4 bytes (the size of one int), landing exactly on the next element.

Common Pointer Arithmetic Patterns

// Iterating with pointer arithmetic instead of index int arr[5] = {10, 20, 30, 40, 50}; int* ptr = arr; // These are all equivalent: arr[3] // 40 (index notation) *(arr + 3) // 40 (pointer arithmetic) *(ptr + 3) // 40 (same thing, ptr == arr) ptr[3] // 40 (yes, you can use [] on pointers too!)
Looping With Pointers
// Print array using pointer movement int arr[5] = {10, 20, 30, 40, 50}; int* ptr = arr; for(int i = 0; i < 5; i++) { cout << *ptr << " "; ptr++; // move to next element }

Output: 10 20 30 40 50

Dynamic Memory

new, delete, and creating arrays whose size you don't know until the program runs.

The Problem: Unknown Size at Compile Time

Regular arrays need a size at compile time: int arr[100];. But what if the user decides the size?

int size; cout << "How many grades? "; cin >> size; // int arr[size]; // NOT standard C++! Some compilers allow it, but it's bad practice.

The solution: dynamic memory allocation with new.

new and delete

Single Variable
int* ptr = new int; // allocate one int on the heap *ptr = 42; // store a value cout << *ptr; // 42 delete ptr; // free the memory when done!
Dynamic Array
int size; cin >> size; int* arr = new int[size]; // allocate array of 'size' ints // Use it like a normal array! for(int i = 0; i < size; i++) cin >> arr[i]; for(int i = 0; i < size; i++) cout << arr[i] << " "; delete[] arr; // free the array when done! Note the []
⚠️ Critical Rules
new → deleteFor single variables
new[] → delete[]For arrays (MUST use brackets)
Always delete what you new. Forgetting = memory leak.

The null Pointer

int* ptr = nullptr; // points to nothing (safe default) if(ptr != nullptr) cout << *ptr; // only dereference if it's pointing to something!

Always initialize pointers. An uninitialized pointer holds garbage and dereferencing it crashes your program.

Quick Check

What's wrong with this code?

int* arr = new int[10]; for(int i = 0; i < 10; i++) arr[i] = i * 2; delete arr;

Pass by Address

The third way to pass arguments: using pointers instead of references.

Three Ways to Pass Arguments

MethodSyntaxCan modify original?
Pass by Valuevoid f(int x)No (works on a copy)
Pass by Referencevoid f(int &x)Yes (nickname for original)
Pass by Address (Pointer)void f(int *x)Yes (has the address of original)
Example: Swap with Pointers
void swap(int* a, int* b) { int temp = *a; // get value at address a *a = *b; // put b's value into a's location *b = temp; // put temp into b's location } int main() { int x = 3, y = 7; swap(&x, &y); // MUST pass addresses with & cout << x << " " << y; // 7 3 return 0; }
⚠️ Key Difference from Pass by Reference

With pass by reference (&): the caller just passes the variable normally: swap(x, y)

With pass by address (*): the caller must pass the address: swap(&x, &y)

Inside the function, you use *a to access the value (dereference) instead of just a.

Part 7 / 10

C-Strings & Pointers

Pointer tricks with character arrays, and the tricky exam question pattern.

C-Strings ARE Pointers

A C-string is just a char array ending with '\0'. And since array names are pointers, a C-string name is a char*.

char st[] = "Hello"; char* p = st; // p points to 'H' cout << *p; // H cout << *(p+1); // e cout << p; // prints entire string "Hello" (special char* behavior)
⚠️ Special cout Behavior

When you cout << ptr where ptr is a char*, it prints the entire string (not the address). This is different from int* which prints the address. If you want the address of a char*, use: cout << (void*)ptr;

The Exam Pattern: Pointer + Increment Through String

Common Exam Question
char st[] = "Once upon a time there was an ugly barnacle"; for(int i = 0; i < strlen(st); i++) st[i++] = toupper(st[i]);

Trace it: The loop variable i gets incremented twice per iteration: once by i++ inside the assignment (post-increment), and once by the loop's i++. So it processes every other character.

Output: OnCe uPoN A TiMe tHeRe wAs aN UgLy bArNaClE

Only alternate characters get uppercased because of the double increment.

Function Taking char*

This is how your professor's exam questions phrase it:

int count(char* st) { int c = 0; for(int i = 0; st[i] != '\0'; i++) if(st[i] == ' ') c++; return c; } // Counts spaces in a C-string

Inside the function, st[i] works normally. cout << *st[i] would be an error (double dereference on a single pointer).

Problem 1: Double or Half (Pointer Arithmetic)

Write a C++ program to:

  1. Declare a pointer to an integer variable
  2. Create an array of size 10 and read in the array. Assign the array to the pointer.
  3. Use pointer arithmetic to iterate through the array, replacing every positive element with triple the element and every negative element with the absolute value.
  4. Print the resulting array using pointer arithmetic.

Sample: Input 5 -3 8 -1 0 7 -6 2 -4 9 → Output 15 3 24 1 0 21 6 6 4 27

#include<iostream> using namespace std; int main() { int arr[10]; int* ptr; // Read in values cout << "Enter 10 values: "; for(int i = 0; i < 10; i++) cin >> arr[i]; // Assign array to pointer ptr = arr; // Modify using pointer arithmetic for(int i = 0; i < 10; i++) { if(*(ptr + i) > 0) *(ptr + i) = *(ptr + i) * 3; else if(*(ptr + i) < 0) *(ptr + i) = -(*(ptr + i)); // 0 stays 0 } // Print using pointer arithmetic for(int i = 0; i < 10; i++) cout << *(ptr + i) << " "; cout << endl; return 0; }

Problem 1: Pointer Arithmetic, Odd/Even Replace

Write a C++ program to:

  1. Declare a pointer to an integer variable
  2. Create an array of size 15 and read in the array. Assign the array to the pointer.
  3. Use pointer arithmetic to iterate through the array, replacing every odd element with twice the element and every even element with half the element.
  4. Print the resulting array using pointer arithmetic, this time moving the pointer to always point at the current element.

Sample: Input 23 61 72 18 -40 100 151 30 0 -8 -19 12 90 1 -7

Output: 46 122 36 9 -20 50 302 15 0 -4 -38 6 45 2 -14

#include<iostream> using namespace std; int main() { int arr[15]; int* ptr; // Read in values cout << "Enter the array values: " << endl; for(int i = 0; i < 15; i++) cin >> arr[i]; // Assign array to pointer ptr = arr; // Modify using pointer arithmetic for(int i = 0; i < 15; i++) { if(*(ptr + i) % 2 != 0) // odd *(ptr + i) = *(ptr + i) * 2; else // even *(ptr + i) = *(ptr + i) / 2; } // Print by MOVING the pointer (part d says "moving the pointer") ptr = arr; // reset to beginning cout << "The resulting array is" << endl; for(int i = 0; i < 15; i++) { cout << *ptr << " "; ptr++; // move pointer to next element } cout << endl; return 0; }
Key Points

Part (c) uses *(ptr + i) for read/write. Part (d) specifically says "moving the pointer," so we use ptr++ and *ptr instead. Note 0 is even, so 0/2 = 0 stays 0. Negative odd numbers like -19 become -38 and -7 becomes -14.

Problem 2: Complex Number Structure

A complex number is a number of the form x + iy, where x is the real part, and y is the imaginary part, with i representing the square root of -1.

Create a complex number structure called Complex, that contains 2 double data elements: real and img. Write a function called read that accepts an array of Complex types, the size of the array, and reads in the values. In the main function, create an array of 10 Complex variables. Use the read function to read in the values. Then add all the complex numbers and print the sum. (The real and imaginary parts are added separately.)

#include<iostream> using namespace std; struct Complex { double real; double img; }; void read(Complex arr[], int size); int main() { Complex nums[10]; read(nums, 10); // Add all complex numbers double totalReal = 0, totalImg = 0; for(int i = 0; i < 10; i++) { totalReal += nums[i].real; totalImg += nums[i].img; } cout << "Sum: " << totalReal << " + " << totalImg << "i" << endl; return 0; } void read(Complex arr[], int size) { for(int i = 0; i < size; i++) { cout << "Enter real and imaginary for #" << (i+1) << ": "; cin >> arr[i].real >> arr[i].img; } }
Notes

This uses a struct (which we'll cover in Session 4). For now, the pointer-relevant part is that the array of structs is passed to the function the same way as any array. The read function modifies the original because arrays are always passed by reference.

Problem 3: Timestamp Structure

Write a C++ program to:

  1. Create a structure called Timestamp. It has 5 integer values: year, month, day, hour, and minute.
  2. In main, create a Timestamp object and read in its value from the user in the format: MM/DD/YYYY HH:MM
  3. Check if the object is after 04/19/2026 23:59. If so, print "Yes". If not, print "No".

Sample: Enter the time: 04/20/2026 13:20 → Yes

#include<iostream> using namespace std; struct Timestamp { int year, month, day, hour, minute; }; int main() { Timestamp ts; char slash, colon; cout << "Enter the time: "; cin >> ts.month >> slash >> ts.day >> slash >> ts.year >> ts.hour >> colon >> ts.minute; // Compare with 04/19/2026 23:59 bool after = false; if(ts.year > 2026) after = true; else if(ts.year == 2026) { if(ts.month > 4) after = true; else if(ts.month == 4) { if(ts.day > 19) after = true; else if(ts.day == 19) { if(ts.hour > 23) after = true; else if(ts.hour == 23) { if(ts.minute > 59) after = true; } } } } if(after) cout << "Yes" << endl; else cout << "No" << endl; return 0; }
Key Tricks

Reading 04/20/2026 13:20 requires a char variable to "eat" the slashes and colon. The comparison works from most significant (year) to least significant (minute), using nested if-else chains.

Common Mistakes to Avoid

Don't Do These on the Exam
  • Using delete instead of delete[] for arrays
  • Forgetting to delete dynamic memory
  • Dereferencing a null or uninitialized pointer
  • Confusing * in declaration (int* p) vs expression (*p)
  • Double dereference: *st[i] when st is a char* (st[i] is already a char)
  • Forgetting & when calling pass-by-address: f(x) instead of f(&x)
