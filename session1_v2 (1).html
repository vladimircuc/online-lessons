Part 1 / 8 — Decision Problems & What "P" Really Means

Everything Starts with Decision Problems

In algorithms, a decision problem is any problem whose answer is YES or NO. This might seem restrictive — but it's not. Most optimization problems have a natural decision version:

Optimization versionDecision version
Find max flow from s to tIs there a flow of value ≥ k from s to t?
Find max matching in bipartite graphIs there a matching of size ≥ k?
Find shortest path from u to vIs there a path from u to v with ≤ k edges?
Find minimum set coverIs there a set cover of size ≤ k?
If you can solve the decision version efficiently, you can usually binary search on k to solve the optimization version too. So decision problems capture the full difficulty.

What Does "Polynomial Time" Actually Mean?

We say an algorithm runs in polynomial time if its runtime is O(nᶜ) for some constant c, where n is the input size. This is the formal definition of "efficient."

Why does this matter? Let's feel the difference with real numbers. Suppose you have a graph with n = 50 vertices:

RUNTIME COMPARISON FOR n = 50
✅ Polynomial algorithms (like DFS at O(V+E), max flow at O(VE²)) stay totally manageable even for large inputs. You ran these in previous sessions — they finished instantly.
❌ Exponential algorithms like O(2ⁿ) or O(n!) become completely unusable beyond n≈30. For n=50, 2⁵⁰ ≈ one quadrillion operations — years on any computer.

The Class P

P (Polynomial time) is the set of all decision problems that can be solved in polynomial time. These are the "tractable" problems — the ones we can actually handle in practice.

Problems you already know are in P:

O(V+E) — linear in graph size
O(VE²) Edmonds-Karp — polynomial
Reduces to max flow — polynomial
Part 2 / 8 — NP, NP-Hard, and the Big Question

NP Does NOT Mean "Not Polynomial"

This is the most common misconception. NP stands for Non-deterministic Polynomial. The formal definition: a problem is in NP if, given a claimed solution, you can verify whether it's correct in polynomial time.

Key distinction: solving vs verifying. Verification is easy — solving might not be.

Example you already know — Max Flow: If I hand you a claimed flow assignment and say "this is a valid flow of size 17," can you check it quickly? YES — just check capacity constraints on each edge and conservation at each vertex. That's O(V+E). So max flow verification is easy → max flow is in NP. (It's also in P since we can solve it.)

Now here's a harder example. Say I give you a graph G and claim "there's a way to color every vertex with 3 colors such that no two adjacent vertices share a color." Can you verify a claimed coloring quickly? YES — just check every edge. But can you find such a coloring efficiently? Nobody knows how.

The Complexity Landscape

Let's place the complexity classes carefully:

Solvable in poly time. Also verifiable (P ⊆ NP). DFS, max flow, matching all live here.
Verifiable in poly time. Contains P. Also contains NP-complete problems — harder ones we can't solve efficiently.
At least as hard as the hardest problem in NP. If you could solve an NP-hard problem fast, you could solve EVERYTHING in NP fast.
NP-Hard AND in NP. The "hardest" problems within NP itself. Edge-Disjoint Paths lives here.

The P vs NP Question — $1,000,000

The most famous open problem in computer science: does P = NP?

If P = NP: every problem whose solution can be quickly verified can also be quickly solved. Cryptography breaks. Drug discovery becomes trivial. AI becomes solved. Most researchers believe P ≠ NP — but nobody has proven it.

💡 For this course, we assume P ≠ NP. Under this assumption, NP-hard problems have no polynomial-time algorithm, and we need approximation algorithms instead.

Part 3 / 8 — SAT: The Foundation of NP-Hardness

Why We Need SAT

To prove a problem is NP-hard, you need a starting point — a problem that's already known to be NP-hard. The original one, proved by Cook and Levin in 1971, is Boolean Satisfiability (SAT).

Everything else chains back to SAT through reductions. Understanding SAT is essential before reductions make sense.

What is SAT?

You have boolean variables x₁, x₂, x₃, ... — each is either TRUE or FALSE. You also have clauses — OR combinations of variables (or their negations). The formula is an AND of all clauses.

Question: Is there an assignment of TRUE/FALSE to variables that makes the entire formula TRUE?

A formula is satisfiable if at least one assignment makes every clause true. The SAT problem asks: does such an assignment exist?

C₂: ¬x₁ ∨ x₂
C₃: x₁ ∨ x₂ ∨ ¬x₃
C₄: ¬x₁ ∨ ¬x₂ ∨ ¬x₃
Set variable values to check satisfiability...

Why is SAT Hard?

With n variables, there are 2ⁿ possible assignments. A brute-force solver checks all of them. For n=100 variables, that's 2¹⁰⁰ ≈ 10³⁰ assignments — completely intractable.

No polynomial-time algorithm for SAT is known. Cook's theorem (1971) proved SAT is NP-complete — it's in NP (verify a solution in poly time by just checking each clause), and it's NP-hard (any NP problem can be reduced to it).

🔑 SAT is the first NP-complete problem. Every other NP-hardness proof ultimately traces back to SAT. If you could solve SAT fast, you'd solve all of NP fast.

3-SAT: The Restricted Version We'll Use

3-SAT is SAT where every clause has exactly 3 literals. It's also NP-complete (SAT reduces to 3-SAT easily). We use 3-SAT as our starting point for reductions because its structure is easier to work with in gadget constructions.

Part 4 / 8 — Reductions: The Transfer of Hardness

What is a Polynomial-Time Reduction?

A polynomial-time reduction from problem A to problem B (written A ≤ₚ B) is a function f computable in polynomial time such that: x is a YES-instance of A if and only if f(x) is a YES-instance of B.

In other words: you can solve A by converting it to B, solving B, and reading the answer back. The conversion itself must be fast (polynomial time).

A ≤ₚ B means "A is no harder than B."
Equivalently: if B is easy (in P), then A is easy too.
Contrapositively: if A is hard (NP-hard), then B must also be hard.

Reduction Example: Max-Flow → Bipartite Matching

You already did this in Session 4! We reduced bipartite matching to max flow. Let's re-examine it as a formal reduction:

The transformation f:
Given bipartite graph G = (L ∪ R, E), build flow network H:
• Add super-source s with edges s→v for all v ∈ L (capacity 1)

• Run max flow on H

Result: max flow value = max matching size. This transformation is O(V+E) — polynomial!

This shows Bipartite Matching ≤ₚ Max Flow. Since max flow is in P, bipartite matching is also in P. The reduction proves tractability here.

The same logic works in reverse for hardness: if A is hard and A ≤ₚ B, then B is hard.

The NP-Hardness Proof Template

Every NP-hardness proof follows the same template. Memorize this:

To prove problem B is NP-hard:
1. Choose a known NP-hard problem A
2. Show how to transform any instance of A into an instance of B in poly time
3. Prove the transformation is correct: x ∈ A ⟺ f(x) ∈ B
4. Conclude: B is NP-hard (if B were easy, A would be easy — contradiction)
The chain always starts at SAT or 3-SAT (proven NP-complete by Cook-Levin). Then: 3-SAT → Independent Set → Vertex Cover → many others. Edge-Disjoint Paths is at the end of one such chain.
Part 5 / 8 — Edge-Disjoint Paths: Definition & the Max Flow Connection

The k-Edge-Disjoint Paths Problem

From your lecture notes: given a graph G = (V, E) and k pairs of vertices (s₁,t₁), (s₂,t₂), ..., (sₖ,tₖ), find k paths P₁, P₂, ..., Pₖ such that:

Each path Pᵢ connects sᵢ to tᵢ
No edge appears in more than one path (edge-disjoint)
Real-world interpretation: routing k phone calls through a network where each link can only carry one call at a time. Or routing k data streams with no wire conflicts.

The k=1 Case: This is Just Max Flow!

Before jumping to the hard general case, notice something important. When k=1 (one source-sink pair), edge-disjoint paths has a direct connection to what you already know.

Question: Does there exist at least 2 edge-disjoint paths from s to t in graph G?

This is equivalent to asking: is the max flow from s to t (with all capacities = 1) at least 2?

Click STEP to see how 2 edge-disjoint s→t paths = max flow of 2

All edge capacities = 1. Can we push flow 2 from s to t? That's exactly asking: do 2 edge-disjoint paths exist?
✅ 1-commodity EDP = Max Flow. For a single (s,t) pair: the maximum number of edge-disjoint s→t paths equals the max flow value when all capacities are 1. This is a direct consequence of the Max-Flow Min-Cut theorem.

Where Does It Break Down? The k=2 Case

With two different commodity pairs (s₁,t₁) and (s₂,t₂), max flow no longer directly applies. The issue: a single max flow treats all flow as fungible — it doesn't distinguish which unit of flow belongs to pair 1 vs pair 2.

With k ≥ 2 commodity pairs, you'd need to track which flow belongs to which source-sink pair. This is the multi-commodity flow problem — and it's NP-hard for integer flows.

This is precisely what your lecture notes highlight — the greedy algorithm and approximation strategies are needed specifically because the multi-commodity version can't be solved optimally in polynomial time.

Part 7 / 8 — Proving EDP is NP-Hard

The Jump From k=1 to k≥2

We just saw that 1-commodity EDP (single s,t pair) = max flow = polynomial time. But the moment you have multiple source-sink pairs that all need routing simultaneously, the problem becomes NP-hard.

The reduction comes from your lecture notes — it goes from 3-SAT to k-EDP. Here's the conceptual structure:

3-SAT
NP-Complete
≤ₚ
k-EDP
NP-Hard!

The Gadget Construction (Conceptually)

Given a 3-SAT instance with variables x₁...xₙ and clauses C₁...Cₘ, the reduction builds a graph G as follows:

Main path: A backbone of nodes going from s to t: s → v₁ → v₂ → ... → vₘ → t. One node per clause.
Variable gadgets: For each variable xᵢ, add a pair (sᵢ, tᵢ). The path for pair i can go through node vⱼ only if xᵢ appears (positively or negatively) in clause Cⱼ.
The key property: A valid set of k edge-disjoint paths (one per variable pair) exists if and only if the 3-SAT formula is satisfiable. Choosing which way a variable's path goes through each clause = setting that variable TRUE or FALSE.

Let's see a tiny example with 2 variables and 2 clauses: (x₁ ∨ x₂) ∧ (¬x₁ ∨ x₂)

GADGET: 3-SAT formula (x₁ ∨ x₂) ∧ (¬x₁ ∨ x₂) mapped to EDP instance

How to read this: The solid backbone goes s→C₁→C₂→t. Variable paths (dashed) thread through clause nodes they can satisfy. Path for x₁ can enter C₁ from above (satisfying C₁ with x₁=TRUE), then bypass C₂ above. Path for x₂ enters C₁ from below and exits C₂ from below. Edge-disjoint constraint means each clause node can only be "dipped into" by one variable path — that variable satisfies that clause. A valid edge-disjoint routing = a satisfying assignment.

What the Reduction Proves

The reduction shows: a valid set of k edge-disjoint paths exists in this constructed graph if and only if the 3-SAT formula is satisfiable.

If the formula is satisfiable, use the satisfying assignment to route variable paths through the clauses they satisfy. The edge-disjoint property holds because each clause is satisfied by exactly one variable.
If no satisfying assignment exists, some clause has no variable satisfying it. The corresponding clause node can't be "dipped into" by any path → no edge-disjoint routing exists.
✅ The transformation runs in polynomial time (graph has O(n+m) nodes). The correctness holds both ways. Therefore 3-SAT ≤ₚ k-EDP, which means k-EDP is NP-hard.

Problem 1 — Complexity Classification
Which class does each problem belong to?

For each problem below, determine whether it's in P, NP-Complete, or NP-Hard but not in NP. Justify each answer in 1-2 sentences.

(a) Given a directed graph G and vertices s, t: does a path from s to t exist?
(b) Given a flow network with integer capacities and a claimed max flow f: is f actually maximum?
(c) Given a graph G: is there a subset of vertices S such that every edge has at least one endpoint in S, with |S| ≤ k? (Vertex Cover)
(d) Given a bipartite graph G: does a perfect matching exist?
Complete Solution

(a) P. This is just DFS/BFS reachability. Run DFS from s — if you reach t, output YES, else NO. Runtime O(V+E). This is the definition of a tractable problem — you've been doing this since session 1.

(b) P. Verifying a max flow is in NP by definition. But checking if it's actually MAXIMUM is also in P — run Edmonds-Karp and compare. Or equivalently: check if there's an augmenting path in the residual graph. If no augmenting path exists (BFS finds nothing), the flow is maximum. O(VE²).

(c) NP-Complete. Vertex Cover is in NP: given a claimed cover S, verify in O(V+E) that every edge has an endpoint in S. It's also NP-hard via reduction from Independent Set: S is a vertex cover in G iff V\S is an independent set. (Every edge must be covered ↔ no two non-covered vertices form an edge.) Since Independent Set is NP-complete, so is Vertex Cover.

(d) P. You solved this in session 4! Reduce to max flow: add super-source and super-sink with capacity-1 edges. If max flow = |L|, perfect matching exists. Polynomial time via Edmonds-Karp. The key is it reduces to a problem already in P.

Problem 2 — Reduction Construction (Draw This!)
Reduce Independent Set to Vertex Cover

Given graph G = (V, E) and integer k, show that G has an independent set of size ≥ k if and only if G has a vertex cover of size ≤ |V| − k.

On your board: Draw a small graph (4-5 vertices), find an independent set, then identify the corresponding vertex cover. Verify the relationship |S| + |C| = |V| holds.

Then: why does this reduction prove that if you can solve Vertex Cover in polynomial time, you can solve Independent Set in polynomial time?

Complete Solution

The key observation: S ⊆ V is an independent set ↔ V\S is a vertex cover.

Proof (⟹): Suppose S is an independent set. Take any edge (u,v) ∈ E. Since S is independent, u and v cannot both be in S — at least one of them is in V\S. So V\S covers every edge. Therefore V\S is a vertex cover of size |V| − |S| ≤ |V| − k.

Proof (⟸): Suppose C = V\S is a vertex cover of size ≤ |V| − k. Then |S| = |V| − |C| ≥ k. For any edge (u,v), since C covers it, at least one of u,v is in C = V\S, meaning not both are in S. So S is an independent set of size ≥ k. ✅

Example: Path graph 1-2-3-4. Independent set S = {1,3} (no edges between them). Vertex cover C = {2,4}. |S| + |C| = 2+2 = 4 = |V|. Check: edges (1,2), (2,3), (3,4) — all have an endpoint in C. ✅

Why this proves hardness: The reduction f(G,k) = (G, |V|−k) is computable in O(1) — just change k! So Independent Set ≤ₚ Vertex Cover. Since IS is NP-hard, Vertex Cover is NP-hard too. If any poly-time algorithm for Vertex Cover existed, we'd apply it with k' = |V|−k to solve IS in poly time — contradiction with NP-hardness of IS.

Problem 3 — Edge-Disjoint Paths as Max Flow (Draw This!)
Is the max number of edge-disjoint s→t paths equal to the min cut?

Consider this graph with vertices {s, a, b, c, t} and edges: s→a, s→b, a→c, b→c, a→t, c→t.

On your board:

(1) Draw the graph with all edges labeled capacity=1.

(2) Find the maximum number of edge-disjoint paths from s to t by running max flow mentally.

(3) Find the minimum cut. Verify that min cut = max flow (Menger's theorem).

(4) List ALL distinct edge-disjoint paths.

Complete Solution

Max flow computation:

All capacities = 1. Augmenting paths:

• Path 1: s → a → t — bottleneck = 1. Flow = 1.

• Path 2: s → b → c → t — all edges still have residual. Bottleneck = 1. Flow = 2.

• Path 3: Try s→a→c→t — but a→c still has cap. s→a already used! Residual of s→a = 0. No augmenting path.

Max flow = 2.

Min cut: Cut {s} | {a,b,c,t}. Edges crossing: s→a (cap 1) and s→b (cap 1). Total = 2. ✅ Min cut = max flow = 2.

All edge-disjoint paths:

• s→a→t (uses edges s→a, a→t)

• s→b→c→t (uses edges s→b, b→c, c→t)

These two share no edges — they are edge-disjoint. Maximum is 2 (limited by the min cut of size 2 at s).

Problem 4 — Reduction Validity
Does this reduction work? Find the flaw (or confirm it's correct)

A student claims the following reduction from Bipartite Matching to SAT:

"Given bipartite graph G = (L ∪ R, E), create a boolean variable xₑ for each edge e ∈ E. Write a SAT formula that is satisfiable iff a matching of size k exists: for each vertex v, at most one of its adjacent edge-variables is TRUE (at-most-one constraint). Add a clause requiring at least k edge-variables to be TRUE."

(a) Is this reduction correct? Does it preserve YES/NO answers?

(b) What is its runtime? Is it polynomial?

(c) Does this prove that Bipartite Matching is NP-hard? Why or why not?

Complete Solution

(a) The reduction is correct in concept — but encoding "at most one of k variables is TRUE" requires O(k²) clauses (one for each pair). The at-most-one constraint for a vertex with degree d requires O(d²) clauses. Total formula size is O(|E|²). The YES/NO mapping is valid: a satisfying assignment with k TRUE edge-variables = a matching of size k.

(b) Runtime: Polynomial! O(|E|²) clauses, each of constant size. The transformation takes polynomial time. ✅

(c) This does NOT prove Bipartite Matching is NP-hard. This is a reduction FROM Bipartite Matching TO SAT — i.e., Bipartite Matching ≤ₚ SAT. For hardness you need the reduction to go FROM a hard problem TO Bipartite Matching. This reduction goes the wrong direction! It actually proves that Bipartite Matching is in NP (we can encode it as SAT, which is in NP), not that it's NP-hard. The fact that Bipartite Matching is efficiently solvable via max flow is what puts it in P — not anything about reductions from it to SAT.

Key lesson: the direction of a reduction matters enormously. A ≤ₚ B means A is no harder than B — it says nothing about whether B is hard. To prove B is hard, you need a hard problem A such that A ≤ₚ B.
Problem 5 — EDP Impossibility (Draw This!)
Prove that 3 edge-disjoint paths cannot exist in this graph

You have graph G with pairs (s₁,t₁), (s₂,t₂), (s₃,t₃). All three pairs share source s₁=s₂=s₃=s and target t₁=t₂=t₃=t (same s and t).

The graph has edges: s→a, s→b, a→c, b→c, c→t. All capacities = 1.

On your board:

(1) Draw the graph.

(2) Without computing max flow, identify why 3 edge-disjoint paths from s to t cannot exist. Use the min-cut argument.

(3) What IS the maximum number of edge-disjoint paths? Show them.

(4) If you changed the capacity of edge c→t to 3, what would the max EDP become? Why?

Complete Solution

(2) Min-cut argument: Consider the cut ({s,a,b,c}, {t}). The only edge crossing this cut is c→t with capacity 1. Therefore by Max-Flow Min-Cut theorem, max flow ≤ 1. Since max EDP = max flow (all caps = 1), at most 1 edge-disjoint path can exist — let alone 3.

Alternatively: look at the cut ({s}, {a,b,c,t}). Edges s→a and s→b cross it. Capacity = 2. So max flow ≤ 2 from this cut. But the tighter cut gives max = 1.

(3) Maximum EDP = 1. The only path is s→a→c→t (or s→b→c→t, but they share c→t so you can only use one). Max EDP = 1 because the min cut is 1 (the c→t edge bottleneck).

(4) If c→t has capacity 3: Now the cut ({s,a,b,c},{t}) has capacity 3. But the cut ({s},{a,b,c,t}) still has capacity 2 (edges s→a and s→b). So the binding constraint is now at s — max flow = 2. You can route: s→a→c→t AND s→b→c→t — both use c→t but now c→t has capacity 3 so it can carry 2 units. Max EDP = 2.

The maximum number of edge-disjoint paths is always determined by the minimum cut. Identifying the bottleneck edge or cut is the key to analyzing EDP instances.
