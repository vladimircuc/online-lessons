Part 1 / 8 — Writing Duals: The Mechanical Recipe

The Dual in 30 Seconds

In Session 3 we saw that every LP has a dual. Now we need to write duals quickly and correctly on the exam. Here is the complete mechanical recipe.

Start from the standard-form primal:

PRIMAL (standard form): maximize cᵀx subject to Ax ≤ b x ≥ 0 DUAL: minimize bᵀy subject to Aᵀy ≥ c y ≥ 0

The transformation is purely mechanical: the matrix A gets transposed, the objective vector and the constraint vector swap roles, ≤ becomes ≥, and maximize becomes minimize.

The General Recipe (Any LP)

Real problems don't always arrive in standard form. Here is the full correspondence table that handles every case:

Primal ↔ Dual Correspondence Rules
Primal (max)Dual (min)
Objective: max cᵀx↔Objective: min bᵀy
Constraint i: (Ax)ᵢ ≤ bᵢ↔Variable yᵢ ≥ 0
Constraint i: (Ax)ᵢ ≥ bᵢ↔Variable yᵢ ≤ 0
Constraint i: (Ax)ᵢ = bᵢ↔Variable yᵢ unrestricted
Variable xⱼ ≥ 0↔Constraint j: (Aᵀy)ⱼ ≥ cⱼ
Variable xⱼ ≤ 0↔Constraint j: (Aᵀy)ⱼ ≤ cⱼ
Variable xⱼ unrestricted↔Constraint j: (Aᵀy)ⱼ = cⱼ

Each primal constraint spawns a dual variable. Each primal variable spawns a dual constraint. The direction flips.

🔑 Memory aid: "≤ constraints in a max problem give ≥ 0 dual variables." Think of it as: a ≤ constraint in a max problem is the 'normal' direction, so the dual variable is also 'normal' (non-negative). Anything that deviates from the standard form flips the dual correspondingly.

Worked Example: Vertex Cover Dual

Let's apply the recipe to the Vertex Cover LP relaxation from Session 4.

PRIMAL (Vertex Cover LP, a minimization): minimize Σᵥ xᵥ s.t. xᵤ + xᵥ ≥ 1 for each edge (u,v) ∈ E xᵥ ≥ 0 for all v ∈ V

Step 1: This is a minimization. By the table, the dual will be a maximization.

Step 2: There is one dual variable yₑ per primal constraint, so one yₑ per edge.

Step 3: Primal constraints are ≥ in a min problem (the "normal" direction for min), so dual variables yₑ ≥ 0.

Step 4: Primal variables xᵥ ≥ 0, so dual constraints are ≤ (normal direction for max dual). Each primal variable xᵥ appears in every edge constraint incident to v, so the dual constraint for v is: Σ(edges e incident to v) yₑ ≤ 1.

Step 5: Dual objective is the primal RHS: maximize Σₑ yₑ (since each bᵢ = 1).

DUAL (Fractional Matching LP): maximize Σₑ yₑ s.t. Σ_{e incident to v} yₑ ≤ 1 for each v ∈ V yₑ ≥ 0 for all e ∈ E
✅ The Vertex Cover dual is a fractional matching! Each edge gets a weight yₑ, and no vertex is "overloaded" (total weight of its edges ≤ 1). Strong duality says: min fractional vertex cover = max fractional matching. This is the LP version of König's theorem for bipartite graphs.

Part 2 / 8 — Weak Duality & Its Consequences

The Weak Duality Theorem

This is one of the most important theorems for the exam. It gives us free bounds on optimal values.

Weak Duality Theorem: For any feasible primal solution x and any feasible dual solution y: (max) cᵀx ≤ bᵀy (min) The primal objective is always ≤ the dual objective.

In words: any feasible dual solution gives an upper bound on the max primal value. Any feasible primal solution gives a lower bound on the min dual value. They can never cross.

Visualizing Weak Duality

Every primal feasible value sits to the left of every dual feasible value. They never cross. At optimality (strong duality), they meet.

Consequences for Approximation

Weak duality is the engine behind every LP-rounding proof. Here is why:

For Minimization (e.g. Vertex Cover)

LP_OPT ≤ ILP_OPT = OPT. Any dual feasible solution gives a lower bound on OPT. We prove: Rounded ≤ α · LP_OPT ≤ α · OPT.

For Maximization (e.g. Independent Set)

LP_OPT ≥ ILP_OPT = OPT. The LP gives an upper bound. We need: Rounded ≥ OPT/α. Harder to prove, since LP_OPT is on the wrong side.

Three Key Consequences

1. Certificates of Optimality

If you find a feasible primal x and feasible dual y with cᵀx = bᵀy, then both are optimal. No further computation needed. This is because weak duality says cᵀx ≤ OPT_P ≤ OPT_D ≤ bᵀy, so if the endpoints are equal, everything in between must be equal too.

2. Infeasibility Detection

If the primal is unbounded (objective → ∞), then the dual must be infeasible (no feasible y exists). If the dual is unbounded (objective → -∞ for min), the primal must be infeasible.

3. Bounding OPT for NP-Hard Problems

For an NP-hard problem formulated as an ILP, we can't find OPT efficiently. But we can solve the LP relaxation. Weak duality guarantees that LP_OPT is a valid bound on OPT, which is exactly what we use in rounding proofs.

🔑 Strong Duality (which holds for LP but not for ILP) says that at optimality, primal = dual. So LP_OPT is simultaneously the best lower bound (for min problems) or best upper bound (for max problems) that any LP-based argument can provide. The gap between LP_OPT and ILP_OPT is the integrality gap.

Quick Self-Check

For feasible x (Ax ≤ b, x ≥ 0) and feasible y (Aᵀy ≥ c, y ≥ 0): cᵀx ≤ (Aᵀy)ᵀx = yᵀAx ≤ yᵀb = bᵀy. The first ≤ uses Aᵀy ≥ c and x ≥ 0. The second ≤ uses Ax ≤ b and y ≥ 0. Done.

Part 3 / 8 — Independent Set: Full Pipeline

What is Independent Set?

MAXNP-HARD

Maximum Independent Set: Given an undirected graph G = (V, E), find the largest subset S ⊆ V such that no two vertices in S are adjacent (no edge connects any pair in S).

Independent Set is the complement of Vertex Cover: S is an independent set if and only if V \ S is a vertex cover. If OPT_VC is the min vertex cover, then OPT_IS = n - OPT_VC.

(i) Greedy Approximation

The natural greedy for Independent Set:

Greedy-IS(G): S ← ∅ while V is non-empty: pick vertex v with minimum degree in G add v to S remove v and all its neighbors from G return S

Why minimum degree? By choosing a low-degree vertex, we remove the fewest other vertices from consideration, keeping more candidates for the IS.

Analysis: What ratio do we get?

Let Δ = max degree of any vertex in G. When we add v to S, we remove v plus at most Δ neighbors, so at most Δ + 1 vertices total. Since the algorithm processes all n vertices this way:

|S| ≥ n / (Δ + 1)

Since OPT ≤ n, we get |S| ≥ OPT / (Δ + 1). This is a (Δ+1)-approximation for maximization (meaning our answer is at least 1/(Δ+1) of optimal).

⚠️ For general graphs, Δ can be as large as n-1 (e.g. a star graph), so this ratio is not a fixed constant. Independent Set is famously hard to approximate: it is NP-hard to get within n^(1-ε) for any ε > 0. But the (Δ+1)-approximation is the best simple greedy can achieve, and it is what the exam expects.

(ii) ILP Formulation

For each vertex v, define xᵥ ∈ {0, 1} (1 = in the IS, 0 = not).

maximize Σᵥ xᵥ subject to: xᵤ + xᵥ ≤ 1 for each edge (u,v) ∈ E xᵥ ∈ {0, 1} for all v ∈ V

Each constraint says: for every edge, at most one endpoint can be in the IS. The objective maximizes the number of chosen vertices.

(iii) The Dual

LP relaxation (replace xᵥ ∈ {0,1} with 0 ≤ xᵥ ≤ 1):

PRIMAL (IS LP Relaxation): maximize Σᵥ xᵥ s.t. xᵤ + xᵥ ≤ 1 for each edge (u,v) xᵥ ≥ 0 for all v (Note: xᵥ ≤ 1 is implied by edge constraints when v has at least one neighbor.)

Applying the mechanical recipe from Part 1: one dual variable yₑ per edge constraint, ≤ constraint in max → yₑ ≥ 0, primal variable xᵥ ≥ 0 → dual constraint ≥.

DUAL: minimize Σₑ yₑ s.t. Σ_{e incident to v} yₑ ≥ 1 for each v ∈ V yₑ ≥ 0 for all e ∈ E
✅ The IS dual is a fractional edge cover! Assign non-negative weights to edges such that every vertex is "covered" (total weight of its incident edges ≥ 1), while minimizing total weight. Compare with the Vertex Cover dual which was fractional matching, the "opposite" problem.

(iv) Rounding: Why It Fails for IS

Here is the critical issue. Consider any graph. The LP relaxation can always set xᵥ = 1/2 for every vertex. Check: for each edge (u,v), x_u + x_v = 1/2 + 1/2 = 1 ≤ 1. ✓

So LP_OPT ≥ n/2 always. But OPT_IS could be as small as 1 (on a complete graph Kₙ). The integrality gap = LP_OPT / OPT_IS = (n/2) / 1 = n/2, which is unbounded.

❌ Rounding fails for IS. If we round at threshold 1/2 (include v if xᵥ ≥ 1/2), we might include every vertex, which is not a valid IS on most graphs. Rounding down (include only if xᵥ > 1/2) could give an empty set. The LP relaxation is too loose for IS to give any bounded approximation via rounding. This stands in sharp contrast to Vertex Cover, where the same LP gives a tight 2-approximation.

On the exam, you should state: "LP rounding does not yield a useful approximation for IS because the integrality gap is unbounded (Ω(n)). The LP relaxation always has a feasible solution of value n/2 regardless of the true IS size."

Part 4 / 8 — Clique: The Complement of Independent Set

What is Clique?

MAXNP-HARD

Maximum Clique: Given an undirected graph G = (V, E), find the largest subset S ⊆ V such that every pair of vertices in S is connected by an edge.

A clique is a "fully connected" subgraph. Clique and Independent Set are intimately related:

🔑 Key relationship: S is a clique in G if and only if S is an independent set in the complement graph Ḡ (where Ḡ has an edge (u,v) exactly when G does not). So max clique in G = max independent set in Ḡ. Every result about IS transfers to Clique by complementing the graph.

Interactive: Graph vs. Complement

A Graph and Its Complement
GRAPH G
COMPLEMENT Ḡ

In G: {B,C,D} form a triangle (clique of size 3). In Ḡ: {A,E} form an independent set of size 2. Max clique in G = 3; max IS in Ḡ would also be 3 (check: {B,C,D} have no edges in Ḡ).

(i) Greedy Approximation

Greedy-Clique(G): S ← ∅ while V is non-empty: pick vertex v with maximum degree in G add v to S remove all vertices NOT adjacent to v (keep only v's neighbors, since any clique containing v must be within N(v)) return S

Equivalently, run Greedy-IS on the complement graph Ḡ. The analysis mirrors IS: this gives a (Δ̄+1)-approximation where Δ̄ is the max degree of Ḡ. Since Δ̄ can be up to n-1, the ratio is not a fixed constant.

Like IS, Clique is extremely hard to approximate: NP-hard to get within n^(1-ε).

(ii) ILP Formulation

maximize Σᵥ xᵥ subject to: xᵤ + xᵥ ≤ 1 for each NON-edge (u,v) ∉ E xᵥ ∈ {0, 1} for all v ∈ V

The constraint says: for every pair of vertices that are not connected, at most one can be in the clique. This is exactly the IS ILP on the complement graph.

(iii) The Dual

LP relaxation and dual follow the same structure as IS. One dual variable y_f for each non-edge f ∉ E:

DUAL: minimize Σ_{f ∉ E} y_f s.t. Σ_{non-edges f incident to v} y_f ≥ 1 for each v y_f ≥ 0 for all f ∉ E

This is a fractional edge cover on the complement graph.

(iv) Rounding

Same issue as IS: the LP relaxation sets xᵥ = 1/2 for all v, giving LP_OPT = n/2 regardless of true clique size. Integrality gap is unbounded, so LP rounding does not give a useful approximation for Clique either.

🎯 Exam strategy for Clique: State that Clique on G is equivalent to IS on Ḡ. Then reference your IS answers with "complement the graph" for each part. The ILP has non-edge constraints instead of edge constraints. The greedy picks max-degree vertices. Rounding fails for the same reason (unbounded integrality gap).

Part 5 / 8 — Hitting Set: Set Cover's Twin

What is Hitting Set?

MINNP-HARD

Hitting Set: Given a universe U = {1, ..., n} and sets S₁, S₂, ..., Sₘ (subsets of U), find the smallest subset H ⊆ U such that H ∩ Sⱼ ≠ ∅ for every j. In other words, H "hits" every set.

Hitting Set vs. Set Cover

These two problems are the same problem viewed from opposite sides of the incidence matrix. Consider an element-set incidence matrix A where Aⱼᵢ = 1 iff element i ∈ Sⱼ:

Set Cover

Choose columns (sets) to cover all rows (elements). minimize #columns chosen s.t. every row has a 1 in some chosen column.

Hitting Set

Choose rows (elements) to hit all columns (sets). minimize #rows chosen s.t. every column has a 1 in some chosen row.

Formally: Hitting Set with sets S₁,...,Sₘ over universe U is equivalent to Set Cover with the roles of elements and sets transposed. Every Hitting Set instance is a Set Cover instance (and vice versa) after relabeling.

(i) Greedy Approximation

Greedy-HittingSet(U, S₁,...,Sₘ): H ← ∅ while some set Sⱼ is not hit: pick element e that hits the most unhit sets add e to H return H

This is the same greedy strategy as Set Cover (pick the most "useful" element). It gives an O(ln m)-approximation where m is the number of sets, by the same analysis as the Set Cover greedy (which gives O(ln n) where n is the universe size).

Why? Think of it as Set Cover with transposed roles. Each time we pick an element, it "covers" some unhit sets. The harmonic-series charging argument from Set Cover applies identically, giving the ln m bound.

🔑 The greedy gives O(ln m). But LP rounding (Part iv below) gives the cleaner f-approximation where f = max set size. For the exam, you should know both: state the greedy with its ln m ratio for part (i), and the rounding with its f ratio for part (iv).

(ii) ILP Formulation

For each element i ∈ U, define xᵢ ∈ {0,1} (1 = in H, 0 = not).

minimize Σᵢ xᵢ subject to: Σ_{i ∈ Sⱼ} xᵢ ≥ 1 for each set Sⱼ, j=1,...,m xᵢ ∈ {0, 1} for all i ∈ U

Each constraint says: at least one element from each set must be in H.

Concrete Example

minimize x₁ + x₂ + x₃ + x₄ + x₅ subject to: x₁ + x₂ + x₃ ≥ 1 (S₁ = {1,2,3}) x₂ + x₄ ≥ 1 (S₂ = {2,4}) x₃ + x₅ ≥ 1 (S₃ = {3,5}) x₄ + x₅ ≥ 1 (S₄ = {4,5}) xᵢ ∈ {0, 1}

(iii) The Dual

LP relaxation: replace xᵢ ∈ {0,1} with 0 ≤ xᵢ ≤ 1. Apply the recipe:

One dual variable yⱼ per set constraint (j = 1,...,m). Primal is a min with ≥ constraints and xᵢ ≥ 0 variables.

DUAL: maximize Σⱼ yⱼ s.t. Σ_{j: i ∈ Sⱼ} yⱼ ≤ 1 for each element i ∈ U yⱼ ≥ 0 for all j

The dual says: assign a non-negative weight yⱼ to each set, such that for each element, the total weight of the sets containing it is at most 1. Maximize total weight. This is a fractional set packing.

✅ Duality connection: Hitting Set LP dual = fractional set packing. Just as Vertex Cover LP dual = fractional matching. The pattern is always: covering problem ↔ packing problem.

(iv) Rounding

Since Hitting Set is a minimization problem (like Vertex Cover and Set Cover), LP rounding works well.

Define f = max set size = max |Sⱼ|. The rounding rule:

Rounding Rule: include element i if xᵢ* ≥ 1/f

Feasibility: For any set Sⱼ with |Sⱼ| ≤ f, the constraint says Σ_{i ∈ Sⱼ} xᵢ* ≥ 1. Since there are at most f terms summing to ≥ 1, at least one must be ≥ 1/f (pigeonhole). That element is included. ✓

Approximation: For each included element: 1 ≤ f · xᵢ*. Summing: |H| ≤ f · LP_OPT ≤ f · OPT. This is an f-approximation. ✓

🎯 The proof is identical to Set Cover rounding from Session 4! Replace "sets" with "elements" and "elements" with "sets." The max set size f plays the same role as the max frequency f in Set Cover. Both give an f-approximation via the same pigeonhole argument.

Part 6 / 8 — Knapsack: The ILP Perspective

Knapsack: What We Know vs. What We Need

In Session 5, we built an FPTAS for Knapsack using scaling + DP. That gives (1-ε)-approximation for any ε. Now the exam asks us to also treat Knapsack through the ILP/dual/rounding lens.

(i) Greedy Approximation

Greedy-Knapsack(items, W): Sort items by value/weight ratio: v₁/w₁ ≥ v₂/w₂ ≥ ... Greedily pack items in this order until one doesn't fit. Let G = total value of items packed by greedy. Let v_max = value of the single most valuable item that fits in the knapsack. Return max(G, v_max).

Key insight: The greedy-by-ratio alone can be arbitrarily bad (consider one item worth $1000 with weight W, and many items worth $1 with weight 1). But taking the better of greedy and the single best item guarantees a 2-approximation.

Why is max(G, v_max) a 2-approximation?

Let the greedy pack items 1, 2, ..., k and let item k+1 be the first that doesn't fit. Let G = v₁+...+vₖ. If we could also add item k+1, we'd have G + v_{k+1}.

The LP relaxation (fractional knapsack) solves by ratio-order and splits the first item that doesn't fit. So LP_OPT ≤ G + v_{k+1} (the fractional solution can take a fraction of item k+1).

Since LP_OPT ≥ OPT: OPT ≤ G + v_{k+1}.

Now: max(G, v_max) ≥ max(G, v_{k+1}) ≥ (G + v_{k+1})/2 ≥ OPT/2.

(The inequality max(a,b) ≥ (a+b)/2 holds for any non-negative a, b.)

So max(G, v_max) ≥ OPT/2, which is a 2-approximation for maximization. ✓

(ii) ILP Formulation

For each item i, define xᵢ ∈ {0,1} (1 = take it, 0 = leave it).

maximize Σᵢ vᵢxᵢ subject to: Σᵢ wᵢxᵢ ≤ W (capacity constraint) xᵢ ∈ {0, 1} for all i

(iii) The Dual

LP relaxation: 0 ≤ xᵢ ≤ 1. But we also need the upper bound xᵢ ≤ 1 explicitly. In standard form:

PRIMAL LP: maximize Σᵢ vᵢxᵢ s.t. Σᵢ wᵢxᵢ ≤ W (one constraint, dual var: y) xᵢ ≤ 1 (n constraints, dual vars: zᵢ) xᵢ ≥ 0 for all i

Two types of constraints: the capacity constraint (dual variable y ≥ 0) and the upper bound constraints (dual variables zᵢ ≥ 0).

DUAL: minimize Wy + Σᵢ zᵢ s.t. wᵢy + zᵢ ≥ vᵢ for each item i y ≥ 0, zᵢ ≥ 0 for all i

Interpretation: y is the "price per unit of weight" (shadow price of capacity). zᵢ is the "bonus" for item i's upper bound constraint. The dual says: set prices so that each item's price (wᵢy + zᵢ) covers its value vᵢ, while minimizing the total "cost" of capacity and bounds.

If we omit the xᵢ ≤ 1 constraints (relying on the capacity constraint alone), the dual simplifies to:

minimize Wy s.t. wᵢy ≥ vᵢ for each i y ≥ 0

This forces y ≥ max(vᵢ/wᵢ) = the best value-to-weight ratio. So the dual just picks the ratio of the best item and multiplies by W. Simple, but less tight. The full dual with zᵢ variables gives a tighter bound.

(iv) Rounding for Knapsack

The LP relaxation of Knapsack (fractional knapsack) has a beautiful structure: sort by vᵢ/wᵢ ratio, pack items greedily, and split the first item that doesn't fit. At most one item is fractional in the LP solution.

LP Rounding for Knapsack: 1. Solve LP relaxation (fractional knapsack). 2. At most one item k has 0 < x_k* < 1. 3. Option A: round DOWN (drop item k). Total value = G. 4. Option B: take ONLY item k (if it fits). Value = v_k. 5. Return max(G, v_max) where v_max ≥ v_k.

This is exactly the same 2-approximation as the greedy! The LP relaxation confirms the greedy is doing the right thing.

✅ Knapsack summary: Greedy-by-ratio + best single item = 2-approximation. LP rounding gives the same. But the FPTAS from Session 5 does much better: (1-ε)-approximation in poly(n, 1/ε) time. Knapsack is the "easiest" NP-hard problem to approximate.

Part 7 / 8 — LP Modeling: Writing Problems as LPs

Why LP Modeling Matters

The exam (DPV Chapter 7 problems 7.2–7.6, 7.8, 7.27–7.29) tests your ability to take a word problem and express it as a linear program. This is a translation skill: identify the decision variables, write the objective, and list the constraints.

The Modeling Checklist

Step 1: Variables

What are you deciding? Each variable should represent a quantity you control. Label them clearly: xᵢ = amount of product i, fₑ = flow on edge e, etc.

Step 2: Objective

What are you optimizing? Write it as a linear function of your variables. Check: is it max or min?

Step 3: Constraints

What rules must be satisfied? Each rule becomes a linear inequality or equality. Don't forget non-negativity (x ≥ 0) if applicable.

Modeling Pattern: Network Flow as LP

You saw this in Session 3. It's the prototype for all LP modeling:

Variables: fₑ for each edge e Objective: maximize Σ_{e out of s} fₑ Constraints: fₑ ≤ cₑ (capacity) for all e Σ(in) fₑ = Σ(out) fₑ (conservation) for all v ≠ s,t fₑ ≥ 0 (non-negativity) for all e

Modeling Pattern: Shortest Path as LP

This is a classic DPV-style question. Given a directed graph with edge weights wₑ ≥ 0, find the shortest s-t path.

Variables: dᵥ for each vertex v (dᵥ = distance from s to v) Objective: maximize d_t Constraints: d_s = 0 d_v ≤ d_u + w(u,v) for each edge (u,v) (triangle inequality / relaxation)

The constraints say d_v ≤ d_u + w(u,v), which means d_v can be at most the distance along any path. Any feasible solution gives a lower bound on the true shortest-path distance. By maximizing d_t, we push it up to the tightest possible lower bound, which is the actual shortest path.

This is a neat trick: the LP "discovers" shortest paths by maximizing under relaxation constraints. The dual of this LP gives the max-flow formulation (LP duality connects shortest paths and max flow).

Modeling Pattern: Two-Player Zero-Sum Games

This appears in DPV 7.20. Player 1 chooses row i, Player 2 chooses column j. Payoff matrix Gᵢⱼ: Player 1 gets Gᵢⱼ, Player 2 loses Gᵢⱼ. Players use mixed strategies (probability distributions).

Player 1's LP (maximize worst-case payoff): Variables: p₁, p₂, ..., pₘ (probabilities), z Objective: maximize z Constraints: Σᵢ pᵢGᵢⱼ ≥ z for each column j Σᵢ pᵢ = 1 pᵢ ≥ 0

The variable z represents the guaranteed minimum payoff. Player 1 maximizes this guarantee. The dual gives Player 2's optimal strategy (minimize max loss), and strong duality gives the minimax theorem.

Modeling Pattern: Diet / Blending Problems

Classic LP word problems: choose amounts of foods to minimize cost while meeting nutritional requirements.

Variables: xⱼ = amount of food j purchased Objective: minimize Σⱼ cⱼxⱼ (total cost) Constraints: Σⱼ aᵢⱼxⱼ ≥ bᵢ (nutrient i requirement) xⱼ ≥ 0 (can't buy negative food)

The Big Patterns

Pattern 1: Min problems round well

Vertex Cover, Set Cover, Hitting Set are all minimization. LP relaxation gives a lower bound (LP_OPT ≤ OPT). Rounding at threshold 1/f gives f-approximation via pigeonhole. The proof template is always the same.

Pattern 2: Max problems round poorly

Independent Set, Clique have unbounded integrality gaps. LP relaxation gives an upper bound, but rounding can violate feasibility or produce empty solutions. You need fundamentally different techniques (FPTAS for Knapsack, nothing good for IS/Clique).

Pattern 3: Covering ↔ Packing duality

Every covering LP (Vertex Cover, Set Cover, Hitting Set) has a packing dual (Matching, Packing, Set Packing). The LP dual swaps the covering/packing perspective. Strong duality says their optima are equal.

Pattern 4: Weak Duality = Free Bounds

In every rounding proof, the chain is: Rounded ≤ α·LP_OPT ≤ α·OPT (for min). The second ≤ comes from weak duality. You never need to find OPT itself, just bound it via LP_OPT.

Exam Checklist: The Four Questions

For each problem (IS, Clique, Knapsack, Hitting Set), the exam asks four things. Here is your recipe:

The Four-Part Recipe
#QuestionWhat to write
(i) Greedy approximation State the greedy rule (what to pick at each step). Prove the ratio by bounding ALG in terms of OPT using a counting/charging argument.
(ii) ILP formulation Define binary variables. Write objective (max or min). Write constraints (one per "covering" or "packing" requirement). Don't forget x ∈ {0,1}.
(iii) Dual of the LP relaxation Drop integrality to get LP. Apply the mechanical recipe: swap max/min, transpose A, swap c and b, flip inequality directions. Name the dual variables and interpret them.
(iv) Rounding algorithm Solve LP relaxation. State the rounding rule (threshold 1/f for min problems). Prove feasibility via pigeonhole. Prove ratio via the chain: Rounded ≤ α·LP_OPT ≤ α·OPT. For IS/Clique: explain why rounding fails (unbounded gap). For Knapsack: round down + best single item = 2-approx.

What to Remember for LP Modeling Questions
TopicKey Points
Standard formmax cᵀx, s.t. Ax ≤ b, x ≥ 0. Convert min → max by negating c. Convert ≥ → ≤ by multiplying by -1. Convert = → two inequalities.
Writing the DualMechanical recipe from Part 1. One dual variable per constraint. Transpose A, swap c↔b, flip direction.
Weak DualityAny feasible primal ≤ any feasible dual (for max/min pair). Gives free bounds. Proof: one line of algebra.
Strong DualityAt optimality, primal = dual. Max-Flow Min-Cut is a special case.
Modeling trickVariables = decisions. Constraints = rules. Objective = goal. If something is "at most" → ≤. If "at least" → ≥. If "exactly" → =.
