Q1

Which operation has the lowest precedence?

a) *A    b) A.B    c) A->B    d) A[B]

Answer: a) *A

The dot (.), arrow (->), and subscript ([]) operators are all postfix/member-access operators with very high precedence. The dereference operator (*) is a unary prefix operator with lower precedence than all of them.

Q2

Which of the following declarations is NOT a valid c-string?

a) char you[] = {'b','i','g',0};

b) char will[] = "WOW";

c) char pass[256] = "";

d) char cop3014[4] = {'H','U','G','E'};

Answer: d)

Option d has exactly 4 chars in a size-4 array, leaving no room for the null terminator '\0'. Without '\0' at the end, it's a char array but NOT a valid c-string. Option a uses 0 (which is the same as '\0'), b auto-adds '\0', and c initializes to an empty string with '\0' at position 0.

Q3

Which of the following is NOT a C++ primitive type?

a) long    b) bool    c) string    d) int

Answer: c) string

string is a class from the C++ standard library (<string>), not a primitive/built-in type. long, bool, and int are all primitive types built into the language.

Q4

Which functions are NOT included in cmath?

a) int abs(int)    b) int pow(int, int)    c) int rand()    d) double sin(double)

Answer: c) rand()

rand() is in <cstdlib>, not <cmath>. abs, pow, and sin are all math functions found in <cmath>.

Q5

Given int foo(char b);, which is a valid overload?

a) int foo2(char b);

b) char foo(char c);

c) int foo(char c='z');

d) int foo(float f);

Answer: d)

Overloading requires the same function name with different parameter types. a) has a different name (foo2), not an overload. b) same name and same parameter type (char), only differs by return type, which doesn't count for overloading, so it's a redefinition error. c) same signature as the original (int foo(char)), the default value doesn't create a new overload. d) different parameter type (float vs char), valid overload.

Q6

Which would be valid cases for a switch statement?

a) an integer literal    b) a constant character    c) a non-constant boolean variable    d) a float literal

Answer: a) and b)

Switch cases must be constant integral expressions. Integer literals (a) and constant characters (b) qualify. A non-constant variable (c) is not a compile-time constant, so it can't be a case label. Floats (d) are not integral types and can't be used in switch at all.

For each function declaration, state whether it's legal or illegal. If illegal, state why.

1

ILLEGAL.

ofstream cannot be passed by value because its copy constructor is deleted. Stream objects must be passed by reference: ofstream& file.

2
int**** confuse(char[][1], int = 'c');

LEGAL.

Returns a pointer-to-pointer-to-pointer-to-pointer-to-int (unusual but valid). Takes a 2D char array (first dimension can be empty) and an int with default value 'c' (char implicitly converts to int, which is 99). Weird, but syntactically correct.

3

ILLEGAL.

Identifiers cannot start with a digit. "1st" is not a valid function name. It would need to be something like "first" or "_1st".

4
long short();

ILLEGAL.

"short" is a C++ reserved word (a primitive type). You cannot use reserved words as function names.

5

(Assume struct cat has been declared above)

cat* alyst();

LEGAL.

Returns a pointer to cat. The function name "alyst" is a valid identifier (making "cat" + "alyst" = "catalyst", clever naming).

LEGAL.

Returns a char, takes a cat struct parameter named "dog." The parameter name being "dog" is misleading but perfectly legal. ("char" + "acter" = "character")

LEGAL.

Returns a cat. Takes three parameters: a const cat by value (no name, which is allowed in declarations), a const cat by reference, and a cat by reference. Parameter names are optional in declarations.

Q1

Functions can pass parameters by value, reference, or address. Explain how each is distinct.

Pass by value void f(int x): a copy of the argument is made. Changes inside the function don't affect the original. Call: f(num).

Pass by reference void f(int &x): the parameter becomes an alias (nickname) for the original variable. Changes inside the function DO affect the original. Call: f(num).

Pass by address void f(int *x): the function receives a pointer holding the address of the original. You use *x to access/modify the value at that address. Call: f(&num).

Reference and address both allow modification of the original, but differ in syntax: reference uses & in the declaration and is used like a normal variable inside the function, while address uses * in the declaration, requires & at the call site, and uses *x to access the value.

Q2

Stack allocated arrays must have their size known at compile time. Why? And why do dynamically allocated arrays not have the same restriction?

Stack-allocated arrays (like int arr[100];) live in the function's stack frame. The compiler needs to know exactly how much stack space to reserve when it compiles the code, before the program ever runs. A variable size would mean the compiler can't calculate the frame size.

Dynamic arrays (like new int[n]) live on the heap, which is managed at runtime by the operating system. The heap can allocate any amount of memory on the fly when the program asks for it, so the size can be determined during execution.

Q3

Why is it a good idea to initialize pointers to null, and reset them to null when no longer in use?

An uninitialized pointer holds a garbage address. If you accidentally dereference it, you access random memory, which can crash your program or cause unpredictable behavior with no helpful error message. Initializing to nullptr means if you accidentally dereference it, you get a clear null pointer error instead of silent corruption.

Resetting to null after delete prevents "dangling pointers," where the pointer still holds the address of freed memory. You can also check if(ptr != nullptr) before using it to avoid errors.

Q4

What is a memory leak? Show an example and explain why it's a problem.

A memory leak occurs when dynamically allocated memory is never freed, making it permanently unavailable to the program.

int* p = new int(5); // allocate memory p = new int(10); // allocate NEW memory, old address is LOST // the first allocation can never be freed because no pointer references it anymore

This is a problem because leaked memory accumulates over time. In long-running programs (servers, games), repeated leaks can exhaust available memory and eventually crash the system.

Q5

Many functions of the c-string library are "unsafe," while their string library counterparts are "safe." Why?

C-string functions like strcpy, strcat, and gets don't check bounds. They blindly write characters into the destination buffer without checking if there's enough room. If the source is longer than the destination, they write past the end of the array, corrupting adjacent memory (buffer overflow).

The C++ string class manages its own memory automatically. It knows its own size, resizes when needed, and checks bounds. Operations like =, +, and .append() can't write past the end because the string object handles allocation internally.

Problem 1: 3D Dynamic Matrix

Read in three integers: height, width, depth. Dynamically allocate a 3D integer matrix of that shape. Populate it so each slot holds the sum of its indices (e.g., slot [h][w][d] = h+w+d). Print it as a series of 2D slices (one per depth level). Deallocate with no leaks.

Sample: H=2 W=3 D=4

0 1 2 1 2 3 2 3 4 3 4 5 1 2 3 2 3 4 3 4 5 4 5 6
#include<iostream> using namespace std; int main() { int H, W, D; cout << "Enter the H,W,D: "; cin >> H >> W >> D; // Allocate 3D array: array of pointers to arrays of pointers to ints int*** matrix = new int**[H]; for(int h = 0; h < H; h++) { matrix[h] = new int*[W]; for(int w = 0; w < W; w++) matrix[h][w] = new int[D]; } // Populate: each slot = sum of indices for(int h = 0; h < H; h++) for(int w = 0; w < W; w++) for(int d = 0; d < D; d++) matrix[h][w][d] = h + w + d; // Print as 2D slices (one per depth) for(int d = 0; d < D; d++) { for(int h = 0; h < H; h++) { for(int w = 0; w < W; w++) cout << matrix[h][w][d] << " "; cout << endl; } cout << endl; } // Deallocate in reverse order for(int h = 0; h < H; h++) { for(int w = 0; w < W; w++) delete[] matrix[h][w]; delete[] matrix[h]; } delete[] matrix; return 0; }
Key Points

3D dynamic allocation requires 3 levels of new: int*** for height, int** for each row, int* for each column. Deallocation must happen in reverse order (innermost first). The printing loops are depth-first to create 2D slices.

Problem 2: Caesar Cipher with Files

Create a function caesar taking an input stream and output stream. Read through the input, shift each letter 3 positions forward (wrapping: x→a, y→b, z→c). Leave non-letters unchanged. In main, read a filename, open it, call caesar with the file as input and cout as output.

Sample file "message.txt":

Alea iacta est. Veni, vidi, vici. What are you gonna do, stab me?

Output:

Dohd ldfwd hvw. Yhql, ylgl, ylfl. Zkdw duh brx jrqqd gr, vwde ph?
#include<iostream> #include<fstream> #include<string> using namespace std; void caesar(istream& in, ostream& out); int main() { string filename; cout << "Enter the filename: "; cin >> filename; ifstream inFile; inFile.open(filename); if(!inFile) { cout << "Error opening file" << endl; return 0; } caesar(inFile, cout); inFile.close(); return 0; } void caesar(istream& in, ostream& out) { char c; while(in.get(c)) { if(c >= 'a' && c <= 'z') out << (char)(('a') + (c - 'a' + 3) % 26); else if(c >= 'A' && c <= 'Z') out << (char)(('A') + (c - 'A' + 3) % 26); else out << c; } }
Key Points

Use istream& and ostream& as parameter types (not ifstream/ofstream), so the function works with any stream. Use in.get(c) to read character by character (including spaces and newlines). The shift formula (c - 'a' + 3) % 26 + 'a' handles wrapping: 'x'→'a', 'y'→'b', 'z'→'c'. Handle uppercase and lowercase separately. Non-letters pass through unchanged.
