What is a Graph? 🗺️

Before DFS makes sense, you need to understand what a graph is. Forget math for a second.

Think of a graph like a map of cities connected by roads. Each city is a vertex (V) and each road is an edge (E). That's it. G = (V, E) just means "a graph made of vertices and edges."

🏙️ Cities = Vertices (V)  |  Roads = Edges (E)  |  Map = Graph G

Edges can be directed (one-way street: A→B means you can go from A to B but NOT B to A) or undirected (two-way road). Your lecture uses directed graphs.

What is DFS? 🔦

Imagine you're exploring a cave with a flashlight. You pick a tunnel, go as deep as possible, hit a dead end, then backtrack and try the next tunnel. That's Depth-First Search.

DFS rule: always go deeper before going sideways. Only backtrack when you're stuck.

There are two functions your notes show:

Visits everything reachable from one starting vertex v. Goes deep first.
Calls Explore on EVERY vertex. Makes sure the whole graph gets visited, even disconnected parts.
Explore(G, v):
  visited[v] = True
  pre(v) ← record ARRIVAL time
  for each neighbor w of v:
    if not visited[w]:
      Explore(G, w)
  post(v) ← record DEPARTURE time

DFS(G):
  for v in V: visited[v] = False
  for v in V:
    if not visited[v]: Explore(G, v)

The key insight — it's recursive. Explore calls itself on neighbors. It goes deeper and deeper until it can't, then returns (backtracks). Think of a stack of function calls piling up.

⚡ Time complexity: O(|E| + |V|) — you visit every vertex once and check every edge once. Super fast!

Chapter 4 of 6

Edge Types — What DFS Reveals 🔍

When DFS runs, it classifies every edge in the graph into one of these types. This is huge because the types tell you about the structure of the graph.

Tree Edges

Edges that DFS actually used to travel. They form the "skeleton" of the DFS traversal — literally the paths the algorithm walked.

When we go from u to v via DFS for the first time → that's a tree edge.

Back Edges ← CYCLES!

An edge u→v where v is an ancestor of u in the DFS tree. It literally goes BACK up the tree.

🔥 Critical fact: A directed graph has a cycle IF AND ONLY IF DFS finds a back edge. No back edges = no cycles = it's a DAG.

In the previous demo, the edge C→A is a back edge. C was discovered through A→B→C, so A is an ancestor of C. Going C→A means going backward = cycle!

Cross Edges

Edges that connect nodes in different branches of the DFS tree, where neither is an ancestor of the other.

Cross edges only appear in directed graphs. They go from a later-visited branch to an earlier-completed one.
How to identify them using timestamps

Given an edge u → v, you can tell what type it is just from pre/post values:

Edge u → vConditionType
u → vv discovered through u🌲 Tree
u → vpre(v) < pre(u) < post(u) < post(v)⬅️ Back (v is ancestor)
u → vpost(v) < pre(u)✂️ Cross (v already finished)

Strongly Connected Components 🔄

This is the big one. A Strongly Connected Component (SCC) is a group of vertices where every vertex can reach every other vertex.

Think of it like a friend group where everyone can call everyone else. If you can get from A to B AND from B to A (maybe through other people), you're in the same SCC.

Click each group below to understand it:

The SCC Algorithm (Kosaraju's)

Your notes describe this algorithm to find ALL SCCs. Here's the idea in plain English:

Step 1: Run DFS on the reversed graph GR (flip all edge directions). Record the post-order finish times.
Step 2: Process vertices in decreasing post-order (highest post number first) on the ORIGINAL graph.
Step 3: Each DFS tree in step 2 = one SCC. Done!

Why reverse the graph? The vertex with the highest post number in GR is guaranteed to be in a "source" SCC — meaning nothing flows INTO it from outside. Running DFS from there on the original G captures exactly that SCC before moving on.

Chapter 6 of 6

Topological Sort + Big Picture 🏁

A topological sort orders the vertices of a DAG so that every edge points "forward." Think of it like ordering tasks where some must come before others.

🎓 Example: To take CS 6515, you need CS 6505 first. CS 6505 needs CS 3510 first. Topological sort gives you a valid order to take these courses.

The algorithm is shockingly simple:

Run DFS → output vertices in DECREASING post-order. That's it.

Why does this work? Because if there's an edge u→v in a DAG, then post(u) > post(v) — u always finishes AFTER v. So sorting by decreasing post puts u before v. ✅

Only works on DAGs! If there's a cycle (back edge), you can't do a topological sort — you'd need to take a class before its own prerequisite.
