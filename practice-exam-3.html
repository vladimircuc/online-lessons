For each case below, determine what happens in an attempt to match the pattern with the indicated data.

Section 2 of 5
Learning Target P.2
P.2: I can use pattern matching and algebraic datatypes to write data-directed functions/programs.

Scenario

A music streaming service tracks user activity. Each event is one of:

  • A play, with a song title and play duration in seconds.
  • A skip, with a song title.
  • A like, with a song title.

The service calculates an "engagement score" per event:

  • A play of 30 seconds or more scores 5 points; shorter plays score 1 point.
  • A skip scores 0 points.
  • A like scores 10 points.
data Event = Play String Integer
           | Skip String
           | Like String
           deriving (Show)

e1, e2, e3, e4 :: Event
e1 = Play "Viva la Vida" 180
e2 = Play "Intro" 8
e3 = Skip "Ad"
e4 = Like "Clair de Lune"

log1 :: [Event]
log1 = [e1, e3, e2, e4, Play "Outro" 45]

Note: These types do not derive Eq, so you cannot use == or /=, nor the show function, in place of pattern matching.

Q2.1 Write a Haskell function

Write a function score :: Event -> Integer that computes the engagement score for a given event.

Examples:

  • score e1 evaluates to 5 (Play of 180s ≥ 30s)
  • score e2 evaluates to 1 (Play of 8s < 30s)
  • score e3 evaluates to 0 (Skip)
  • score e4 evaluates to 10 (Like)
Show Solution
Solution
score :: Event -> Integer
score (Play _ dur)
  | dur >= 30 = 5
  | otherwise = 1
score (Skip _) = 0
score (Like _) = 10

Explanation using the 4-step template:

  1. Read the type: 3 constructors (Play, Skip, Like).
  2. One equation per constructor.
  3. Destructure: Play needs dur (ignore title); Skip and Like don't need the title.
  4. Guards: Play uses a guard on duration.
Q2.2 Write a Haskell function

Write a function totalScore :: [Event] -> Integer that sums the engagement scores across a list of events. You may reuse score.

Examples:

  • totalScore log1 evaluates to 21 (5 + 0 + 1 + 10 + 5)
  • totalScore [] evaluates to 0
Show Solution
Solution

Option A, recursion:

totalScore :: [Event] -> Integer
totalScore [] = 0
totalScore (e:es) = score e + totalScore es

Option B, using higher-order functions:

totalScore :: [Event] -> Integer
totalScore es = sum (map score es)

Both are correct.

Learning Target P.3
P.3: I can convert between the following list-processing approaches: recursion with patterns, list comprehensions, and higher-order functions (map and filter).

Consider the following two Haskell functions:

meadow :: [(Integer, Integer)] -> [Integer]
meadow ps = [ a - b | (a, b) <- ps, a > b ]

hill :: Integer -> [Integer] -> [Integer]
hill t [] = []
hill t (n:ns)
  | n * n > t = (n + 1) : hill t ns
  | otherwise = hill t ns
Q3.1 Fill in the blanks to write an equivalent function to hill:
hill1 :: Integer -> [Integer] -> [Integer]
hill1 t ns = map ________ (filter ________ ns)
Show Solution
Solution

Identify T and P from the recursion:

  • Keep branch: (n + 1) : hill t ns. So T(n) = n + 1.
  • Guard: n * n > t. So P(n) = n * n > t.

T is simple, can use a section (+1). P needs t, use a lambda.

hill1 :: Integer -> [Integer] -> [Integer]
hill1 t ns = map (+1) (filter (\n -> n * n > t) ns)
Q3.2 Fill in the blanks to write an equivalent function to hill:
hill2 :: Integer -> [Integer] -> [Integer]
hill2 t ns = [ ________ | ________________ ]
Show Solution
Solution

Same T and P. List comp template: [T n | n <- ns, P n]:

hill2 :: Integer -> [Integer] -> [Integer]
hill2 t ns = [ n + 1 | n <- ns, n * n > t ]
Q3.3 Write a recursive function

Write a recursive function meadow1 :: [(Integer, Integer)] -> [Integer] that uses patterns and is equivalent to meadow.

Show Solution
Solution

From the list comp:

  • T((a, b)) = a - b
  • P((a, b)) = a > b

Each element is a tuple, destructure in the pattern.

meadow1 :: [(Integer, Integer)] -> [Integer]
meadow1 [] = []
meadow1 ((a, b) : rest)
  | a > b     = (a - b) : meadow1 rest
  | otherwise = meadow1 rest

The nested pattern ((a, b) : rest) matches a non-empty list, destructures the head tuple, and names the rest.

Learning Target T.1
T.1: I can trace and explain the execution behavior of a recursive function defined over a tree-like data structure.

Consider the following Haskell code:

data BTree a = Empty
             | BNode a (BTree a) (BTree a)
             deriving (Show)

willow :: BTree Integer
willow = BNode 10 (BNode 4 Empty (BNode 7 Empty Empty))
                  (BNode 15 (BNode 12 Empty Empty) Empty)

echo :: BTree Integer -> [Integer]
echo Empty = [-1]
echo (BNode v left right)
  | v > 9     = echo right ++ [v]
  | otherwise = [v] ++ echo left
          10
        /    \
       4     15
        \     /
        7   12
Q4.1 Trace and explain

Trace and explain what happens when echo willow is evaluated.

Your answer should provide enough information to clearly demonstrate how the evaluation progresses: for each recursive call, explicitly indicate the result of the pattern matching (i.e., the bindings to variables), how the result is evaluated, and the final result of that function call.

Show Solution
Full trace
Call 1: echo (BNode 10 L R)
  where L = BNode 4 Empty (BNode 7 Empty Empty)
        R = BNode 15 (BNode 12 Empty Empty) Empty
  Match (BNode v left right):
    v = 10, left = L, right = R
  Guard: 10 > 9 = True → first branch
  Expr: echo R ++ [10]
  Need echo R first:

    Call 2: echo (BNode 15 (BNode 12 Empty Empty) Empty)
      Match: v = 15, left = BNode 12 Empty Empty, right = Empty
      Guard: 15 > 9 = True → first branch
      Expr: echo Empty ++ [15]

        Call 3: echo Empty → [-1] (base case)

      Back to Call 2: [-1] ++ [15] = [-1, 15]
      Returns [-1, 15].

  Back to Call 1: [-1, 15] ++ [10] = [-1, 15, 10]
  Returns [-1, 15, 10].
Final: echo willow = [-1, 15, 10]

Key observation: the left subtree (nodes 4 and 7) was never visited because at node 10 the guard v > 9 was True, so we only recursed on the right. Similarly at node 15, we only recursed on its right child (Empty). Node 12 is also never visited.

Setup

Recall the Haskell representation of trees:

data BTree a = Empty

cedar :: BTree Char
cedar = BNode 'A' (BNode 'x' Empty Empty)
                  (BNode 'B' (BNode 'm' Empty Empty) (BNode 'C' Empty Empty))

forest :: BTree Integer
forest = BNode 7 (BNode 3 (BNode 1 Empty Empty) (BNode 5 Empty Empty))

Q5.1 Write a recursive Haskell function

Write a recursive Haskell function

vowelLeaf :: BTree Char -> Bool

such that vowelLeaf tr determines whether there is at least one leaf node in tr whose label is a vowel (one of 'a','e','i','o','u','A','E','I','O','U').

A leaf is a node with both children Empty.

Examples:

  • vowelLeaf cedar evaluates to False (leaves are 'x', 'm', 'C', none are vowels)
  • vowelLeaf (BNode 'b' (BNode 'a' Empty Empty) (BNode 'z' Empty Empty)) evaluates to True (leaf 'a')
  • vowelLeaf Empty evaluates to False
  • vowelLeaf (BNode 'E' Empty Empty) evaluates to True (E is a vowel leaf)
Show Solution
Solution
vowelLeaf :: BTree Char -> Bool
vowelLeaf Empty = False
vowelLeaf (BNode v Empty Empty) = v `elem` "aeiouAEIOU"
vowelLeaf (BNode _ left right)  = vowelLeaf left || vowelLeaf right

Three equations:

  1. Empty → False (no leaves).
  2. Leaf case: check if value is a vowel using elem.
  3. Internal case: OR the subtrees.

Same leaf-aware pattern as smallLeaf from the real exam. The leaf equation must come before the general BNode equation.

Q5.2 Write a recursive Haskell function

Write a recursive Haskell function

trim :: Integer -> BTree Integer -> BTree Integer

such that trim num tr returns the tree obtained from tr by removing any node (plus its descendants) whose label is even AND less than num.

Recall that removing a node is equivalent to replacing it with an empty tree.

Examples:

  • trim 10 forest evaluates to: 
    BNode 7 (BNode 3 (BNode 1 Empty Empty) (BNode 5 Empty Empty))
        (BNode 9 Empty (BNode 12 Empty Empty))
    (8 is even AND < 10, so removed; 12 is even but NOT < 10, so kept)
  • trim 0 forest returns forest unchanged (nothing is < 0).
  • trim 100 forest removes every even node whose value is < 100. Result: 
    BNode 7 (BNode 3 (BNode 1 Empty Empty) (BNode 5 Empty Empty))
        (BNode 9 Empty Empty)
Show Solution
Solution
trim :: Integer -> BTree Integer -> BTree Integer
trim _   Empty = Empty
trim num (BNode v left right)
  | even v && v < num = Empty
  | otherwise          = BNode v (trim num left) (trim num right)

Explanation using the 3 questions:

  1. Empty answer: Empty.
  2. Data needed: v, left, right, plus the threshold num.
  3. Combine: if even v AND v < num, return Empty. Otherwise rebuild with pruned subtrees.

Same pattern as prune from the real exam, but with a compound condition using &&.

Verification for trim 10 forest:

  • 7: odd, skip condition. Rebuild.
  • 3: odd, skip. Rebuild. Children 1 (odd), 5 (odd), both stay.
  • 9: odd, skip. Rebuild.
    • 8: even AND < 10. Empty.
    • 12: even but NOT < 10. Stays as BNode 12 Empty Empty.
    Returns BNode 9 Empty (BNode 12 Empty Empty).
  • Result: BNode 7 (BNode 3 (BNode 1 Empty Empty) (BNode 5 Empty Empty)) (BNode 9 Empty (BNode 12 Empty Empty)). ✓
