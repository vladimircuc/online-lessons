Loops & Lists

Iteration and data collections. If she can trace a loop over a list, she can handle most of the exam. This is the highest-value skill to master.

In this lesson: 1. REPEAT n TIMES 2. REPEAT UNTIL 3. Lists (1-Based Indexing!) 4. List Operations: APPEND, INSERT, REMOVE 5. FOR EACH (Loops + Lists) 6. Common Patterns: Sum, Count, Min/Max, Filter 7. End of Lesson Review (18 Questions)

1. REPEAT n TIMES

The simplest loop. It runs the code block a fixed number of times. You don't get a counter variable, it just repeats.

REPEAT 3 TIMES
{
    DISPLAY("hello")
}

This prints "hello" three times. That's it.

Using REPEAT with a Counter

REPEAT doesn't give you an automatic counter, but you can make your own:

count ← 0
REPEAT 5 TIMES
{
    count ← count + 1
    DISPLAY(count)
}
← Displays: 1, 2, 3, 4, 5

Tracing a REPEAT Loop

The exam will give you a loop and ask what it outputs. Trace it step by step:

x ← 2
REPEAT 4 TIMES
{
    x ← x * 2
}
DISPLAY(x)
Iterationx beforex after (x*2)
124
248
3816
41632

DISPLAY → 32

Trace tables are your best friend. When the exam gives you a loop, draw a quick table with columns for each variable and a row for each iteration. This prevents mistakes and shows your work.

Quick Check: REPEAT

What does this display? sum ← 0 REPEAT 4 TIMES { sum ← sum + 3 } DISPLAY(sum)
Iteration 1: sum = 0+3 = 3 Iteration 2: sum = 3+3 = 6 Iteration 3: sum = 6+3 = 9 Iteration 4: sum = 9+3 = 12 DISPLAY → 12 (Adding 3 four times = 3*4 = 12)
What does this display? x ← 1 REPEAT 5 TIMES { x ← x + x } DISPLAY(x)
x doubles each time: 1 → 2 → 4 → 8 → 16 → 32 DISPLAY → 32 (x + x is the same as x * 2)
What does this display? result ← "" REPEAT 3 TIMES { result ← result + "ab" } DISPLAY(result)
"" → "ab" → "abab" → "ababab" DISPLAY → "ababab"

2. REPEAT UNTIL

This loop keeps running until the condition becomes true. It checks the condition BEFORE each iteration.

x ← 1
REPEAT UNTIL (x > 10)
{
    x ← x + 3
}
DISPLAY(x)
Checkx > 10?Actionx after
11 > 10? Nox = 1+34
24 > 10? Nox = 4+37
37 > 10? Nox = 7+310
410 > 10? Nox = 10+313
513 > 10? YesSTOP13

DISPLAY → 13

REPEAT UNTIL stops BEFORE executing when the condition is true. It checks first, then decides whether to run the body. So x ends at 13, not 10, because 10 > 10 is false (10 is NOT greater than 10), so it runs one more time.
Infinite loops. If the condition can NEVER become true, the loop runs forever. Example: if you REPEAT UNTIL (x > 10) but never change x inside the loop, it never stops.

Quick Check: REPEAT UNTIL

What does this display? n ← 100 REPEAT UNTIL (n < 10) { n ← n / 2 } DISPLAY(n)
100 < 10? No → n = 50 50 < 10? No → n = 25 25 < 10? No → n = 12.5 12.5 < 10? No → n = 6.25 6.25 < 10? Yes → STOP DISPLAY → 6.25
How many times does the body run? x ← 5 REPEAT UNTIL (x ≤ 0) { x ← x - 2 }
x=5: 5≤0? No → x=3 x=3: 3≤0? No → x=1 x=1: 1≤0? No → x=-1 x=-1: -1≤0? Yes → STOP Body runs 3 times. x ends at -1.

3. Lists (1-Based Indexing!)

A list is an ordered collection of values. Think of it as a numbered row of boxes.

colors ← ["red", "blue", "green"]
AP pseudocode uses 1-BASED indexing. The first element is at index 1, not 0. This is different from Python, Java, and most real programming languages. If she knows Python, this WILL trip her up. colors[1] = "red" colors[2] = "blue" colors[3] = "green" There is NO colors[0].

Accessing Elements

nums ← [10, 20, 30, 40]

DISPLAY(nums[1])    ← 10 (first element)
DISPLAY(nums[3])    ← 30 (third element)
DISPLAY(LENGTH(nums)) ← 4 (four elements in the list)

Changing Elements

nums ← [10, 20, 30]
nums[2] ← 99
← nums is now [10, 99, 30]

You can overwrite any element by assigning to its index.

Quick Check: Lists Basics

list ← ["a", "b", "c", "d", "e"] What is list[2]? What is list[5]? What is LENGTH(list)?
list[2] = "b" (1-based: 1=a, 2=b) list[5] = "e" (last element) LENGTH(list) = 5 Remember: index 1 is the first element!
nums ← [5, 10, 15, 20] nums[3] ← nums[1] + nums[2] DISPLAY(nums)
nums[1] = 5, nums[2] = 10 nums[3] ← 5 + 10 = 15... wait, it was already 15! Actually: nums[3] ← 5 + 10 = 15 nums = [5, 10, 15, 20] (unchanged in this case) But if the values were different, the overwrite would change the list. The key concept is that the RIGHT side evaluates first, THEN assigns.

4. List Operations: APPEND, INSERT, REMOVE

Reference Table

OperationWhat it does
APPEND(list, value)Adds value to the END of the list. List grows by 1.
INSERT(list, i, value)Inserts value at index i. Everything at i and after shifts RIGHT by 1. List grows by 1.
REMOVE(list, i)Removes the element at index i. Everything after shifts LEFT by 1. List shrinks by 1.
LENGTH(list)Returns how many elements are in the list.
list[i]Access element at index i (1-based).

APPEND: Add to End

list ← [1, 2, 3]
APPEND(list, 4)
← list is now [1, 2, 3, 4]
APPEND(list, 5)
← list is now [1, 2, 3, 4, 5]

INSERT: Add at Position (Things Shift Right)

list ← ["a", "b", "c"]
INSERT(list, 2, "X")
← list is now ["a", "X", "b", "c"]
← "X" went INTO index 2
← "b" shifted from index 2 to index 3
← "c" shifted from index 3 to index 4

REMOVE: Delete at Position (Things Shift Left)

list ← ["a", "b", "c", "d"]
REMOVE(list, 2)
← Removes "b" (index 2)
← list is now ["a", "c", "d"]
← "c" shifted from index 3 to index 2
← "d" shifted from index 4 to index 3
The index shift after REMOVE is the #1 list trap on the exam. After you remove an element, every element after it moves down by 1 index. If you're removing inside a loop, the indices change mid-loop and you can accidentally skip elements or go out of bounds.

Quick Check: List Operations

list ← [10, 20, 30, 40, 50] REMOVE(list, 3) DISPLAY(list) DISPLAY(list[3])
Remove index 3 (which is 30). list = [10, 20, 40, 50] list[3] is now 40 (it shifted left from index 4). DISPLAY list → [10, 20, 40, 50] DISPLAY list[3] → 40
list ← ["x", "y", "z"] INSERT(list, 1, "w") DISPLAY(list) DISPLAY(LENGTH(list))
INSERT at index 1 pushes everything right. list = ["w", "x", "y", "z"] LENGTH = 4 "w" is now at index 1, "x" moved to 2, etc.
list ← [1, 2, 3] APPEND(list, 4) REMOVE(list, 1) INSERT(list, 2, 99) DISPLAY(list)
Start: [1, 2, 3] APPEND 4: [1, 2, 3, 4] REMOVE index 1: [2, 3, 4] INSERT 99 at index 2: [2, 99, 3, 4] DISPLAY → [2, 99, 3, 4]

5. FOR EACH (Loops + Lists)

This is how you process every element in a list. The loop variable takes on each value in the list, one at a time.

animals ← ["cat", "dog", "fish"]

FOR EACH animal IN animals
{
    DISPLAY(animal)
}
← Displays: "cat", "dog", "fish" (one per iteration)

How FOR EACH Works

The variable animal is NOT an index. It's the actual value from the list. On each iteration, it becomes the next element:

Iterationanimal =Action
1"cat"DISPLAY "cat"
2"dog"DISPLAY "dog"
3"fish"DISPLAY "fish"
FOR EACH gives you the value, not the index. You can't use it to change the list (like setting list[i] ← something) because you don't have the index. Use it when you need to READ every element. Use index-based loops when you need to MODIFY elements.

Index-Based Loop (Alternative)

If you need the index (to modify elements or track position), use a counter loop:

i ← 1
REPEAT UNTIL (i > LENGTH(list))
{
    DISPLAY(list[i])
    i ← i + 1
}

This does the same as FOR EACH but gives you control over the index.

Quick Check: FOR EACH

What does this display? nums ← [2, 4, 6] total ← 0 FOR EACH n IN nums { total ← total + n } DISPLAY(total)
n=2: total = 0+2 = 2 n=4: total = 2+4 = 6 n=6: total = 6+6 = 12 DISPLAY → 12 (sum of all elements)
What does this display? words ← ["AP", "CS", "P"] result ← "" FOR EACH w IN words { result ← result + w + "-" } DISPLAY(result)
"" + "AP" + "-" = "AP-" "AP-" + "CS" + "-" = "AP-CS-" "AP-CS-" + "P" + "-" = "AP-CS-P-" DISPLAY → "AP-CS-P-" (Note the trailing dash at the end)

6. Common Patterns

These patterns show up on literally every AP CSP exam. Learn them cold.

Pattern 1: Sum All Elements

sum ← 0
FOR EACH num IN list
{
    sum ← sum + num
}
← sum now holds the total of all elements

Initialize to 0 before the loop. Add each element inside the loop.

Pattern 2: Count Elements That Match

count ← 0
FOR EACH num IN list
{
    IF (num > 50)
    {
        count ← count + 1
    }
}
← count = how many elements are greater than 50

Pattern 3: Find the Maximum

max ← list[1]
FOR EACH num IN list
{
    IF (num > max)
    {
        max ← num
    }
}
← max now holds the largest value in the list

Start max at the first element (not 0, because the list could have negative numbers). Update max whenever you find something bigger.

Pattern 4: Find Minimum

min ← list[1]
FOR EACH num IN list
{
    IF (num < min)
    {
        min ← num
    }
}

Same as max but flip the comparison.

Pattern 5: Filter into a New List

result ← []
FOR EACH num IN list
{
    IF (num MOD 2 = 0)
    {
        APPEND(result, num)
    }
}
← result contains only the even numbers from list

Start with an empty list. APPEND only the elements that pass the condition.

Don't initialize max/min to 0. If all numbers in the list are negative, max would incorrectly stay at 0. Always initialize to list[1] (the first actual element).

Quick Check: Patterns

list ← [3, 7, 2, 9, 1] What is the result of the find-max pattern?
max = list[1] = 3 3: 3>3? No 7: 7>3? Yes, max=7 2: 2>7? No 9: 9>7? Yes, max=9 1: 1>9? No max = 9
list ← [4, 11, 8, 3, 15, 6] count ← 0 FOR EACH n IN list { IF (n > 5) { count ← count + 1 } } DISPLAY(count)
4>5? No. 11>5? Yes(1). 8>5? Yes(2). 3>5? No. 15>5? Yes(3). 6>5? Yes(4). DISPLAY → 4 (four elements greater than 5)
list ← [1, 2, 3, 4, 5, 6] evens ← [] FOR EACH n IN list { IF (n MOD 2 = 0) { APPEND(evens, n) } } DISPLAY(evens)
1: odd, skip. 2: even, append. 3: skip. 4: even, append. 5: skip. 6: even, append. evens = [2, 4, 6]
