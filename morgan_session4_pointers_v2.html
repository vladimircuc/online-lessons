Pointers from Scratch

Let's take this slow. No rushing. By the end of this lesson, pointers will feel natural.

Step 1: Every Variable Has an Address

When you write int x = 42;, the computer picks a spot in memory and puts 42 there. That spot has a number called an address, like a house number on a street.

Picture It
Your variable: x
42
Address: 0x100

The variable x holds the value 42. It lives at address 0x100. You normally don't care about the address, you just use x.

Step 2: A Pointer Stores an Address

A pointer is just a variable that, instead of storing a number like 42, stores an address. That's it. It's a variable whose value is an address.

Real World Analogy

Regular variable = a box with a value inside (like a mailbox with a letter)

Pointer = a sticky note that says "the letter is in mailbox #100"

The sticky note doesn't have the letter. It tells you WHERE the letter is.

int x = 42; // x holds 42 int* ptr = &x; // ptr holds the ADDRESS of x

Let's break down that second line piece by piece:

int*This means "pointer to an int." The star says it's a pointer.
ptrThe name of our pointer variable. Could be any name.
=Assign it a value.
&xThe & gets the address of x. If x lives at 0x100, then &x = 0x100.

The Star: Dereference

Using * to follow the address and get (or change) the value at the other end.

Two Jobs for the Star *

The * symbol does two completely different things depending on where it appears. This is the #1 thing that confuses people.

Job 1: In a Declaration
int* ptr; // "ptr is a pointer to int"

Here * just means "this variable is a pointer." You're creating the pointer.

Job 2: Everywhere Else
cout << *ptr; // "go to the address and get the value" *ptr = 99; // "go to the address and CHANGE the value"

Here * means dereference: follow the address and access what's there.

Pointers & Arrays

The big connection: an array name is secretly a pointer to the first element.

The Rule

⭐ Key Fact

When you write int arr[5];, the name arr by itself is the same as &arr[0]: a pointer to the first element.

int arr[5] = {10, 20, 30, 40, 50}; int* ptr = arr; // ptr points to arr[0] cout << *ptr; // 10 (same as arr[0]) cout << *(ptr+1); // 20 (same as arr[1]) cout << *(ptr+3); // 40 (same as arr[3])

new and delete

Creating memory on the fly, and cleaning up when you're done.

Why Do We Need This?

Normal arrays need a fixed size: int arr[100];. But what if you don't know the size until the program runs? The user tells you "I have 7 grades," and you need an array of exactly 7.

int size; cout << "How many? "; cin >> size; int* arr = new int[size]; // creates array of exactly 'size' ints // use it like a normal array for(int i = 0; i < size; i++) cin >> arr[i]; // MUST free when done delete[] arr;

The Rules (just 3)

Memorize These
new int→delete ptr;For ONE variable
new int[n]→delete[] ptr;For an ARRAY (note the brackets!)
Every new must have a matching delete. No exceptions.

Part 5 / 10

Pass by Address

The third way to pass arguments to functions: with pointers.

Three Ways Compared

MethodDeclarationCallModifies original?
Valuevoid f(int x)f(num)No
Referencevoid f(int &x)f(num)Yes
Address (pointer)void f(int *x)f(&num)Yes
Example
void doubleIt(int* p) // receives an address { *p = *p * 2; // go to that address, double the value } int main() { int num = 5; doubleIt(&num); // pass the ADDRESS of num cout << num; // 10! return 0; }
⚠️ The & Difference

Pass by reference: the & goes in the function declaration → void f(int &x). Caller writes f(num).

Pass by address: the * goes in the declaration → void f(int *x). Caller must send the address → f(&num).

Part 6 / 10

C-Strings & Pointers

Character arrays, char*, and the exam tricks.

C-Strings ARE char Pointers

char st[] = "Hello"; // array of chars: H, e, l, l, o, \0 char* p = st; // p points to 'H' cout << *p; // H (first character) cout << *(p+1); // e (second character) cout << p; // "Hello" (prints the whole string!)
⚠️ Special Rule for char*

cout << ptr when ptr is int* → prints the address.

cout << ptr when ptr is char* → prints the entire string.

This is a special C++ rule. If you want the address, cast it: cout << (void*)ptr;

The Double Increment Trick (from your study guide)

Exam Pattern: What's the Output?
char st[] = "Once upon a time there was an ugly barnacle"; for(int i = 0; i < strlen(st); i++) st[i++] = toupper(st[i]);

Tricky part: i gets incremented twice each loop iteration:

  1. i++ inside st[i++] (post-increment: uses current i, THEN adds 1)
  2. i++ in the for loop header

So it processes positions 0, 2, 4, 6, ... (every other character gets uppercased).

Output: OnCe uPoN A TiMe tHeRe wAs aN UgLy bArNaClE

Problem 1: Basic Pointer

Write a program that:

  1. Creates an int variable with value 100
  2. Creates a pointer pointing to it
  3. Prints the value using the pointer (should print 100)
  4. Changes the value to 200 using the pointer
  5. Prints the variable directly (should print 200)
#include<iostream> using namespace std; int main() { int x = 100; int* ptr = &x; cout << *ptr << endl; // 100 *ptr = 200; cout << x << endl; // 200 return 0; }

Problem 2: Two Pointers

Write a program that:

  1. Creates two int variables: a = 10, b = 20
  2. Creates two pointers, one pointing to each
  3. Swaps the values using ONLY the pointers (don't touch a or b directly)
  4. Prints a and b (should show 20 and 10)
#include<iostream> using namespace std; int main() { int a = 10, b = 20; int* p1 = &a; int* p2 = &b; int temp = *p1; // temp = 10 *p1 = *p2; // a = 20 *p2 = temp; // b = 10 cout << a << " " << b << endl; // 20 10 return 0; }

Problem 3: Dynamic Array

Write a program that:

  1. Asks the user for a number n
  2. Creates a dynamic array of n integers
  3. Reads in n values from the user
  4. Prints them in reverse order
  5. Frees the memory

Sample: n=4, values: 10 20 30 40 → Output: 40 30 20 10

#include<iostream> using namespace std; int main() { int n; cout << "How many? "; cin >> n; int* arr = new int[n]; cout << "Enter " << n << " values: "; for(int i = 0; i < n; i++) cin >> arr[i]; for(int i = n - 1; i >= 0; i--) cout << arr[i] << " "; cout << endl; delete[] arr; return 0; }

Problem 4: Pass by Address Function

Write a function called tripleIt that takes an integer pointer and triples the value it points to. In main, create a variable with value 4, call tripleIt, and print the result (should be 12).

#include<iostream> using namespace std; void tripleIt(int* p); int main() { int num = 4; tripleIt(&num); cout << num << endl; // 12 return 0; } void tripleIt(int* p) { *p = *p * 3; }

Problem 5: Array Sum with Pointer Arithmetic

Write a program that:

  1. Creates an array of 8 integers and reads values from the user
  2. Creates a pointer to the array
  3. Uses pointer arithmetic (NOT array indexing) to find the sum of all elements
  4. Prints the sum

Sample: 3 7 1 4 2 8 5 6 → Sum: 36

#include<iostream> using namespace std; int main() { int arr[8]; cout << "Enter 8 values: "; for(int i = 0; i < 8; i++) cin >> arr[i]; int* ptr = arr; int sum = 0; for(int i = 0; i < 8; i++) sum += *(ptr + i); cout << "Sum: " << sum << endl; return 0; }

Problem 6: Dynamic Array with Function

Write a program that:

  1. Asks the user for a size
  2. Creates a dynamic array of doubles
  3. Reads in the values
  4. Write a function findMax that takes the array and size and returns the largest value
  5. Call the function and print the result
  6. Free the memory
#include<iostream> using namespace std; double findMax(double* arr, int size); int main() { int size; cout << "How many? "; cin >> size; double* arr = new double[size]; cout << "Enter values: "; for(int i = 0; i < size; i++) cin >> arr[i]; cout << "Max: " << findMax(arr, size) << endl; delete[] arr; return 0; } double findMax(double* arr, int size) { double max = arr[0]; for(int i = 1; i < size; i++) if(arr[i] > max) max = arr[i]; return max; }

Exam Problem 1: Pointer Arithmetic, Odd/Even Replace

Activity for 04/13/2026

Write a C++ program to:

  1. Declare a pointer to an integer variable
  2. Create an array of size 15 and read in the array. Assign the array to the pointer.
  3. Use pointer arithmetic to iterate through the array, replacing every odd element with twice the element and every even element with half the element.
  4. Print the resulting array using pointer arithmetic, this time moving the pointer to always point at the current element.

Sample Input: 23 61 72 18 -40 100 151 30 0 -8 -19 12 90 1 -7

Sample Output: 46 122 36 9 -20 50 302 15 0 -4 -38 6 45 2 -14

#include<iostream> using namespace std; int main() { int arr[15]; int* ptr; cout << "Enter the array values:" << endl; for(int i = 0; i < 15; i++) cin >> arr[i]; ptr = arr; // assign array to pointer // (c) modify using pointer arithmetic for(int i = 0; i < 15; i++) { if(*(ptr + i) % 2 != 0) // odd *(ptr + i) = *(ptr + i) * 2; else // even (including 0) *(ptr + i) = *(ptr + i) / 2; } // (d) print by MOVING the pointer ptr = arr; // reset to start cout << "The resulting array is" << endl; for(int i = 0; i < 15; i++) { cout << *ptr << " "; ptr++; // move pointer forward } cout << endl; return 0; }
Key Points
  • Part (c): use *(ptr + i) to read and write each element
  • Part (d): "moving the pointer" means use *ptr then ptr++, not *(ptr+i)
  • 0 is even, so 0/2 = 0
  • Negative odd: -19 * 2 = -38, -7 * 2 = -14
  • Negative even: -40 / 2 = -20, -8 / 2 = -4

Exam Problem 2: Output Tracing (from Study Guide Q8)

char st[] = "Once upon a time there was an ugly barnacle"; for(int i = 0; i < strlen(st); i++) st[i++] = toupper(st[i]); cout << st << endl;

The "Don't Do This" List

Common Exam Mistakes
  • delete arr; when you used new int[n] → must be delete[] arr;
  • *st[i] on a char* → double dereference error
  • f(val) when function expects int* → must be f(&val)
  • Forgetting to delete dynamic memory → memory leak
  • Dereferencing nullptr or uninitialized pointer → crash
  • Confusing * in declaration vs expression
